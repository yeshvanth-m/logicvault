Overview

A Tri-state Buffer (three-state buffer) is a digital circuit whose output can take one of three states: logic 0, logic 1, or high-impedance (Z). In the high-impedance state, the output is effectively disconnected from the circuit, allowing multiple outputs to share a single wire without conflict.

Tri-state buffers are essential for bus architectures in computers and embedded systems, where multiple devices (CPU, memory, I/O) need to drive a shared data bus. Only one device drives the bus at any time; all others present high-impedance outputs to avoid short circuits.

Truth Table

EN D Y State
0 0 Z High-impedance
0 1 Z High-impedance
1 0 0 Active low
1 1 1 Active high
Active-High Enable

EN=1 enables output. EN=0 → Hi-Z. Most common logic convention.

Y = EN ? D : Z
Active-Low Enable

EN̄=0 enables output. EN̄=1 → Hi-Z. Common in TTL ICs (74LS244).

Y = ĒN̄ ? D : Z

High-Impedance State

What is Hi-Z?

Output transistors are OFF, presenting very high resistance (~MΩ) to the wire.

Like an open switch
Why Hi-Z?

Allows multiple outputs on one wire. Only one active driver; others in Hi-Z.

Enables bus sharing
Floating Node

When all drivers are Hi-Z, wire floats to undefined state. Use pull-up/down resistors.

Risk: noise, metastability
Bus Contention

Two drivers active simultaneously → short circuit, damage, undefined value.

⚠️ Never enable 2+ drivers!

CMOS Tri-state Implementation

A CMOS tri-state buffer uses two transmission paths controlled by enable:

When EN=1 Both PMOS (pull-up) and NMOS (pull-down) active
When EN=0 Both transistor paths disabled → output floats
Transistor count ~6-8 transistors (vs 4 for standard buffer)

Bus Architecture

Tri-state buffers enable shared bus architectures where multiple devices communicate over common wires. Critical rule: only one driver enabled at any time.

RTL Implementation

VERILOG tristate.v 
// Single-bit tri-state buffer
module tristate_buf (
    input  d,
    input  en,
    output y
);

// 'z' represents high-impedance
assign y = en ? d : 1'bz;

endmodule

// N-bit tri-state buffer (bus driver)
module tristate_bus #(
    parameter WIDTH = 8
)(
    input  [WIDTH-1:0] d,
    input              en,
    output [WIDTH-1:0] y
);

assign y = en ? d : {WIDTH{1'bz}};

endmodule

// Bidirectional I/O pad
module bidir_io (
    inout  pin,        // External bidirectional pin
    input  data_out,   // Data to send
    output data_in,    // Data received
    input  output_en   // 1 = drive, 0 = receive
);

// Drive pin when output enabled
assign pin = output_en ? data_out : 1'bz;

// Always read pin (input buffer)
assign data_in = pin;

endmodule

// Shared bus with multiple drivers
module shared_bus (
    input  [7:0] data_a, data_b, data_c,
    input        en_a, en_b, en_c,
    output [7:0] bus
);

// WARNING: Only one enable should be high!
assign bus = en_a ? data_a :
              en_b ? data_b :
              en_c ? data_c : 8'bz;

endmodule
VHDL tristate.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;

entity tristate_buf is
    generic (WIDTH : integer := 8);
    port (
        d  : in  std_logic_vector(WIDTH-1 downto 0);
        en : in  std_logic;
        y  : out std_logic_vector(WIDTH-1 downto 0)
    );
end tristate_buf;

architecture behavioral of tristate_buf is
begin
    y <= d when en = '1' else (others => 'Z');
end behavioral;

-- Bidirectional I/O
entity bidir_io is
    port (
        pin       : inout std_logic;
        data_out  : in    std_logic;
        data_in   : out   std_logic;
        output_en : in    std_logic
    );
end bidir_io;

architecture behavioral of bidir_io is
begin
    pin <= data_out when output_en = '1' else 'Z';
    data_in <= pin;
end behavioral;

Applications

CPU Data Bus

CPU, RAM, ROM, I/O all share data bus using tri-state drivers.

Only selected device drives bus
Bidirectional I/O

GPIO pins that can be input or output. Tri-state when input mode.

MCU port pins
Memory Interface

Memory data pins are bidirectional — read or write but not both.

DDR, SRAM interfaces
Level Shifting

Tri-state buffers with different supply voltages for level translation.

3.3V ↔ 5V conversion

74LS245 Octal transceiver (8-bit, bidirectional)
74LS125 Quad buffer with individual enables
74LS126 Quad buffer (active-high enable)

Interview Notes

  • Tri-state has 3 states: 0, 1, and Z (high-impedance). Z = effectively disconnected.
  • Bus contention: Two drivers enabled → short circuit. Use one-hot enable decoding.
  • Floating bus: All Hi-Z → undefined voltage. Add pull-up/pull-down resistors.
  • In Verilog/VHDL, use 'z' or 'Z' for high-impedance. Synthesis infers tri-state buffers.
  • FPGAs: Internal tri-state largely replaced by MUXes. External I/O still uses tri-state.
  • Dead time: Brief period where old driver disables before new driver enables. Prevents contention.
