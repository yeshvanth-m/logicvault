Overview

A 2:1 Multiplexer (MUX) is a combinational circuit that selects one of two input signals and forwards it to a single output line. The selection is controlled by a single select line S: when S=0, input I₀ is passed to output; when S=1, input I₁ is passed.

Multiplexers are fundamental building blocks in digital design, used for data routing, bus arbitration, function generation, and implementing complex logic with minimal gates. A 2:1 MUX can implement any 2-variable Boolean function by connecting constants or variables to its inputs.

Truth Table

S I₁ I₀ Y Description
0 0 0 0 Select I₀ (0)
0 0 1 1 Select I₀ (1)
0 1 0 0 Select I₀ (0)
0 1 1 1 Select I₀ (1)
1 0 0 0 Select I₁ (0)
1 0 1 0 Select I₁ (0)
1 1 0 1 Select I₁ (1)
1 1 1 1 Select I₁ (1)
Simplified View
Y = I₀ when S = 0
Y = I₁ when S = 1

Boolean Expression

Multiplexer Equation
Y = S̄·I₀ + S·I₁
Ternary Operator Form

Common in HDL and programming:

Y = S ? I₁ : I₀
If-Then-Else Form

Behavioral description:

if (S == 0) Y = I₀
else Y = I₁

Derivation from Truth Table

Canonical SOP (minterms where Y=1):
Y = S̄·I₁̄·I₀ + S̄·I₁·I₀ + S·I₁·I₀̄ + S·I₁·I₀

Factor out common terms:
Y = S̄·I₀·(I₁̄ + I₁) + S·I₁·(I₀̄ + I₀)
Y = S̄·I₀·1 + S·I₁·1
Y = S̄·I₀ + S·I₁

Gate Implementations

Standard AND-OR

Direct implementation of the Boolean equation.

2 AND + 1 OR + 1 NOT
Delay: 2 gate levels
NAND-Only

Using DeMorgan's theorem.

4 NAND gates
Delay: 3 gate levels
Transmission Gate

CMOS pass transistor logic — very efficient.

2 transmission gates
Delay: ~1 gate delay
Tri-state Buffers

Using tri-state outputs on shared wire.

2 tri-state buffers
Requires complementary S

CMOS Implementation

In CMOS technology, a 2:1 MUX is typically implemented using transmission gates for minimal area and delay. Each transmission gate consists of an NMOS and PMOS transistor in parallel, passing the input signal when enabled.

Transistor Count 6 transistors (2 TG + inverter)
Propagation Delay ~0.5-1 gate delay

RTL Implementation

VERILOG mux2.v 
// 2:1 MUX - Behavioral (ternary operator)
module mux2_behavioral (
    input  i0,
    input  i1,
    input  sel,
    output y
);

assign y = sel ? i1 : i0;

endmodule

// 2:1 MUX - Dataflow (Boolean expression)
module mux2_dataflow (
    input  i0,
    input  i1,
    input  sel,
    output y
);

assign y = (~sel & i0) | (sel & i1);

endmodule

// 2:1 MUX - Parametrized width
module mux2 #(
    parameter WIDTH = 8
)(
    input  [WIDTH-1:0] i0,
    input  [WIDTH-1:0] i1,
    input              sel,
    output [WIDTH-1:0] y
);

assign y = sel ? i1 : i0;

endmodule
VHDL mux2.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;

entity mux2 is
    generic (WIDTH : integer := 8);
    port (
        i0  : in  std_logic_vector(WIDTH-1 downto 0);
        i1  : in  std_logic_vector(WIDTH-1 downto 0);
        sel : in  std_logic;
        y   : out std_logic_vector(WIDTH-1 downto 0)
    );
end mux2;

architecture behavioral of mux2 is
begin
    y <= i1 when sel = '1' else i0;
end behavioral;

-- Using with-select
architecture with_select of mux2 is
begin
    with sel select
        y <= i0 when '0',
             i1 when others;
end with_select;

Applications

Function Implementation

Any 2-variable function can be implemented by setting I₀, I₁ to 0, 1, S, or S̄.

Example: XOR = MUX(S, S̄, S)
Data Selection

Choose between two data sources based on control signal.

Building Larger MUX

Cascade 2:1 MUXes to build 4:1, 8:1, or larger multiplexers.

4:1 = three 2:1 MUXes
Conditional Assignment

Hardware equivalent of if-else statements in software.

Register file write-back mux

Implementing Logic Functions with 2:1 MUX

Use S as one variable, connect I₀ and I₁ appropriately:

AND gate:  Y = A·B     → MUX(A, 0, B)    [I₀=0, I₁=B, S=A]
OR gate:   Y = A+B     → MUX(A, B, 1)    [I₀=B, I₁=1, S=A]
XOR gate:  Y = A⊕B     → MUX(A, B, B̄)   [I₀=B, I₁=B̄, S=A]
NOT gate:  Y = Ā       → MUX(A, 1, 0)    [I₀=1, I₁=0, S=A]

Interview Notes

  • 2:1 MUX equation: Y = S̄·I₀ + S·I₁. Fundamental building block for data routing.
  • Can implement any 2-variable function by connecting inputs to 0, 1, or variables.
  • N:1 MUX requires log₂(N) select lines. 2:1 has 1 select, 4:1 has 2, 8:1 has 3.
  • CMOS transmission gate implementation is most efficient: 6 transistors total.
  • Glitch-free if inputs are stable before select changes (important for clock muxing).
  • Can build a 4:1 MUX from three 2:1 MUXes in a tree structure.
