Overview

A Demultiplexer (DEMUX) performs the inverse function of a multiplexer: it takes a single data input D and routes it to one of 2ⁿ output lines based on n select lines. The selected output receives the input data while all other outputs are held at logic 0.

Demultiplexers are essential for data distribution systems, memory address decoding (selecting one chip among many), serial-to-parallel conversion, and signal demultiplexing in communication systems. A DEMUX is functionally equivalent to a decoder with an enable input.

DEMUX vs Decoder

A demultiplexer is functionally identical to a decoder with an enable input. The enable acts as the data input, and the decoder outputs become the demux outputs.

Aspect Decoder Demultiplexer
Primary Input n-bit address only Data D + n-bit select
Output When Selected Always 1 Follows input D
Enable Input Optional (E) Is the data (D)
Use Case Address decoding Data distribution
Conversion Decoder + Enable = DEMUX
Using 74LS138 (3:8 Decoder) as 1:8 DEMUX

Connect G1=1, G2A=0, G2B=data. The data input acts as active-low enable. When data=0, selected output goes low (active). Invert outputs for positive logic DEMUX.

RTL Implementation

VERILOG demux.v 
// 1:4 DEMUX - Behavioral
module demux_1to4 (
    input        d,
    input  [1:0] sel,
    output [3:0] y
);

assign y[0] = (sel == 2'b00) ? d : 1'b0;
assign y[1] = (sel == 2'b01) ? d : 1'b0;
assign y[2] = (sel == 2'b10) ? d : 1'b0;
assign y[3] = (sel == 2'b11) ? d : 1'b0;

endmodule

// 1:4 DEMUX - Using shift
module demux_1to4_shift (
    input        d,
    input  [1:0] sel,
    output [3:0] y
);

// Shift data to selected position
assign y = d << sel;

endmodule

// Parameterized 1:N DEMUX
module demux #(
    parameter SEL_WIDTH = 3,
    parameter N = 1 << SEL_WIDTH
)(
    input  [SEL_WIDTH-1:0] sel,
    input                   d,
    output [N-1:0]          y
);

assign y = d << sel;

endmodule

// 1:8 DEMUX with multi-bit data
module demux_1to8_wide #(
    parameter WIDTH = 8
)(
    input  [WIDTH-1:0] d,
    input  [2:0]        sel,
    output reg [WIDTH-1:0] y [0:7]
);

integer i;
always @(*) begin
    for (i = 0; i < 8; i = i + 1)
        y[i] = (i == sel) ? d : 0;
end

endmodule
VHDL demux.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

entity demux_1to4 is
    port (
        d   : in  std_logic;
        sel : in  std_logic_vector(1 downto 0);
        y   : out std_logic_vector(3 downto 0)
    );
end demux_1to4;

architecture behavioral of demux_1to4 is
begin
    process(d, sel)
    begin
        y <= (others => '0');
        y(to_integer(unsigned(sel))) <= d;
    end process;
end behavioral;

-- Using shift operation
architecture shift_based of demux_1to4 is
    signal temp : std_logic_vector(3 downto 0);
begin
    temp <= (0 => d, others => '0');
    y <= std_logic_vector(
        shift_left(unsigned(temp), 
        to_integer(unsigned(sel)))
    );
end shift_based;

Applications

Memory Address Decoding

Select one memory chip based on upper address bits. DEMUX activates chip select.

4 memory banks → 1:4 DEMUX
Serial to Parallel

Convert serial data stream to parallel outputs using counter-controlled DEMUX.

UART RX → shift register → DEMUX
Data Distribution

Send single data stream to one of multiple destinations.

CPU → multiple peripherals
Seven Segment Display

Time-multiplex digits using DEMUX to select which display receives data.

4-digit display → 1:4 DEMUX

MUX + DEMUX Data Transmission

A MUX and DEMUX pair can transmit multiple signals over a single wire:

Transmitter N:1 MUX selects input
Channel Single wire + sync signal
Receiver 1:N DEMUX routes to output
Requirement Synchronized select signals

Interview Notes

  • DEMUX is the inverse of MUX: routes 1 input to 1 of N outputs. Others are 0 (or high-Z).
  • Yᵢ = mᵢ(S) · D — selected output gets data, others are 0. Uses AND gates.
  • DEMUX ≡ Decoder with enable. Use decoder IC with enable as data input.
  • 1:2ⁿ DEMUX needs n select lines. 1:4 = 2 select, 1:8 = 3 select.
  • Common use: memory address decoding — upper address bits select memory bank.
  • 74LS154 is 4:16 decoder/demux; 74LS138 is 3:8 decoder/demux.
