Overview

A 4:1 Multiplexer selects one of four data inputs (I₀ through I₃) and routes it to a single output Y. The selection is controlled by two select lines S₁ and S₀, which together form a 2-bit binary address specifying which input to pass.

The 4:1 MUX is ubiquitous in digital design: it appears in ALU operand selection, register file read ports, bus arbitration, and as a programmable function generator. With four inputs, it can implement any Boolean function of up to three variables.

Truth Table

S₁ S₀ Y Selected Input
0 0 I₀ Input 0
0 1 I₁ Input 1
1 0 I₂ Input 2
1 1 I₃ Input 3
Select Encoding

The select lines form a binary index: S₁S₀ = 00 selects I₀, 01 selects I₁, 10 selects I₂, 11 selects I₃.

Y = I[2×S₁ + S₀]

Boolean Expression

4:1 MUX Equation
Y = S̄₁·S̄₀·I₀ + S̄₁·S₀·I₁ + S₁·S̄₀·I₂ + S₁·S₀·I₃
General Form
Y = Σ(mᵢ · Iᵢ) for i = 0 to 3
where mᵢ = minterm of select lines
Indexed Notation
Y = I[S]
where S is 2-bit select value

Implementation Methods

Direct Gate Implementation

Four 3-input AND gates (one per input) feeding a 4-input OR gate.

4 × AND-3 + 1 × OR-4 + 2 × NOT
Using 2:1 MUXes (Tree)

Three 2:1 MUXes in a two-level tree structure.

Level 1: 2 MUXes (S₀)
Level 2: 1 MUX (S₁)
Decoder + AND-OR

2:4 decoder enables one AND gate, outputs OR together.

1 × 2:4 decoder + 4 × AND-2 + OR-4
CMOS Transmission Gates

Four transmission gates with decoded enables.

Most efficient in silicon

Building 4:1 from 2:1 MUXes

                    ┌───────┐
        I₀ ────────┤       │
                   │ MUX-A ├──────┐
        I₁ ────────┤  S₀   │      │     ┌───────┐
                    └───────┘      │────┤       │
                                        │ MUX-C ├──── Y
                    ┌───────┐      │────┤  S₁   │
        I₂ ────────┤       │      │     └───────┘
                   │ MUX-B ├──────┘
        I₃ ────────┤  S₀   │
                    └───────┘

Level 1: MUX-A selects I₀/I₁, MUX-B selects I₂/I₃ (using S₀)
Level 2: MUX-C selects between MUX-A and MUX-B outputs (using S₁)

RTL Implementation

VERILOG mux4.v 
// 4:1 MUX - Behavioral (case statement)
module mux4_case #(
    parameter WIDTH = 8
)(
    input  [WIDTH-1:0] i0, i1, i2, i3,
    input  [1:0]         sel,
    output reg [WIDTH-1:0] y
);

always @(*) begin
    case (sel)
        2'b00: y = i0;
        2'b01: y = i1;
        2'b10: y = i2;
        2'b11: y = i3;
    endcase
end

endmodule

// 4:1 MUX - Using array indexing
module mux4_index #(
    parameter WIDTH = 8
)(
    input  [WIDTH-1:0] i [0:3],  // Input array
    input  [1:0]         sel,
    output [WIDTH-1:0] y
);

assign y = i[sel];

endmodule

// 4:1 MUX - Built from 2:1 MUXes
module mux4_tree #(
    parameter WIDTH = 8
)(
    input  [WIDTH-1:0] i0, i1, i2, i3,
    input  [1:0]         sel,
    output [WIDTH-1:0] y
);

wire [WIDTH-1:0] mux_lo, mux_hi;

// Level 1: Select within pairs
assign mux_lo = sel[0] ? i1 : i0;
assign mux_hi = sel[0] ? i3 : i2;

// Level 2: Select between pairs
assign y = sel[1] ? mux_hi : mux_lo;

endmodule
VHDL mux4.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;

entity mux4 is
    generic (WIDTH : integer := 8);
    port (
        i0, i1, i2, i3 : in  std_logic_vector(WIDTH-1 downto 0);
        sel              : in  std_logic_vector(1 downto 0);
        y                : out std_logic_vector(WIDTH-1 downto 0)
    );
end mux4;

architecture behavioral of mux4 is
begin
    with sel select
        y <= i0 when "00",
             i1 when "01",
             i2 when "10",
             i3 when others;
end behavioral;

-- Using process with case
architecture proc_case of mux4 is
begin
    process(i0, i1, i2, i3, sel)
    begin
        case sel is
            when "00" => y <= i0;
            when "01" => y <= i1;
            when "10" => y <= i2;
            when others => y <= i3;
        end case;
    end process;
end proc_case;

Function Generator

A 4:1 MUX can implement any Boolean function of 3 variables by using two variables as select lines and connecting the inputs to appropriate values of the third variable.

Implementing F(A,B,C) = Σm(1,2,4,7) using 4:1 MUX

Use A, B as select lines (S₁=A, S₀=B):

  A B | C=0  C=1 | F = ?
  ----+----------+-------
  0 0 |  0    1  | F = C     → I₀ = C
  0 1 |  1    0  | F = C̄     → I₁ = C̄
  1 0 |  1    0  | F = C̄     → I₂ = C̄
  1 1 |  0    1  | F = C     → I₃ = C

Result: Connect I₀=C, I₁=C̄, I₂=C̄, I₃=C
        Select S₁=A, S₀=B
        Output Y = F(A,B,C)
Input Patterns

For each select combination, determine if F depends on C:

00: F=0 → I₀=0
00: F=1 → I₀=1
00: F=C → I₀=C
00: F=C̄ → I₀=C̄

Interview Notes

  • 4:1 MUX: Y = S̄₁S̄₀I₀ + S̄₁S₀I₁ + S₁S̄₀I₂ + S₁S₀I₃. Uses 2 select lines for 4 inputs.
  • Can implement any 3-variable function by using 2 vars as select, 3rd var on inputs.
  • Build from 2:1 MUXes: 3 total — 2 at first level (S₀), 1 at second level (S₁).
  • 74LS153 is dual 4:1 MUX IC; 74LS157 is quad 2:1 MUX IC.
  • In FPGA, LUTs are n-input MUXes where n is typically 4-6 inputs.
  • For timing-critical paths, tree structure has lower delay than flat AND-OR.
