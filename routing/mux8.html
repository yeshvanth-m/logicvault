Overview

An 8:1 Multiplexer selects one of eight data inputs (I₀ through I₇) and routes it to a single output Y. Three select lines (S₂, S₁, S₀) form a 3-bit binary address that determines which of the eight inputs is connected to the output.

The 8:1 MUX is powerful enough to implement any Boolean function of up to 4 variables, making it a universal function generator. It's commonly used in register files (8-register read port), instruction decoders, and memory address multiplexing.

Binary Addressing

Select lines form a 3-bit address: S₂S₁S₀ = 000 through 111 selects I₀ through I₇.

Y = I[4×S₂ + 2×S₁ + S₀]

Boolean Expression

8:1 MUX Equation
Y = S̄₂S̄₁S̄₀·I₀ + S̄₂S̄₁S₀·I₁ + S̄₂S₁S̄₀·I₂ + S̄₂S₁S₀·I₃
    + S₂S̄₁S̄₀·I₄ + S₂S̄₁S₀·I₅ + S₂S₁S̄₀·I₆ + S₂S₁S₀·I₇
Minterm Form
Y = Σ(mᵢ · Iᵢ) for i = 0 to 7
Compact Notation
Y = I[S] where S ∈ {0..7}

Implementation Methods

Direct Gate Implementation

Eight 4-input AND gates feeding an 8-input OR gate.

8 × AND-4 + OR-8 + 3 × NOT
Delay: 3 gate levels
Using 2:1 MUXes

Seven 2:1 MUXes in a 3-level tree.

Level 1: 4 MUX (S₀)
Level 2: 2 MUX (S₁)
Level 3: 1 MUX (S₂)
Using 4:1 MUXes

Two 4:1 MUXes feeding one 2:1 MUX.

2 × 4:1 (S₁,S₀) + 1 × 2:1 (S₂)
3:8 Decoder + AND

Decoder enables one AND gate per input.

1 decoder + 8 AND-2 + OR-8

Building 8:1 from 2:1 MUXes (Tree Structure)

Level 1 (4 MUXes using S₀):
  MUX-A: I₀, I₁ → A    MUX-B: I₂, I₃ → B
  MUX-C: I₄, I₅ → C    MUX-D: I₆, I₇ → D

Level 2 (2 MUXes using S₁):
  MUX-E: A, B → E      MUX-F: C, D → F

Level 3 (1 MUX using S₂):
  MUX-G: E, F → Y

Total: 7 × 2:1 MUX
Delay: 3 MUX delays (one per level)

RTL Implementation

VERILOG mux8.v 
// 8:1 MUX - Case statement
module mux8 #(
    parameter WIDTH = 8
)(
    input  [WIDTH-1:0] i0, i1, i2, i3,
    input  [WIDTH-1:0] i4, i5, i6, i7,
    input  [2:0]         sel,
    output reg [WIDTH-1:0] y
);

always @(*) begin
    case (sel)
        3'd0: y = i0;
        3'd1: y = i1;
        3'd2: y = i2;
        3'd3: y = i3;
        3'd4: y = i4;
        3'd5: y = i5;
        3'd6: y = i6;
        3'd7: y = i7;
    endcase
end

endmodule

// 8:1 MUX - Tree of 4:1 and 2:1
module mux8_tree #(
    parameter WIDTH = 8
)(
    input  [WIDTH-1:0] i [0:7],
    input  [2:0]         sel,
    output [WIDTH-1:0] y
);

wire [WIDTH-1:0] lo, hi;

// Two 4:1 MUXes select from each half
assign lo = (sel[1:0] == 2'd0) ? i[0] :
            (sel[1:0] == 2'd1) ? i[1] :
            (sel[1:0] == 2'd2) ? i[2] : i[3];

assign hi = (sel[1:0] == 2'd0) ? i[4] :
            (sel[1:0] == 2'd1) ? i[5] :
            (sel[1:0] == 2'd2) ? i[6] : i[7];

// Final 2:1 MUX selects between halves
assign y = sel[2] ? hi : lo;

endmodule
VHDL mux8.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

entity mux8 is
    generic (WIDTH : integer := 8);
    port (
        i0, i1, i2, i3 : in  std_logic_vector(WIDTH-1 downto 0);
        i4, i5, i6, i7 : in  std_logic_vector(WIDTH-1 downto 0);
        sel              : in  std_logic_vector(2 downto 0);
        y                : out std_logic_vector(WIDTH-1 downto 0)
    );
end mux8;

architecture behavioral of mux8 is
    type input_array is array(0 to 7) of 
         std_logic_vector(WIDTH-1 downto 0);
    signal inputs : input_array;
begin
    inputs <= (i0, i1, i2, i3, i4, i5, i6, i7);
    y <= inputs(to_integer(unsigned(sel)));
end behavioral;

Implementing 4-Variable Function with 8:1 MUX

F(A,B,C,D) using 8:1 MUX with S₂=A, S₁=B, S₀=C:

For each minterm of A,B,C (8 combinations):
  - If F=0 for both D=0 and D=1 → Iᵢ = 0
  - If F=1 for both D=0 and D=1 → Iᵢ = 1
  - If F=D (1 only when D=1)    → Iᵢ = D
  - If F=D̄ (1 only when D=0)    → Iᵢ = D̄

Example: F = Σm(0,2,5,7,10,11,14,15)

1 0 1 |  1   1  | I₅ = 1

Interview Notes

  • 8:1 MUX uses 3 select lines to choose among 8 inputs. Formula: 2ⁿ inputs need n select lines.
  • Can implement any 4-variable function using 3 vars as select, 4th var on inputs (0, 1, D, D̄).
  • Building from 2:1 MUXes requires 2ⁿ-1 = 7 MUXes in a 3-level tree.
  • 74LS151 is single 8:1 MUX; 74LS150 is 16:1 MUX.
  • For N:1 MUX, tree delay is log₂(N) MUX delays — scales logarithmically.
  • Commonly used in register file read ports: 8 registers → 8:1 MUX per bit.
