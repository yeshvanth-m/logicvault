Overview

The Booth Multiplier uses Booth's algorithm (1951) to efficiently multiply signed two's complement numbers. The key insight is that consecutive 1s in the multiplier can be replaced with a subtraction at the start and addition at the end, reducing operations.

Modified Booth Encoding (Radix-4 Booth) examines 3 bits at a time to generate one of five values: −2, −1, 0, +1, +2. This halves the number of partial products compared to a basic array multiplier, significantly reducing area and delay.

Booth Encoding (Radix-2)

Original Booth's algorithm examines pairs of adjacent bits (bi, bi-1):

bi bi-1 Action Meaning
0 0 0 (skip) Middle of zeros
0 1 +A End of ones
1 0 −A Start of ones
1 1 0 (skip) Middle of ones
Example: Multiplying by 0110 (6)

Instead of adding shifted A three times, Booth sees 0110 as: subtract at position 1, add at position 3.

0110 = 1000 − 0010 = 2³ − 2¹ = 8 − 2 = 6

Radix-4 Modified Booth Encoding

Modified Booth examines 3 bits at a time, overlapping by 1 bit. For an N-bit multiplier, this generates only ⌈N/2⌉ partial products.

b2i+1 b2i b2i-1 Encoded Value Operation
00000
001+1+A
010+1+A
011+2+2A
100−2−2A
101−1−A
110−1−A
11100
Generating +2A and −2A

Simply shift A left by 1 bit. For −2A, invert and add 1 (two's complement).

Bit Grouping

For 8-bit B: groups are (b₁,b₀,b₋₁), (b₃,b₂,b₁), (b₅,b₄,b₃), (b₇,b₆,b₅) where b₋₁=0.

Algorithm Steps

Example: 5 × (−3) using 4-bit Radix-4 Booth

A = 0101 (5)      B = 1101 (−3 in 2's complement)

Step 1: Extend B with b₋₁ = 0
B = 1101, b₋₁ = 0 → examine: (b₁,b₀,b₋₁) = (0,1,0) = +1
                              (b₃,b₂,b₁) = (1,1,0) = −1

Step 2: Generate partial products
PP₀ = +A × 2⁰ = 0101 (shifted 0 positions)
PP₁ = −A × 2² = 1011 × 4 = 101100 (shifted 2 positions, negated)

Step 3: Add partial products with sign extension
  0000 0101     (+5)
+ 1111 0100     (−12, which is −3 × 4)
-----------
  1111 1001     (−7... wait, should be −15)

Correct: 0101 × 1101 = 5 × (−3) = −15 = 1111 0001
Key Operations in Hardware
  • 0: Output zero (gated off)
  • ±A: Pass or complement multiplicand
  • ±2A: Shift multiplicand left by 1, then pass/complement
  • Sign extension: Extend each partial product to full width

RTL Implementation

VERILOG booth_mult.v 
module booth_mult #(
    parameter N = 8
)(
    input  signed [N-1:0] a,  // Multiplicand
    input  signed [N-1:0] b,  // Multiplier
    output signed [2*N-1:0] product
);

// Extended multiplier with implicit -1 bit
wire [N:0] b_ext = {b, 1'b0};

// Partial products (N/2 rows)
localparam NUM_PP = (N + 1) / 2;
wire signed [2*N-1:0] pp [NUM_PP-1:0];

// Booth encoder and PP generator
genvar i;
generate
    for (i = 0; i < NUM_PP; i = i + 1) begin : booth_enc
        wire [2:0] code = b_ext[2*i+2 : 2*i];
        reg signed [N:0] pp_val;
        
        always @(*) begin
            case (code)
                3'b000: pp_val = 0;
                3'b001: pp_val = {a[N-1], a};      // +A
                3'b010: pp_val = {a[N-1], a};      // +A
                3'b011: pp_val = {a, 1'b0};        // +2A
                3'b100: pp_val = -{a, 1'b0};       // -2A
                3'b101: pp_val = -{a[N-1], a};    // -A
                3'b110: pp_val = -{a[N-1], a};    // -A
                3'b111: pp_val = 0;
            endcase
        end
        
        // Sign-extend and shift by 2i positions
        assign pp[i] = (pp_val <<< (2 * i));
    end
endgenerate

// Sum all partial products
integer j;
reg signed [2*N-1:0] sum;
always @(*) begin
    sum = 0;
    for (j = 0; j < NUM_PP; j = j + 1)
        sum = sum + pp[j];
end

assign product = sum;

endmodule
VHDL booth_mult.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

entity booth_mult is
    generic (N : integer := 8);
    port (
        a       : in  signed(N-1 downto 0);
        b       : in  signed(N-1 downto 0);
        product : out signed(2*N-1 downto 0)
    );
end booth_mult;

architecture behavioral of booth_mult is
begin
    -- Direct signed multiplication
    product <= a * b;
end behavioral;

Interview Notes

  • Booth's algorithm exploits patterns in multiplier: 0111 = 1000 − 0001 (one sub + one add vs three adds).
  • Radix-4 Modified Booth halves partial products by examining 3 bits at a time with 1-bit overlap.
  • Encoding values {−2, −1, 0, +1, +2} — 2A is just a left shift, negation uses two's complement.
  • Works natively with two's complement signed numbers — no separate sign handling needed.
  • Combined with Wallace/Dadda trees for O(log N) reduction of the N/2 partial products.
  • Standard in modern processors — Intel, ARM, and DSP cores all use Booth encoding.
