Structure

#

An N-bit ripple carry adder consists of N full adders connected in series:

For each bit position i (0 to N-1)

Sum[i] = A[i] ⊕ B[i] ⊕ C[i]

C[i+1] = (A[i] · B[i]) + (C[i] · (A[i] ⊕ B[i]))

Carry chain connection

C[0] = Cin  →  C[1]  →  C[2]  →  ...  →  C[N] = Cout

Key concept: The carry must propagate through ALL stages sequentially. The final sum bit S[N-1] cannot be computed until C[N-1] arrives from the previous stage.

Timing Analysis

#

Critical Path Delay

First full adder (Cin to C1) 2 gate delays
Each subsequent carry propagation 2 gate delays
Final sum computation 1 gate delay (XOR)
Total for N-bit RCA 2N + 1 gate delays

Worst-case scenario: Adding numbers like 11111111 + 00000001. The carry must propagate through every single bit position.

Bit WidthGate DelaysApprox. Delay @ 1ns/gate
4-bit99 ns
8-bit1717 ns
16-bit3333 ns
32-bit6565 ns
64-bit129129 ns

Performance issue: A 64-bit RCA at 1GHz would need 129ns, but the clock period is only 1ns! This is why faster adder architectures (CLA, Kogge-Stone) are essential for high-performance processors.

RTL Implementation

#
Verilog ripple_carry_adder.v 
// Parameterized Ripple Carry Adder
module ripple_carry_adder #(
    parameter N = 8
)(
    input  [N-1:0] a,
    input  [N-1:0] b,
    input          cin,
    output [N-1:0] sum,
    output         cout
);

    wire [N:0] carry;
    
    assign carry[0] = cin;
    assign cout = carry[N];

    genvar i;
    generate
        for (i = 0; i < N; i = i + 1) begin : fa_gen
            full_adder fa (
                .a    (a[i]),
                .b    (b[i]),
                .cin  (carry[i]),
                .sum  (sum[i]),
                .cout (carry[i+1])
            );
        end
    endgenerate

endmodule
VHDL ripple_carry_adder.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity ripple_carry_adder is
    generic (N : integer := 8);
    port (
        a    : in  std_logic_vector(N-1 downto 0);
        b    : in  std_logic_vector(N-1 downto 0);
        cin  : in  std_logic;
        sum  : out std_logic_vector(N-1 downto 0);
        cout : out std_logic
    );
end ripple_carry_adder;

architecture structural of ripple_carry_adder is
    signal carry : std_logic_vector(N downto 0);
begin
    carry(0) <= cin;
    cout <= carry(N);
    
    gen_fa: for i in 0 to N-1 generate
        fa: entity work.full_adder
            port map (a(i), b(i), carry(i), sum(i), carry(i+1));
    end generate;
end structural;
Verilog rca_4bit_structural.v 
// 4-bit RCA - Explicit structural instantiation
module rca_4bit (
    input  [3:0] a, b,
    input        cin,
    output [3:0] sum,
    output       cout
);
    wire c1, c2, c3;

    full_adder fa0 (.a(a[0]), .b(b[0]), .cin(cin), .sum(sum[0]), .cout(c1));
    full_adder fa1 (.a(a[1]), .b(b[1]), .cin(c1),  .sum(sum[1]), .cout(c2));
    full_adder fa2 (.a(a[2]), .b(b[2]), .cin(c2),  .sum(sum[2]), .cout(c3));
    full_adder fa3 (.a(a[3]), .b(b[3]), .cin(c3),  .sum(sum[3]), .cout(cout));

endmodule

Interview Notes

#
  • Why is it called "ripple" carry? — The carry signal ripples through each stage sequentially, like a wave propagating through the adder chain.
  • What's the critical path? — From Cin through all N carry stages to the final sum bit. Delay = 2N + 1 gate delays.
  • When would you use RCA over CLA? — When area/power is critical and speed isn't (e.g., IoT devices, battery-powered systems, or when N ≤ 8).
  • Can you pipeline an RCA? — Yes, but it defeats the purpose. Better to use faster architectures for high-throughput needs.
  • What's the worst-case input? — All 1s + 1 (e.g., 11111111 + 00000001), forcing carry to propagate through every bit.
