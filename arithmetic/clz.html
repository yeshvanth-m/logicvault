Overview

CLZ (Count Leading Zeros), also known as LZCNT (Leading Zero Count) or NLZO (Number of Leading Zero bits), counts consecutive zero bits starting from the most significant bit until the first '1' is encountered.

CLZ is critical for floating-point operations where normalization requires shifting the mantissa until the MSB is 1. It also enables efficient integer log₂ calculation: floor(log₂(x)) = width - 1 - CLZ(x) for x > 0.

The naive approach of checking each bit sequentially takes O(n) time. Using a binary tree structure, CLZ can be computed in O(log n) time, making it suitable for high-performance implementations.

Architecture

📐 Binary Tree CLZ (8-bit)

Level 0 (pairs): Check if upper half of each pair is zero
    D7 D6  D5 D4  D3 D2  D1 D0
      ↓      ↓      ↓      ↓
    [2b]   [2b]   [2b]   [2b]   ← 2-bit CLZ blocks

Level 1 (quads): Merge pairs
    [2b][2b]     [2b][2b]
        ↓           ↓
      [3b]       [3b]           ← 4-bit CLZ blocks

Level 2 (byte): Merge quads  
      [3b][3b]
          ↓
        [4b]                     ← 8-bit CLZ result

Key insight: At each level, if upper half all zeros,
             add half-width to count and use lower half's CLZ
2-bit CLZ Building Block
Input: D[1:0]
clz[1] = ~D[1] & ~D[0]   // both zero → clz=2
clz[0] = ~D[1]   // MSB zero → at least 1

D=00 → clz=10 (2), D=01 → clz=01 (1)
D=10 → clz=00 (0), D=11 → clz=00 (0)

Worked Example

8-bit CLZ: Input = 00101100

Input:    0 0 1 0 1 1 0 0  (decimal 44)

Level 0 - Pair CLZ:
  Pair 3 (00): clz = 2    Pair 2 (10): clz = 0
  Pair 1 (11): clz = 0    Pair 0 (00): clz = 2

Level 1 - Merge to 4-bit:
  Upper half [00 10]:
    Pair 3 all zero? NO (Pair 3 has clz=2, not all zero)
    Wait - Pair 3 IS all zero. Use Pair 2's clz + 2
    Upper 4-bit clz = 2 + clz(10) = 2 + 0 = 2
    
  Lower half [11 00]:
    Pair 1 all zero? NO
    Lower 4-bit clz = 0 + clz(11) = 0 + 0 = 0

Level 2 - Merge to 8-bit:
  Upper 4 bits (0010) has clz = 2 (not all zero)
  Final clz = 2 + 0 = 2  ✓

Verification: 00101100 has 2 leading zeros ✓

8-bit CLZ: Input = 00000011

Input:    0 0 0 0 0 0 1 1  (decimal 3)

Level 0 - Pair CLZ:
  Pair 3 (00): clz = 2    Pair 2 (00): clz = 2
  Pair 1 (00): clz = 2    Pair 0 (11): clz = 0

Level 1 - Merge to 4-bit:
  Upper half [00 00]: both pairs all zero
    Upper 4-bit clz = 4 (all zeros)
    
  Lower half [00 11]:
    Pair 1 all zero? YES → use Pair 0's clz + 2
    Lower 4-bit clz = 2 + 0 = 2

Level 2 - Merge to 8-bit:
  Upper 4 bits all zero? YES → use lower's clz + 4
  Final clz = 4 + 2 = 6  ✓

Verification: 00000011 has 6 leading zeros ✓

RTL Implementation

VERILOG clz.v 
// 32-bit CLZ - Behavioral (synthesizable)
module clz_32 (
    input      [31:0] data_in,
    output reg [5:0]  count     // 0-32
);

integer i;
always @(*) begin
    count = 6'd32;  // Default: all zeros
    for (i = 31; i >= 0; i = i - 1) begin
        if (data_in[i] && count == 6'd32)
            count = 31 - i;
    end
end

endmodule

// 32-bit CLZ - Binary Tree Structure
module clz_32_tree (
    input      [31:0] data_in,
    output     [5:0]  count
);

// Level 1: 16-bit halves
wire [4:0] clz_hi, clz_lo;
wire hi_zero;

clz_16 u_hi (.data_in(data_in[31:16]), .count(clz_hi));
clz_16 u_lo (.data_in(data_in[15:0]),  .count(clz_lo));

assign hi_zero = (clz_hi == 5'd16);
assign count = hi_zero ? (6'd16 + clz_lo) : {1'b0, clz_hi};

endmodule

// 16-bit CLZ
module clz_16 (
    input      [15:0] data_in,
    output     [4:0]  count
);

wire [3:0] clz_hi, clz_lo;
wire hi_zero;

clz_8 u_hi (.data_in(data_in[15:8]), .count(clz_hi));
clz_8 u_lo (.data_in(data_in[7:0]),  .count(clz_lo));

assign hi_zero = (clz_hi == 4'd8);
assign count = hi_zero ? (5'd8 + clz_lo) : {1'b0, clz_hi};

endmodule

// 8-bit CLZ
module clz_8 (
    input      [7:0] data_in,
    output     [3:0] count
);

wire [2:0] clz_hi, clz_lo;
wire hi_zero;

clz_4 u_hi (.data_in(data_in[7:4]), .count(clz_hi));
clz_4 u_lo (.data_in(data_in[3:0]), .count(clz_lo));

assign hi_zero = (clz_hi == 3'd4);
assign count = hi_zero ? (4'd4 + clz_lo) : {1'b0, clz_hi};

endmodule

// 4-bit CLZ (base case)
module clz_4 (
    input      [3:0] data_in,
    output reg [2:0] count
);

always @(*) begin
    casez (data_in)
        4'b1???: count = 3'd0;
        4'b01??: count = 3'd1;
        4'b001?: count = 3'd2;
        4'b0001: count = 3'd3;
        4'b0000: count = 3'd4;
    endcase
end

endmodule
VHDL clz.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

entity clz_32 is
    port (
        data_in : in  std_logic_vector(31 downto 0);
        count   : out std_logic_vector(5 downto 0)
    );
end clz_32;

architecture behavioral of clz_32 is
begin
    process(data_in)
        variable cnt : integer range 0 to 32;
    begin
        cnt := 32;
        for i in 31 downto 0 loop
            if data_in(i) = '1' then
                cnt := 31 - i;
                exit;
            end if;
        end loop;
        count <= std_logic_vector(to_unsigned(cnt, 6));
    end process;
end behavioral;

-- Alternative: Using priority encoder
architecture tree of clz_32 is
    signal upper_zero : std_logic;
    signal clz_hi, clz_lo : std_logic_vector(4 downto 0);
begin
    -- Instance CLZ for upper and lower 16 bits
    u_hi: entity work.clz_16 port map (data_in(31 downto 16), clz_hi);
    u_lo: entity work.clz_16 port map (data_in(15 downto 0), clz_lo);
    
    upper_zero <= '1' when clz_hi = "10000" else '0';
    
    count <= std_logic_vector(unsigned("010000") + unsigned('0' & clz_lo))
             when upper_zero = '1'
             else '0' & clz_hi;
end tree;

Variations

CLO (Count Leading Ones)

Invert input, then apply CLZ. Used for sign extension checks.

CLO(x) = CLZ(~x)
CTZ (Count Trailing Zeros)

Reverse bits, then CLZ. Or use x & (-x) to isolate LSB.

CTZ(x) = CLZ(reverse(x))
Find First Set (FFS)

Position of first 1 from LSB. FFS = WIDTH - CLZ of reversed input.

FFS = CTZ + 1
Priority Encoder

Related circuit: outputs index of highest priority input.

Similar tree structure

Applications

FP Normalization

After FP add/sub, normalize mantissa by shifting. CLZ finds shift amount.

Shift = CLZ(mantissa)
Integer Division

Estimate quotient bits using CLZ difference: CLZ(dividend) - CLZ(divisor).

Fast division prep
Log₂ Calculation

Integer log₂ needed for memory allocators, hash tables.

log₂ = 31 - CLZ
Graphics/Compression

Entropy coding (Huffman), texture compression use CLZ.

Variable-length coding

Interview Notes

  • CLZ counts leading zeros from MSB. CLZ(0) = width (all zeros), CLZ with MSB=1 gives 0.
  • Relationship: floor(log₂(x)) = width - 1 - CLZ(x) for x > 0.
  • Tree structure gives O(log n) depth. 32-bit CLZ has 5 levels.
  • Key technique: check if upper half all-zero, if so add half-width to lower's CLZ.
  • CTZ (trailing zeros): reverse bits then CLZ, or use isolate-LSB trick: x & (-x).
  • Critical for FP normalization: determines how much to left-shift mantissa.
  • RISC-V: CLZ is part of Zbb (bit manipulation) extension.
