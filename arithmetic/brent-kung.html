Structure

#

The Brent-Kung adder uses a two-phase prefix computation that dramatically reduces cell count and wiring compared to Kogge-Stone, at the cost of increased depth.

Phase 1: Forward Sweep

Builds a sparse prefix tree, computing prefix sums only at positions 2k-1. This phase has log₂N levels and uses (N/2) cells.

Positions: 1, 3, 7, 15, ... (2k-1)
Phase 2: Backward Sweep

Fills in the missing prefix sums by combining results from Phase 1. Uses the same prefix operator in reverse order.

Fills: 2, 4, 5, 6, 8, 9, 10, ...
Phase Levels Purpose Cell Count
Level 01Initial P, G computationN
Forwardlog₂N - 1Build sparse prefix treeN - 1 - log₂N/2
Backwardlog₂N - 2Fill missing prefixesN/2 - 1
Sum1Final sum computationN

Prefix Operator (same as Kogge-Stone)

Gi:k = Gi:j + (Pi:j · Gj-1:k)

Pi:k = Pi:j · Pj-1:k

Key insight: The backward sweep reuses prefix sums computed in the forward sweep. Position i uses the prefix from position (i - LSB(i)) where LSB is the least significant set bit.

Hybrid designs: Many practical implementations use a hybrid approach — Kogge-Stone at the LSB end (where speed matters most) and Brent-Kung at the MSB end (for area efficiency).

RTL Implementation

#
Verilog brent_kung_8bit.v 
// 8-bit Brent-Kung Parallel Prefix Adder
module brent_kung_8bit (
    input  [7:0] a, b,
    input        cin,
    output [7:0] sum,
    output       cout
);

    wire [7:0] p0, g0;         // Level 0: initial P, G
    wire [7:0] g1, g2, g3;     // Forward sweep generates
    wire [7:0] p1, p2, p3;     // Forward sweep propagates
    wire [7:0] gf, pf;         // Final (after backward sweep)
    wire [8:0] c;              // Carries

    // Level 0: Generate initial P and G
    assign p0 = a ^ b;
    assign g0 = a & b;

    // ═══ FORWARD SWEEP ═══
    // Level 1: Odd positions (1, 3, 5, 7)
    assign {g1[0], p1[0]} = {g0[0], p0[0]};
    assign g1[1] = g0[1] | (p0[1] & g0[0]);
    assign p1[1] = p0[1] & p0[0];
    assign {g1[2], p1[2]} = {g0[2], p0[2]};
    assign g1[3] = g0[3] | (p0[3] & g0[2]);
    assign p1[3] = p0[3] & p0[2];
    assign {g1[4], p1[4]} = {g0[4], p0[4]};
    assign g1[5] = g0[5] | (p0[5] & g0[4]);
    assign p1[5] = p0[5] & p0[4];
    assign {g1[6], p1[6]} = {g0[6], p0[6]};
    assign g1[7] = g0[7] | (p0[7] & g0[6]);
    assign p1[7] = p0[7] & p0[6];

    // Level 2: Positions 3, 7
    assign {g2[2:0], p2[2:0]} = {g1[2:0], p1[2:0]};
    assign g2[3] = g1[3] | (p1[3] & g1[1]);
    assign p2[3] = p1[3] & p1[1];
    assign {g2[6:4], p2[6:4]} = {g1[6:4], p1[6:4]};
    assign g2[7] = g1[7] | (p1[7] & g1[5]);
    assign p2[7] = p1[7] & p1[5];

    // Level 3: Position 7 only (combines 0-7)
    assign {g3[6:0], p3[6:0]} = {g2[6:0], p2[6:0]};
    assign g3[7] = g2[7] | (p2[7] & g2[3]);
    assign p3[7] = p2[7] & p2[3];

    // ═══ BACKWARD SWEEP ═══
    // Fill in missing positions using computed prefixes
    assign gf[0] = g3[0];
    assign gf[1] = g3[1];
    assign gf[2] = g0[2] | (p0[2] & g3[1]);     // Uses prefix[1]
    assign gf[3] = g3[3];
    assign gf[4] = g0[4] | (p0[4] & g3[3]);     // Uses prefix[3]
    assign gf[5] = g1[5] | (p1[5] & g3[3]);     // Uses prefix[3]
    assign gf[6] = g0[6] | (p0[6] & gf[5]);     // Uses prefix[5]
    assign gf[7] = g3[7];

    assign pf = p0;  // Only need initial P for sum

    // Final carry computation
    assign c[0] = cin;
    assign c[1] = gf[0] | (p0[0] & c[0]);
    assign c[2] = gf[1] | (p3[1] & c[0]);
    assign c[3] = gf[2] | (p0[2] & p3[1] & c[0]);
    assign c[4] = gf[3] | (p3[3] & c[0]);
    assign c[5] = gf[4] | (p0[4] & p3[3] & c[0]);
    assign c[6] = gf[5] | (p1[5] & p3[3] & c[0]);
    assign c[7] = gf[6] | (p0[6] & p1[5] & p3[3] & c[0]);
    assign c[8] = gf[7] | (p3[7] & c[0]);

    // Sum computation
    assign sum = p0 ^ c[7:0];
    assign cout = c[8];

endmodule
VHDL brent_kung_8bit.vhd 
-- 8-bit Brent-Kung Parallel Prefix Adder
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity brent_kung_8bit is
    port (
        a, b : in  std_logic_vector(7 downto 0);
        cin   : in  std_logic;
        sum   : out std_logic_vector(7 downto 0);
        cout  : out std_logic
    );
end brent_kung_8bit;

architecture rtl of brent_kung_8bit is
    signal p0, g0 : std_logic_vector(7 downto 0);
    signal g1, g2, g3, gf : std_logic_vector(7 downto 0);
    signal p1, p2, p3 : std_logic_vector(7 downto 0);
    signal c : std_logic_vector(8 downto 0);
begin
    -- Level 0: Initial P and G
    p0 <= a xor b;
    g0 <= a and b;

    -- Forward sweep level 1: odd positions
    g1(0) <= g0(0); p1(0) <= p0(0);
    g1(1) <= g0(1) or (p0(1) and g0(0));
    p1(1) <= p0(1) and p0(0);
    g1(2) <= g0(2); p1(2) <= p0(2);
    g1(3) <= g0(3) or (p0(3) and g0(2));
    p1(3) <= p0(3) and p0(2);
    g1(4) <= g0(4); p1(4) <= p0(4);
    g1(5) <= g0(5) or (p0(5) and g0(4));
    p1(5) <= p0(5) and p0(4);
    g1(6) <= g0(6); p1(6) <= p0(6);
    g1(7) <= g0(7) or (p0(7) and g0(6));
    p1(7) <= p0(7) and p0(6);

    -- Forward sweep level 2
    g2(2 downto 0) <= g1(2 downto 0);
    p2(2 downto 0) <= p1(2 downto 0);
    g2(3) <= g1(3) or (p1(3) and g1(1));
    p2(3) <= p1(3) and p1(1);
    g2(6 downto 4) <= g1(6 downto 4);
    p2(6 downto 4) <= p1(6 downto 4);
    g2(7) <= g1(7) or (p1(7) and g1(5));
    p2(7) <= p1(7) and p1(5);

    -- Forward sweep level 3
    g3(6 downto 0) <= g2(6 downto 0);
    p3(6 downto 0) <= p2(6 downto 0);
    g3(7) <= g2(7) or (p2(7) and g2(3));
    p3(7) <= p2(7) and p2(3);

    -- Backward sweep
    gf(0) <= g3(0);
    gf(1) <= g3(1);
    gf(2) <= g0(2) or (p0(2) and g3(1));
    gf(3) <= g3(3);
    gf(4) <= g0(4) or (p0(4) and g3(3));
    gf(5) <= g1(5) or (p1(5) and g3(3));
    gf(6) <= g0(6) or (p0(6) and gf(5));
    gf(7) <= g3(7);

    -- Final carries and sum
    c(0) <= cin;
    c(1) <= gf(0) or (p0(0) and c(0));
    c(2) <= gf(1) or (p3(1) and c(0));
    c(3) <= gf(2) or (p0(2) and p3(1) and c(0));
    c(4) <= gf(3) or (p3(3) and c(0));
    c(5) <= gf(4) or (p0(4) and p3(3) and c(0));
    c(6) <= gf(5) or (p1(5) and p3(3) and c(0));
    c(7) <= gf(6) or (p0(6) and p1(5) and p3(3) and c(0));
    c(8) <= gf(7) or (p3(7) and c(0));

    sum <= p0 xor c(7 downto 0);
    cout <= c(8);
end rtl;

Interview Notes

#
  • Why use Brent-Kung over Kogge-Stone? — When area and power are more important than absolute speed. BK uses O(N) cells/wires vs O(N log N) for KS.
  • What is the two-phase structure? — Forward sweep builds sparse prefix tree (positions 2^k-1), backward sweep fills in missing prefixes using computed results.
  • What's the depth penalty? — 2·log₂N - 2 levels vs log₂N + 1 for Kogge-Stone. For 32-bit: 8 vs 6 levels.
  • Why is wiring simpler? — All wires span at most 1 position horizontally. No long diagonal wires like Kogge-Stone.
  • When would you use a hybrid? — Use KS for LSBs (carry arrives last, needs speed), BK for MSBs (carry arrives first, save area).
  • How does it compare to CLA? — BK is a parallel prefix adder like KS, more regular than hierarchical CLA, easier to generate for arbitrary widths.
