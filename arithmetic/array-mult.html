Overview

The Array Multiplier implements binary multiplication using the traditional shift-and-add algorithm in hardware. It generates all partial products in parallel using AND gates, then sums them using an array of full adders arranged in a regular, rectangular structure.

For an N×N multiplication, the multiplier forms N partial products. Each partial product is the multiplicand ANDed with a single bit of the multiplier, shifted by the bit position. These partial products are summed row by row using full adders to produce the final 2N-bit result.

Multiplication Algorithm

Partial Products

Each bit of multiplier B generates a partial product:

PPi = A × B[i] × 2i
Final Product

Sum of all shifted partial products:

P = Σ PPi for i=0 to N-1

Example: 4-bit × 4-bit (A=1011 × B=1101)

          1 0 1 1     (A = 11)
        × 1 1 0 1     (B = 13)
        ---------
          1 0 1 1     PP₀ = A × B[0] = A × 1
        0 0 0 0 ·     PP₁ = A × B[1] = A × 0
      1 0 1 1 · ·     PP₂ = A × B[2] = A × 1
    1 0 1 1 · · ·     PP₃ = A × B[3] = A × 1
    -------------
  1 0 0 0 1 1 1 1     P = 143 (8 bits)

Array Structure

The array multiplier arranges full adders in an N×N grid. Each row handles one partial product, with carries propagating down and to the left:

Component 4×4 Array 8×8 Array N×N Array
AND Gates 16 64 N²
Full Adders 12 56 N² - N
Half Adders 0 0 0 (can optimize)
Critical Path 7 FA 15 FA 2N - 1 FA
Output Bits 8 16 2N
Carry Propagation

In the basic array structure, carries ripple diagonally through the array. The critical path goes through the rightmost column and bottom row, passing through approximately 2N-1 full adders.

RTL Implementation

VERILOG array_mult.v 
module array_mult #(
    parameter N = 8
)(
    input  [N-1:0] a,
    input  [N-1:0] b,
    output [2*N-1:0] product
);

// Partial products
wire [N-1:0] pp [N-1:0];

// Generate partial products (AND gates)
genvar i, j;
generate
    for (i = 0; i < N; i = i + 1) begin : pp_row
        for (j = 0; j < N; j = j + 1) begin : pp_col
            assign pp[i][j] = a[j] & b[i];
        end
    end
endgenerate

// Sum and carry arrays
wire [N:0] sum [N-1:0];
wire [N:0] carry [N-1:0];

// First row - just partial products
assign sum[0] = {1'b0, pp[0]};
assign carry[0] = 0;
assign product[0] = sum[0][0];

// Array of full adders
generate
    for (i = 1; i < N; i = i + 1) begin : row
        wire [N:0] pp_shifted = {1'b0, pp[i]};
        wire [N:0] prev_sum = {carry[i-1][N-1], sum[i-1][N:1]};
        
        assign {carry[i], sum[i]} = pp_shifted + prev_sum + carry[i-1][N];
        assign product[i] = sum[i][0];
    end
endgenerate

// Upper bits of product
assign product[2*N-1:N] = sum[N-1][N:1];

endmodule
VHDL array_mult.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

entity array_mult is
    generic (N : integer := 8);
    port (
        a       : in  std_logic_vector(N-1 downto 0);
        b       : in  std_logic_vector(N-1 downto 0);
        product : out std_logic_vector(2*N-1 downto 0)
    );
end array_mult;

architecture behavioral of array_mult is
    type pp_array is array(0 to N-1) of 
         unsigned(N-1 downto 0);
    signal pp : pp_array;
begin

    -- Generate partial products
    gen_pp: for i in 0 to N-1 generate
        gen_bit: for j in 0 to N-1 generate
            pp(i)(j) <= a(j) and b(i);
        end generate;
    end generate;

    -- Behavioral multiplication
    product <= std_logic_vector(
        unsigned(a) * unsigned(b)
    );

end behavioral;

Area & Timing Analysis

Complexity Analysis

Gate Count O(N²)
Critical Path Delay O(N) full adder delays
Power Consumption O(N²) — all gates active
Regularity Highly regular — easy layout
Multiplier Type Area Delay Best For
Array O(N²) O(N) Simple designs, small N
Booth O(N²/2) O(N) Signed multiplication
Wallace Tree O(N²) O(log N) High-speed, large N
Dadda O(N²) smaller O(log N) Area-efficient fast mult

Interview Notes

  • Array multiplier has O(N²) area and O(N) delay — simple but slow for large operands.
  • Critical path goes through 2N-1 full adders (diagonal + bottom row).
  • Regular structure makes it easy to layout but not area-efficient.
  • Can be pipelined by inserting registers between rows for higher throughput.
  • Better than repeated addition but worse than tree-based multipliers for large N.
  • Synthesis tools often replace behavioral multiply (*) with optimized structures.
