Overview

Overflow Detection identifies when an arithmetic operation produces a result that cannot be correctly represented in the available bit width. This is different for signed and unsigned numbers:

  • Unsigned overflow (carry): Result exceeds 2ⁿ-1 (addition) or goes below 0 (subtraction)
  • Signed overflow: Result exceeds the signed range, causing incorrect sign

Signed overflow is detected when the carry into the MSB differs from the carry out of the MSB. This occurs when adding two positive numbers yields a negative result, or adding two negative numbers yields a positive result.

Signed vs Unsigned Overflow

Type 8-bit Range Overflow Condition Detection
Unsigned 0 to 255 Result > 255 or < 0 Carry/Borrow out
Signed -128 to +127 Result > 127 or < -128 Cin[7] ⊕ Cout[7]
Unsigned Overflow (C flag)

Simple: carry out of MSB position for add, borrow for subtract.

C = Cout
Signed Overflow (V flag)

XOR of carry into MSB and carry out of MSB.

V = Cin[n-1] ⊕ Cout[n-1]
Alternative V Formula

For addition: V = (A[n-1] == B[n-1]) && (A[n-1] != Sum[n-1])

Same signs → different result sign
No Overflow Possible

Adding numbers with different signs never overflows (signed).

Opposite signs safe

Detection Methods

Signed Overflow Detection
Method 1: Carry XOR
V = Cin[MSB] ⊕ Cout[MSB]

Method 2: Sign Check (Addition)
V = (A[MSB] == B[MSB]) AND (A[MSB] != Sum[MSB])
V = (~A[MSB] & ~B[MSB] & Sum[MSB]) | (A[MSB] & B[MSB] & ~Sum[MSB])

Method 3: Sign Check (Subtraction)
V = (A[MSB] != B[MSB]) AND (A[MSB] != Diff[MSB])

Detailed Example: 100 + 50 (signed 8-bit)

A = 100 = 0110 0100
B =  50 = 0011 0010
          ─────────
Carries:  0110 0100  ← Cin[7] = 0
Sum:      1001 0110  = 0x96 = -106 (wrong!)

Cout[7] = 0 (no final carry)
Cin[7]  = 0 (carry into MSB position)

V = Cin[7] XOR Cout[7] = 0 XOR 0 = 0 ❌

Wait, let me recalculate...

Actually:    0 1 1 0 0 1 0 0   (A = 100)
           + 0 0 1 1 0 0 1 0   (B = 50)
             ─────────────────
Carries:   0 1 1 0 0 0 0 0 _   (C7=1, Cout=0)
           ─────────────────
Sum:         1 0 0 1 0 1 1 0   = 0x96 = -106

Cin[7] = 1, Cout[7] = 0
V = 1 XOR 0 = 1 ✓ OVERFLOW DETECTED

Alternative check:
- A[7] = 0 (positive)
- B[7] = 0 (positive)  
- Sum[7] = 1 (negative)
- Same signs, different result → OVERFLOW ✓

RTL Implementation

VERILOG overflow_detect.v 
// Simple Overflow Detection
module overflow_add #(
    parameter WIDTH = 8
)(
    input      [WIDTH-1:0] a,
    input      [WIDTH-1:0] b,
    output     [WIDTH-1:0] sum,
    output                  carry,      // Unsigned overflow
    output                  overflow    // Signed overflow
);

wire [WIDTH:0] result = {1'b0, a} + {1'b0, b};
assign sum   = result[WIDTH-1:0];
assign carry = result[WIDTH];

// Signed overflow: same sign inputs, different sign output
assign overflow = (~a[WIDTH-1] & ~b[WIDTH-1] & sum[WIDTH-1]) |
                  (a[WIDTH-1] & b[WIDTH-1] & ~sum[WIDTH-1]);

endmodule

// Overflow Detection for Subtraction
module overflow_sub #(
    parameter WIDTH = 8
)(
    input      [WIDTH-1:0] a,
    input      [WIDTH-1:0] b,
    output     [WIDTH-1:0] diff,
    output                  borrow,     // Unsigned underflow
    output                  overflow    // Signed overflow
);

wire [WIDTH:0] result = {1'b0, a} - {1'b0, b};
assign diff   = result[WIDTH-1:0];
assign borrow = result[WIDTH];  // Borrow = ~Cout for subtraction

// Signed overflow: different signs, result sign matches b
assign overflow = (~a[WIDTH-1] & b[WIDTH-1] & diff[WIDTH-1]) |
                  (a[WIDTH-1] & ~b[WIDTH-1] & ~diff[WIDTH-1]);

endmodule

// Using Carry-In XOR Carry-Out Method
module overflow_xor #(
    parameter WIDTH = 8
)(
    input      [WIDTH-1:0] a,
    input      [WIDTH-1:0] b,
    input                   cin,
    output     [WIDTH-1:0] sum,
    output                  cout,
    output                  overflow
);

// Get both carry into MSB and carry out
wire [WIDTH:0] carries;
assign carries[0] = cin;

genvar i;
generate
    for (i = 0; i < WIDTH; i = i + 1) begin : adder
        assign sum[i] = a[i] ^ b[i] ^ carries[i];
        assign carries[i+1] = (a[i] & b[i]) | 
                               (a[i] & carries[i]) | 
                               (b[i] & carries[i]);
    end
endgenerate

assign cout = carries[WIDTH];

// Overflow = Cin to MSB XOR Cout from MSB
assign overflow = carries[WIDTH-1] ^ carries[WIDTH];

endmodule

// Complete ALU with All Flags
module alu_flags #(
    parameter WIDTH = 32
)(
    input      [WIDTH-1:0] a,
    input      [WIDTH-1:0] b,
    input                   is_sub,
    output     [WIDTH-1:0] result,
    output                  Z,  // Zero flag
    output                  N,  // Negative flag
    output                  C,  // Carry/Borrow flag
    output                  V   // Overflow flag
);

wire [WIDTH-1:0] b_operand = is_sub ? ~b : b;
wire cin = is_sub;  // For A - B = A + ~B + 1

wire [WIDTH:0] sum_ext = {1'b0, a} + {1'b0, b_operand} + cin;

assign result = sum_ext[WIDTH-1:0];
assign Z = (result == {WIDTH{1'b0}});
assign N = result[WIDTH-1];
assign C = sum_ext[WIDTH];

// Overflow using sign-based detection
assign V = (a[WIDTH-1] == b_operand[WIDTH-1]) && 
           (a[WIDTH-1] != result[WIDTH-1]);

endmodule
VHDL overflow_detect.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

entity overflow_add is
    generic (WIDTH : natural := 8);
    port (
        a, b     : in  std_logic_vector(WIDTH-1 downto 0);
        sum      : out std_logic_vector(WIDTH-1 downto 0);
        carry    : out std_logic;
        overflow : out std_logic
    );
end overflow_add;

architecture behavioral of overflow_add is
    signal result : unsigned(WIDTH downto 0);
    signal sum_int : std_logic_vector(WIDTH-1 downto 0);
begin
    result <= ('0' & unsigned(a)) + ('0' & unsigned(b));
    sum_int <= std_logic_vector(result(WIDTH-1 downto 0));
    sum <= sum_int;
    carry <= result(WIDTH);
    
    -- Signed overflow: same sign inputs, different sign output
    overflow <= (not a(WIDTH-1) and not b(WIDTH-1) and sum_int(WIDTH-1)) or
                (a(WIDTH-1) and b(WIDTH-1) and not sum_int(WIDTH-1));
end behavioral;

-- ALU with all flags
entity alu_flags is
    generic (WIDTH : natural := 32);
    port (
        a, b   : in  std_logic_vector(WIDTH-1 downto 0);
        is_sub : in  std_logic;
        result : out std_logic_vector(WIDTH-1 downto 0);
        Z, N, C, V : out std_logic
    );
end alu_flags;

architecture behavioral of alu_flags is
    signal b_op : std_logic_vector(WIDTH-1 downto 0);
    signal res : unsigned(WIDTH downto 0);
begin
    b_op <= not b when is_sub = '1' else b;
    res <= ('0' & unsigned(a)) + unsigned(b_op) + 
           ("" & is_sub);
    
    result <= std_logic_vector(res(WIDTH-1 downto 0));
    Z <= '1' when res(WIDTH-1 downto 0) = 0 else '0';
    N <= res(WIDTH-1);
    C <= res(WIDTH);
    V <= (a(WIDTH-1) xnor b_op(WIDTH-1)) and 
         (a(WIDTH-1) xor res(WIDTH-1));
end behavioral;

Applications

ALU Flags

V flag used for signed comparisons and conditional branches.

BGT, BLT, BLE, BGE
Exception Handling

Trap on overflow for arithmetic exceptions in some ISAs.

MIPS: ADD vs ADDU
Saturating Arithmetic

Overflow flag triggers saturation to max/min value.

DSP, audio processing
BigNum Library

Detect overflow to trigger carry propagation to next word.

Multi-word arithmetic

Flag Usage for Signed Comparisons

A == B Z = 1
A != B Z = 0
A < B (signed) N ⊕ V = 1
A >= B (signed) N ⊕ V = 0
A < B (unsigned) C = 0 (borrow occurred)
A >= B (unsigned) C = 1 (no borrow)

Interview Notes

  • Unsigned overflow = carry out. Signed overflow = Cin[MSB] XOR Cout[MSB].
  • Signed addition overflow: same sign operands, different sign result.
  • Adding different-sign operands NEVER causes signed overflow.
  • For subtraction: V = (A[MSB] != B[MSB]) && (A[MSB] != Result[MSB]).
  • Hardware cost: just 1 XOR gate (given access to MSB carries).
  • Signed comparison after CMP: use N XOR V for less-than condition.
  • MIPS: ADD traps on overflow, ADDU ignores overflow (same hardware, different handling).
