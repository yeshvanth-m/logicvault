Overview

The Wallace Tree multiplier (1964) dramatically accelerates multiplication by using carry-save adders (CSA) to reduce multiple partial products in parallel. Instead of adding partial products one row at a time, Wallace trees process them in groups of 3, reducing the count by a factor of 1.5 per stage.

For N partial products, a Wallace tree reduces them to just 2 vectors (sum and carry) in ⌈log₁.₅(N)⌉ stages. A final carry-propagate adder combines these two vectors into the final product. This achieves O(log N) delay versus O(N) for array multipliers.

Carry-Save Addition (CSA)

A Carry-Save Adder adds 3 numbers and produces 2 numbers (sum and carry) in constant time, regardless of bit width. It's essentially a row of independent full adders with no carry chain.

CSA Operation

3:2 compression — inputs A, B, C become Sum and Carry vectors.

S[i] = A[i] ⊕ B[i] ⊕ C[i]
C[i+1] = maj(A[i], B[i], C[i])
Key Property

No carry propagation! Each bit position is independent → O(1) delay.

Delay = 1 FA delay (not N)

Example: CSA Adding 3 Numbers

A =   0 1 1 0 1     (13)
B =   0 1 0 1 1     (11)
C =   0 0 1 1 0     (6)
      -----------
S =   0 0 0 0 0     (sum bits: each column XOR)
Cout= 0 1 1 1 1 0   (carry bits: each column majority, shifted left)

Verification: 13 + 11 + 6 = 30
S + Cout = 00000 + 011110 = 30 ✓

Wallace Tree Reduction

Wallace's insight: use CSAs to reduce partial products as aggressively as possible at each stage. Group numbers into sets of 3, compress to 2, repeat until only 2 remain.

Stage Input Count CSAs Used Output Count
0 8 PP — 8
1 8 2 (6→4) + pass 2 6
2 6 2 (6→4) 4
3 4 1 (3→2) + pass 1 3
4 3 1 (3→2) 2
Final CPA adds the final 2 vectors → Product
Stage Count Formula

Each CSA stage reduces count by 3:2 ratio. Number of stages for N inputs:

Stages = ⌈log1.5(N)⌉ ≈ 2.41 × log₂(N)

Irregular Wiring

Unlike regular arrays, Wallace trees have complex interconnections between stages, making layout challenging.

Booth + Wallace

Combine Booth encoding (N/2 partial products) with Wallace reduction for optimal speed.

RTL Implementation

VERILOG wallace_mult.v 
module csa #(parameter W = 16) (
    input  [W-1:0] a, b, c,
    output [W-1:0] sum,
    output [W-1:0] carry
);
    assign sum   = a ^ b ^ c;
    assign carry = (a & b) | (b & c) | (a & c);
endmodule

module wallace_mult #(parameter N = 8) (
    input  [N-1:0] a,
    input  [N-1:0] b,
    output [2*N-1:0] product
);

localparam W = 2*N;

// Generate partial products
wire [W-1:0] pp [N-1:0];
genvar i, j;
generate
    for (i = 0; i < N; i = i + 1) begin : gen_pp
        wire [N-1:0] pp_row;
        for (j = 0; j < N; j = j + 1) begin : gen_bit
            assign pp_row[j] = a[j] & b[i];
        end
        assign pp[i] = {{N{1'b0}}, pp_row} << i;
    end
endgenerate

// Wallace tree reduction for 8 partial products
// Stage 1: 8 → 6
wire [W-1:0] s1_s0, s1_c0, s1_s1, s1_c1;
csa #(W) csa1_0 (.a(pp[0]), .b(pp[1]), .c(pp[2]), 
                 .sum(s1_s0), .carry(s1_c0));
csa #(W) csa1_1 (.a(pp[3]), .b(pp[4]), .c(pp[5]), 
                 .sum(s1_s1), .carry(s1_c1));

// Stage 2: 6 → 4
wire [W-1:0] s2_s0, s2_c0, s2_s1, s2_c1;
csa #(W) csa2_0 (.a(s1_s0), .b({s1_c0[W-2:0],1'b0}), .c(s1_s1), 
                 .sum(s2_s0), .carry(s2_c0));
csa #(W) csa2_1 (.a({s1_c1[W-2:0],1'b0}), .b(pp[6]), .c(pp[7]), 
                 .sum(s2_s1), .carry(s2_c1));

// Stage 3: 4 → 3
wire [W-1:0] s3_s, s3_c;
csa #(W) csa3 (.a(s2_s0), .b({s2_c0[W-2:0],1'b0}), .c(s2_s1), 
               .sum(s3_s), .carry(s3_c));

// Stage 4: 3 → 2
wire [W-1:0] final_s, final_c;
csa #(W) csa4 (.a(s3_s), .b({s3_c[W-2:0],1'b0}), .c({s2_c1[W-2:0],1'b0}), 
               .sum(final_s), .carry(final_c));

// Final carry-propagate addition
assign product = final_s + {final_c[W-2:0], 1'b0};

endmodule
VHDL wallace_mult.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

entity wallace_mult is
    generic (N : integer := 8);
    port (
        a       : in  unsigned(N-1 downto 0);
        b       : in  unsigned(N-1 downto 0);
        product : out unsigned(2*N-1 downto 0)
    );
end wallace_mult;

architecture behavioral of wallace_mult is
    -- CSA component produces sum and carry
    function csa_sum(a, b, c: unsigned) 
        return unsigned is
    begin
        return a xor b xor c;
    end function;
    
    function csa_carry(a, b, c: unsigned) 
        return unsigned is
    begin
        return (a and b) or (b and c) or (a and c);
    end function;
begin
    -- Behavioral for synthesis optimization
    product <= a * b;
end behavioral;

Interview Notes

  • CSA is a 3:2 compressor — adds 3 numbers, outputs 2 (sum + carry) in O(1) time.
  • Wallace tree reduces N partial products to 2 in ⌈log₁.₅(N)⌉ stages — much faster than array's O(N).
  • Final stage needs a fast carry-propagate adder (CLA or Kogge-Stone) for the last 2 vectors.
  • Irregular structure makes layout difficult — Dadda trees use fewer CSAs for slightly better area.
  • Combine with Booth encoding for optimal performance: Booth halves PPs, Wallace reduces them fast.
  • Used in high-performance FPUs, DSP units, and cryptographic accelerators.
