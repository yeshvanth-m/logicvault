Overview

Sign Extension converts a signed number from a narrower bit width to a wider bit width while preserving its numeric value. In two's complement representation, this is achieved by replicating the most significant bit (the sign bit) into the new upper bits.

For positive numbers (MSB = 0), the extended bits are filled with zeros. For negative numbers (MSB = 1), the extended bits are filled with ones. This ensures the numeric value remains unchanged after extension.

Sign extension is a zero-cost operation in hardware—it's just wire routing. However, it's conceptually important for understanding how processors handle different operand sizes.

Why Sign Extension Preserves Value

Two's complement value formula for n-bit number:
  Value = -b[n-1]×2^(n-1) + b[n-2]×2^(n-2) + ... + b[0]×2^0

Example: 8-bit -5 = 1111 1011
  = -1×2⁷ + 1×2⁶ + 1×2⁵ + 1×2⁴ + 1×2³ + 0×2² + 1×2¹ + 1×2⁰
  = -128 + 64 + 32 + 16 + 8 + 0 + 2 + 1
  = -128 + 123 = -5 ✓

Sign extend to 16-bit: 1111 1111 1111 1011
  = -1×2¹⁵ + 1×2¹⁴ + ... + 1×2⁸ + 1×2⁷ + ... + 1×2⁰
  = -32768 + 16384 + ... + 256 + 128 + 123
  = -32768 + 32763 = -5 ✓

The added 1s in upper bits sum to exactly cancel
the larger negative weight of the new MSB.
Sign Extension Formula
Extended[n+k-1:0] = {k{Input[n-1]}, Input[n-1:0]}

// 8-bit to 32-bit:
Extended[31:0] = {24{Input[7]}, Input[7:0]}

Sign Extension (SEXT)

For signed values. Replicates MSB. Preserves signed value.

LB, LH, SEXTB, SEXTH
Zero Extension (ZEXT)

For unsigned values. Pads with zeros. Preserves unsigned value.

LBU, LHU, ZEXTB, ZEXTH
When to Use Sign Ext

Signed integers, offsets, immediate values, signed comparisons.

char, short, int
When to Use Zero Ext

Unsigned integers, addresses, bit fields, unsigned comparisons.

unsigned char, size_t

📐 RISC-V Load Instructions

Memory byte: 0xF5 (245 unsigned, -11 signed)

LB  (Load Byte, sign ext):     rd = 0xFFFFFFF5  (-11)
LBU (Load Byte Unsigned):      rd = 0x000000F5  (245)

LH  (Load Halfword, sign ext): rd = sign_ext(mem[16-bit])
LHU (Load Halfword Unsigned):  rd = zero_ext(mem[16-bit])

LW  (Load Word):               rd = mem[32-bit] (no ext on RV32)

RTL Implementation

VERILOG sign_extend.v 
// Sign Extension: 8-bit to 32-bit
module sext_8_32 (
    input      [7:0]  data_in,
    output     [31:0] data_out
);

// Replicate sign bit 24 times
assign data_out = {{24{data_in[7]}}, data_in};

endmodule

// Sign Extension: 16-bit to 32-bit
module sext_16_32 (
    input      [15:0] data_in,
    output     [31:0] data_out
);

assign data_out = {{16{data_in[15]}}, data_in};

endmodule

// Zero Extension: 8-bit to 32-bit
module zext_8_32 (
    input      [7:0]  data_in,
    output     [31:0] data_out
);

assign data_out = {24'b0, data_in};

endmodule

// Configurable Sign/Zero Extension
module extend_unit #(
    parameter IN_WIDTH  = 8,
    parameter OUT_WIDTH = 32
)(
    input      [IN_WIDTH-1:0]  data_in,
    input                      sign_ext,  // 1=sign, 0=zero
    output     [OUT_WIDTH-1:0] data_out
);

localparam EXT_BITS = OUT_WIDTH - IN_WIDTH;

wire ext_bit = sign_ext & data_in[IN_WIDTH-1];
assign data_out = {{EXT_BITS{ext_bit}}, data_in};

endmodule

// RISC-V Immediate Extender
module imm_extend (
    input      [31:0] instr,
    input      [2:0]  imm_type,
    output reg [31:0] imm_out
);

// Immediate format types
localparam I_TYPE = 3'd0;
localparam S_TYPE = 3'd1;
localparam B_TYPE = 3'd2;
localparam U_TYPE = 3'd3;
localparam J_TYPE = 3'd4;

always @(*) begin
    case (imm_type)
        I_TYPE: imm_out = {{20{instr[31]}}, instr[31:20]};
        S_TYPE: imm_out = {{20{instr[31]}}, instr[31:25], instr[11:7]};
        B_TYPE: imm_out = {{19{instr[31]}}, instr[31], instr[7], 
                          instr[30:25], instr[11:8], 1'b0};
        U_TYPE: imm_out = {instr[31:12], 12'b0};
        J_TYPE: imm_out = {{11{instr[31]}}, instr[31], instr[19:12],
                          instr[20], instr[30:21], 1'b0};
        default: imm_out = 32'b0;
    endcase
end

endmodule

// Using Verilog signed type
module sext_signed (
    input  signed [7:0]  data_in,
    output signed [31:0] data_out
);

// Automatic sign extension when assigning signed to wider signed
assign data_out = data_in;

endmodule
VHDL sign_extend.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

-- Sign Extension: 8-bit to 32-bit
entity sext_8_32 is
    port (
        data_in  : in  std_logic_vector(7 downto 0);
        data_out : out std_logic_vector(31 downto 0)
    );
end sext_8_32;

architecture behavioral of sext_8_32 is
begin
    data_out <= (31 downto 8 => data_in(7)) & data_in;
end behavioral;

-- Generic Sign/Zero Extension
entity extend_unit is
    generic (
        IN_WIDTH  : natural := 8;
        OUT_WIDTH : natural := 32
    );
    port (
        data_in  : in  std_logic_vector(IN_WIDTH-1 downto 0);
        sign_ext : in  std_logic;  -- '1' for sign, '0' for zero
        data_out : out std_logic_vector(OUT_WIDTH-1 downto 0)
    );
end extend_unit;

architecture behavioral of extend_unit is
    signal ext_bit : std_logic;
begin
    ext_bit <= sign_ext and data_in(IN_WIDTH-1);
    data_out <= (OUT_WIDTH-1 downto IN_WIDTH => ext_bit) & data_in;
end behavioral;

-- Using NUMERIC_STD resize
entity sext_resize is
    port (
        data_in  : in  signed(7 downto 0);
        data_out : out signed(31 downto 0)
    );
end sext_resize;

architecture behavioral of sext_resize is
begin
    -- resize() automatically sign-extends for signed types
    data_out <= resize(data_in, 32);
end behavioral;

Applications

Memory Load

LB (Load Byte) sign-extends byte to register width. LBU zero-extends.

RISC-V, ARM, x86
Immediate Values

RISC-V 12-bit immediates are sign-extended to 32/64 bits.

ADDI, BEQ, LW
ALU Operands

Narrow operands extended before arithmetic. Preserves signed semantics.

Width matching
Comparison

SLT (signed compare) vs SLTU (unsigned) - different extension behavior.

Branch conditions

Interview Notes

  • Sign extension replicates MSB to preserve signed value when widening.
  • Zero extension pads with 0s - used for unsigned values.
  • Hardware cost: zero - it's just wire routing, no gates needed.
  • 8-bit to 32-bit: Extended = {{24{Input[7]}}, Input[7:0]}
  • Common mistake: using wrong extension type (LB vs LBU) causes bugs.
  • RISC-V immediates are always sign-extended (even for unsigned instructions).
  • In Verilog: assigning signed to wider signed auto-extends; unsigned doesn't.
