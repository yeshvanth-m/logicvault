Overview

The Arithmetic Logic Unit (ALU) is the combinational circuit at the heart of every processor. It performs arithmetic operations (addition, subtraction), logical operations (AND, OR, XOR), shifts, and comparisons based on an opcode that selects the operation.

A typical ALU consists of parallel functional units: an adder/subtractor, a logic unit, and a barrel shifter. A multiplexer at the output selects the appropriate result based on the opcode. The ALU also generates condition flags (zero, carry, overflow, negative) that are used for conditional branching.

Adder/Subtractor

Core arithmetic. SUB = ADD with B inverted + Cin=1. Uses CLA for speed.

A + B or A + ~B + 1
Logic Unit

Bitwise operations. AND, OR, XOR, NOT applied to each bit position.

Parallel bit operations
Barrel Shifter

Variable shift in single cycle. Supports logical and arithmetic shifts.

log₂(n) stages
Output MUX

Selects result from functional unit based on opcode.

4:1 or 8:1 MUX

Condition Flags

Flag Name Condition Generation
Z Zero Result is zero Z = ~|Result
N Negative Result MSB is 1 N = Result[n-1]
C Carry Unsigned overflow C = Cout (from adder)
V Overflow Signed overflow V = Cin[n-1] ^ Cout[n-1]
Flag Equations
Zero = (Result == 0) = NOR of all result bits
Negative = Result[MSB]
Carry = Cout from final adder stage
Overflow = Cin[MSB] ⊕ Cout[MSB]

Condition Code Usage

Unsigned < C = 0 (borrow occurred)
Unsigned ≥ C = 1 (no borrow)
Signed < N ≠ V
Signed ≥ N = V
Equal Z = 1
Not Equal Z = 0

RTL Implementation

VERILOG alu.v 
// 32-bit ALU with Flags
module alu #(
    parameter WIDTH = 32
)(
    input      [WIDTH-1:0] a,
    input      [WIDTH-1:0] b,
    input      [3:0]        op,
    input                   cin,       // Carry in (for ADC/SBC)
    output reg [WIDTH-1:0] result,
    output                  zero,      // Z flag
    output                  negative,  // N flag
    output reg              carry,     // C flag
    output                  overflow   // V flag
);

// Operation codes
localparam OP_ADD = 4'b0000;
localparam OP_SUB = 4'b0001;
localparam OP_ADC = 4'b0010;
localparam OP_SBC = 4'b0011;
localparam OP_AND = 4'b0100;
localparam OP_OR  = 4'b0101;
localparam OP_XOR = 4'b0110;
localparam OP_NOT = 4'b0111;
localparam OP_SLL = 4'b1000;
localparam OP_SRL = 4'b1001;
localparam OP_SRA = 4'b1010;
localparam OP_CMP = 4'b1011;
localparam OP_MOV = 4'b1101;

// Internal signals
wire [WIDTH:0] add_result;
wire sub_mode;
wire [WIDTH-1:0] b_operand;
wire carry_in;

// Subtraction uses 2's complement: A - B = A + ~B + 1
assign sub_mode = (op == OP_SUB) || (op == OP_SBC) || (op == OP_CMP);
assign b_operand = sub_mode ? ~b : b;
assign carry_in = (op == OP_SUB || op == OP_CMP) ? 1'b1 :
                   (op == OP_ADC) ? cin :
                   (op == OP_SBC) ? cin : 1'b0;

// Adder (used for ADD, SUB, ADC, SBC, CMP)
assign add_result = {1'b0, a} + {1'b0, b_operand} + carry_in;

// Main ALU logic
always @(*) begin
    carry = 1'b0;
    case (op)
        OP_ADD, OP_SUB, OP_ADC, OP_SBC: begin
            result = add_result[WIDTH-1:0];
            carry = add_result[WIDTH];
        end
        OP_AND: result = a & b;
        OP_OR:  result = a | b;
        OP_XOR: result = a ^ b;
        OP_NOT: result = ~a;
        OP_SLL: result = a << b[4:0];
        OP_SRL: result = a >> b[4:0];
        OP_SRA: result = $signed(a) >>> b[4:0];
        OP_CMP: begin
            result = add_result[WIDTH-1:0];  // For flags only
            carry = add_result[WIDTH];
        end
        OP_MOV: result = b;
        default: result = {WIDTH{1'b0}};
    endcase
end

// Flag generation
assign zero = (result == {WIDTH{1'b0}});
assign negative = result[WIDTH-1];

// Overflow: sign of result differs from expected
// For ADD: overflow if signs of operands match but result differs
// V = (A[MSB] == B[MSB]) && (A[MSB] != Result[MSB])
wire add_overflow = (a[WIDTH-1] == b_operand[WIDTH-1]) && 
                     (a[WIDTH-1] != add_result[WIDTH-1]);
assign overflow = (op == OP_ADD || op == OP_SUB || 
                   op == OP_ADC || op == OP_SBC || 
                   op == OP_CMP) ? add_overflow : 1'b0;

endmodule
VHDL alu.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

entity alu is
    generic (WIDTH : natural := 32);
    port (
        a, b     : in  std_logic_vector(WIDTH-1 downto 0);
        op       : in  std_logic_vector(3 downto 0);
        result   : out std_logic_vector(WIDTH-1 downto 0);
        zero     : out std_logic;
        negative : out std_logic;
        carry    : out std_logic;
        overflow : out std_logic
    );
end alu;

architecture behavioral of alu is
    signal res : std_logic_vector(WIDTH-1 downto 0);
    signal sum : std_logic_vector(WIDTH downto 0);
begin
    process(a, b, op)
        variable tmp : unsigned(WIDTH downto 0);
    begin
        case op is
            when "0000" =>  -- ADD
                tmp := ('0' & unsigned(a)) + unsigned(b);
                res <= std_logic_vector(tmp(WIDTH-1 downto 0));
                carry <= tmp(WIDTH);
            when "0001" =>  -- SUB
                tmp := ('0' & unsigned(a)) - unsigned(b);
                res <= std_logic_vector(tmp(WIDTH-1 downto 0));
            when "0100" => res <= a and b;
            when "0101" => res <= a or b;
            when "0110" => res <= a xor b;
            when others => res <= (others => '0');
        end case;
    end process;
    
    result <= res;
    zero <= '1' when res = (WIDTH-1 downto 0 => '0') else '0';
    negative <= res(WIDTH-1);
end behavioral;

Optimization

Unified Adder

Single adder for ADD/SUB/CMP. SUB uses inverted B + Cin=1.

A - B = A + ~B + 1
Parallel Execution

All functional units compute in parallel. MUX selects result.

Critical path: slowest unit
Flag Bypass

Forward flags to branch unit before result register write.

Reduces branch latency
Fast Zero Detect

Use tree of NOR gates for wide zero detection. O(log n) depth.

Parallel NOR reduction

Interview Notes

  • ALU = adder/subtractor + logic unit + shifter + output MUX. All compute in parallel.
  • SUB uses same adder as ADD: A - B = A + ~B + 1 (invert B, set carry-in to 1).
  • Flags: Z (zero = NOR all bits), N (negative = MSB), C (carry out), V (signed overflow).
  • Overflow: V = Cin[MSB] XOR Cout[MSB]. Or: same signs in, different sign out.
  • Signed comparison: use N XOR V. Unsigned comparison: use C flag.
  • CMP instruction = SUB but discard result, only update flags.
