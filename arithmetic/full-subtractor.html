Truth Table

#

The full subtractor computes A - B - Bin. The borrow-out (Bout) is 1 whenever we need to borrow from the next higher bit position.

A B Bin D Bout A - B - Bin
000000 - 0 - 0 = 0
001110 - 0 - 1 = -1
010110 - 1 - 0 = -1
011010 - 1 - 1 = -2
100101 - 0 - 0 = 1
101001 - 0 - 1 = 0
110001 - 1 - 0 = 0
111111 - 1 - 1 = -1

Borrow semantics: Bout = 1 when the sum (B + Bin) > A. This indicates we need to borrow from the next higher bit. The Difference D gives the 2's complement result modulo 2.

Boolean Expression

#

Difference

D = A ⊕ B ⊕ Bin

Borrow-out

Bout = A'·B + A'·Bin + B·Bin

Alternative Borrow-out (using XOR)

Bout = A'·(A ⊕ B)' + Bin·(A ⊕ B)

Canonical SOP (minterms)

D = Σm(1, 2, 4, 7)  |  Bout = Σm(1, 2, 3, 7)

Key insight: The Difference output is identical to the Full Adder's Sum — both are 3-input XOR (odd parity). The Borrow-out is a "minority function" of A against B+Bin.

Derivation from Half Subtractors

#

A full subtractor can be constructed from two half subtractors and an OR gate, mirroring the full adder construction:

Two-Stage Construction

Stage 1 (HS1): Compute A - B

D1 = A ⊕ B

Borrow1 = A' · B

Stage 2 (HS2): Compute D1 - Bin

D = D1 ⊕ Bin = A ⊕ B ⊕ Bin

Borrow2 = D1' · Bin

Final Borrow-out:

Bout = Borrow1 + Borrow2

OR gate reasoning: A borrow is needed if either HS1 generates a borrow (B > A) OR HS2 generates a borrow (Bin > D1). These are mutually exclusive in terms of "both true", so OR works correctly.

The full subtractor can be built from two half subtractors and an OR gate:

  • Half Subtractor 1: Computes A - B, producing D1 and Borrow1
  • Half Subtractor 2: Computes D1 - Bin, producing final D and Borrow2
  • OR gate: Combines Borrow1 and Borrow2 to produce Bout

RTL Implementation

#
Verilog full_subtractor.v 
// Full Subtractor — Dataflow style
module full_subtractor (
    input  wire a,      // Minuend
    input  wire b,      // Subtrahend
    input  wire bin,    // Borrow-in
    output wire diff,   // Difference
    output wire bout    // Borrow-out
);

    assign diff = a ^ b ^ bin;
    assign bout = (~a & b) | (~a & bin) | (b & bin);

endmodule


// ─── Structural: Two Half Subtractors ──────────────
module half_subtractor (input a, b, output diff, borrow);
    assign diff   = a ^ b;
    assign borrow = ~a & b;
endmodule

module full_subtractor_struct (
    input  a, b, bin,
    output diff, bout
);
    wire d1, borrow1, borrow2;

    half_subtractor HS1 (.a(a),  .b(b),   .diff(d1),   .borrow(borrow1));
    half_subtractor HS2 (.a(d1), .b(bin), .diff(diff), .borrow(borrow2));
    assign bout = borrow1 | borrow2;

endmodule

VHDL full_subtractor.vhd 
-- Full Subtractor - Dataflow architecture
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity full_subtractor is
    port (
        a    : in  std_logic;  -- Minuend
        b    : in  std_logic;  -- Subtrahend
        bin  : in  std_logic;  -- Borrow-in
        diff : out std_logic;  -- Difference
        bout : out std_logic   -- Borrow-out
    );
end full_subtractor;

architecture dataflow of full_subtractor is
begin
    diff <= a xor b xor bin;
    bout <= ((not a) and b) or 
            ((not a) and bin) or 
            (b and bin);
end dataflow;


-- Structural version using half subtractors
architecture structural of full_subtractor is
    component half_subtractor
        port (a, b : in std_logic; diff, borrow : out std_logic);
    end component;
    
    signal d1, borrow1, borrow2 : std_logic;
begin
    HS1: half_subtractor port map (a, b, d1, borrow1);
    HS2: half_subtractor port map (d1, bin, diff, borrow2);
    bout <= borrow1 or borrow2;
end structural;

Interview Notes

#
  • What's the relation to full adder? — Same Difference/Sum equation (3-input XOR). Borrow-out uses A' instead of A in the carry terms.
  • How to cascade for N-bit subtraction? — Chain full subtractors: Bout of stage i connects to Bin of stage i+1. LSB has Bin=0.
  • Can you use adders for subtraction? — Yes: A - B = A + (~B) + 1. Invert B, set Cin=1. This is how ALUs typically implement subtraction.
  • What function is Bout? — It's the "minority" function of A vs (B+Bin): true when A is "outvoted" by B and Bin combined.
  • Gate count comparison? — Full subtractor: ~6-7 gates. Full adder: ~9 gates. Subtractor is slightly simpler due to borrow vs carry logic.
  • Critical path? — Through the XOR chain for Difference (2 XOR delays) or through AND-OR for Bout (2-3 delays).
  • Two's complement advantage? — Using adder with inverted B avoids separate subtractor hardware. One adder handles both operations.
