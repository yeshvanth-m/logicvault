Overview

Cyclic Redundancy Check (CRC) is a powerful error-detection scheme based on polynomial arithmetic over GF(2) (binary field). The sender divides the message polynomial by a generator polynomial and appends the remainder (CRC checksum) to the message. The receiver performs the same division—a zero remainder indicates no detected errors.

CRC is superior to simple parity for detecting burst errors common in serial communication and storage. It's implemented efficiently in hardware using Linear Feedback Shift Registers (LFSRs) and is ubiquitous in Ethernet, USB, ZIP files, and storage systems.

Theory

GF(2) Arithmetic

Addition/subtraction = XOR. No carries. Multiplication follows polynomial rules.

1 + 1 = 0, 1 - 1 = 0
Message Polynomial

Data bits as polynomial coefficients. MSB is highest degree term.

1011 → x³ + x + 1
Generator Polynomial

Standardized polynomial that divides the message. Degree = CRC bits.

G(x) = x³ + x + 1
CRC Calculation

Append n zeros, divide by G(x), remainder is CRC.

CRC = M(x)·xⁿ mod G(x)
Mathematical Definition
Transmitted = M(x) · xn + R(x)

Where:
• M(x) = message polynomial
• n = degree of generator G(x)
• R(x) = M(x)·xn mod G(x) = CRC remainder

Detection Capabilities

All single-bit errors 100% detected
All double-bit errors 100% (if G(x) has factor x+1)
All odd-bit errors 100% (if G(x) has factor x+1)
Burst errors ≤ n bits 100% detected
Burst errors = n+1 bits 1 - 2-(n-1) detected
Random errors 1 - 2-n detected

Polynomial Selection

Good polynomials have high Hamming distance, detect specific error patterns.

Primitive polynomials preferred
CRC Width

More bits = better detection but higher overhead. 32-bit is standard for networks.

Trade-off: coverage vs. overhead

Calculation Process

CRC Encoding Steps

1. Choose generator polynomial G(x) of degree n 2. Append n zeros to message M (shift left by n) 3. Divide M·xⁿ by G using XOR division 4. Remainder R is the CRC checksum 5. Transmit: original message + R

CRC Verification Steps

1. Receive message + CRC 2. Divide entire received word by G(x) 3. If remainder = 0: No error detected 4. If remainder ≠ 0: Error detected
XOR Division Rules
• If MSB of dividend = 1: XOR with divisor, shift left
• If MSB of dividend = 0: XOR with 0 (just shift left)
• Continue until all message bits processed
• Remaining bits in register = CRC remainder

LFSR Circuit

📐 CRC-3 LFSR for G(x) = x³ + x + 1

Data ──→(+)─┬──→[ R₂ ]──→[ R₁ ]──→(+)──→[ R₀ ]───→ out
           │                       ↑
           └───────────────────────┘
           
(+) = XOR gate
[ Rᵢ ] = D flip-flop (register bit)

XOR feedback at positions matching G(x) terms:
G(x) = x³ + x¹ + x⁰ → feedback after R₂ and R₀
Serial Implementation

Process one bit per clock. Simple, low area, but slow for high-speed applications.

n clocks for n-bit message
Parallel Implementation

Process multiple bits (e.g., 8, 32) per clock. Uses lookup tables or matrix math.

Higher throughput
LFSR Structure

Shift register with XOR feedback. Feedback taps match polynomial terms.

Hardware-efficient
Initial Value

Register initialized to all 1s or all 0s depending on standard.

CRC-32: init=0xFFFFFFFF

RTL Implementation

VERILOG crc.v 
// CRC-8 Serial Implementation
// G(x) = x^8 + x^2 + x + 1 (0x07)
module crc8_serial (
    input        clk,
    input        rst_n,
    input        data_in,
    input        data_valid,
    output [7:0] crc
);

reg [7:0] crc_reg;

wire feedback = data_in ^ crc_reg[7];

always @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
        crc_reg <= 8'h00;
    end else if (data_valid) begin
        // Shift and apply feedback at polynomial taps
        crc_reg[7] <= crc_reg[6];
        crc_reg[6] <= crc_reg[5];
        crc_reg[5] <= crc_reg[4];
        crc_reg[4] <= crc_reg[3];
        crc_reg[3] <= crc_reg[2];
        crc_reg[2] <= crc_reg[1] ^ feedback;  // x^2 term
        crc_reg[1] <= crc_reg[0] ^ feedback;  // x^1 term
        crc_reg[0] <= feedback;              // x^0 term
    end
end

assign crc = crc_reg;

endmodule

// CRC-8 Parallel (8-bit input)
module crc8_parallel (
    input        clk,
    input        rst_n,
    input  [7:0] data_in,
    input        data_valid,
    output [7:0] crc
);

reg [7:0] crc_reg;
wire [7:0] crc_next;

// Parallel CRC calculation (derived from serial equations)
wire [7:0] d = data_in;
wire [7:0] c = crc_reg;

assign crc_next[0] = d[7] ^ d[6] ^ d[0] ^ c[0] ^ c[6] ^ c[7];
assign crc_next[1] = d[7] ^ d[1] ^ d[0] ^ c[0] ^ c[1] ^ c[7];
assign crc_next[2] = d[7] ^ d[2] ^ d[1] ^ d[0] ^ c[0] ^ c[1] ^ c[2] ^ c[7];
assign crc_next[3] = d[3] ^ d[2] ^ d[1] ^ c[1] ^ c[2] ^ c[3];
assign crc_next[4] = d[4] ^ d[3] ^ d[2] ^ c[2] ^ c[3] ^ c[4];
assign crc_next[5] = d[5] ^ d[4] ^ d[3] ^ c[3] ^ c[4] ^ c[5];
assign crc_next[6] = d[6] ^ d[5] ^ d[4] ^ c[4] ^ c[5] ^ c[6];
assign crc_next[7] = d[7] ^ d[6] ^ d[5] ^ c[5] ^ c[6] ^ c[7];

always @(posedge clk or negedge rst_n) begin
    if (!rst_n)
        crc_reg <= 8'h00;
    else if (data_valid)
        crc_reg <= crc_next;
end

assign crc = crc_reg;

endmodule

// CRC-32 for Ethernet (parallel 8-bit)
module crc32_eth (
    input         clk,
    input         rst_n,
    input         init,
    input  [7:0]  data_in,
    input         data_valid,
    output [31:0] crc,
    output [31:0] crc_out  // Inverted, bit-reversed for Ethernet
);

reg [31:0] crc_reg;

// CRC-32 polynomial: 0x04C11DB7
// This is a simplified representation; actual parallel
// equations would be much longer

function [31:0] crc32_byte;
    input [7:0] data;
    input [31:0] crc;
    reg [31:0] c;
    reg [7:0] d;
    begin
        c = crc;
        d = data;
        // Simplified - actual implementation uses
        // full parallel equations or lookup table
        crc32_byte[0] = c[24]^c[30]^d[1]^d[7];
        // ... (30 more equations)
        crc32_byte[31] = c[7]^c[31]^d[0]^d[6];
    end
endfunction

always @(posedge clk or negedge rst_n) begin
    if (!rst_n)
        crc_reg <= 32'hFFFFFFFF;
    else if (init)
        crc_reg <= 32'hFFFFFFFF;
    else if (data_valid)
        crc_reg <= crc32_byte(data_in, crc_reg);
end

assign crc = crc_reg;
// Ethernet: invert and bit-reverse
assign crc_out = ~{crc_reg[0],crc_reg[1],crc_reg[2],...,crc_reg[31]};

endmodule
VHDL crc8.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;

entity crc8_serial is
    port (
        clk        : in  std_logic;
        rst_n      : in  std_logic;
        data_in    : in  std_logic;
        data_valid : in  std_logic;
        crc        : out std_logic_vector(7 downto 0)
    );
end crc8_serial;

architecture behavioral of crc8_serial is
    signal crc_reg : std_logic_vector(7 downto 0);
    signal feedback : std_logic;
begin
    feedback <= data_in xor crc_reg(7);
    
    process(clk, rst_n)
    begin
        if rst_n = '0' then
            crc_reg <= (others => '0');
        elsif rising_edge(clk) then
            if data_valid = '1' then
                crc_reg(7) <= crc_reg(6);
                crc_reg(6) <= crc_reg(5);
                crc_reg(5) <= crc_reg(4);
                crc_reg(4) <= crc_reg(3);
                crc_reg(3) <= crc_reg(2);
                crc_reg(2) <= crc_reg(1) xor feedback;
                crc_reg(1) <= crc_reg(0) xor feedback;
                crc_reg(0) <= feedback;
            end if;
        end if;
    end process;
    
    crc <= crc_reg;
end behavioral;

Applications

Ethernet

Every Ethernet frame includes CRC-32 FCS (Frame Check Sequence).

Detects cable/switch errors
Storage

ZIP, PNG, GZIP files use CRC-32 for integrity verification.

Detects corruption
Serial Protocols

Modbus, CAN bus, USB all use CRC for frame checking.

CRC-16, CRC-5
Flash Memory

NAND flash controllers use CRC alongside ECC for data integrity.

Combined with Hamming/BCH

Interview Notes

  • CRC is polynomial division over GF(2). Remainder after dividing message by generator polynomial.
  • Detects all single, double, odd-bit errors. All burst errors up to polynomial degree.
  • Hardware implementation uses LFSR with XOR feedback at polynomial tap positions.
  • CRC-32 (Ethernet): x³² + x²⁶ + x²³ + ... + 1. Initialized to all 1s, output inverted.
  • Parallel CRC processes multiple bits per clock using derived XOR equations or lookup tables.
  • CRC detects errors but cannot correct. Combine with Hamming/BCH for error correction.
