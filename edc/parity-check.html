Overview

A Parity Checker is a combinational circuit that verifies whether received data (including its parity bit) has been corrupted during transmission or storage. It computes the XOR of all bits—if the result is 0 (for even parity), no error is detected; if 1, an error is flagged.

The checker circuit is nearly identical to the generator—both use XOR reduction. The key difference is interpretation: the generator creates a parity bit; the checker validates it. The checker can also be used as a 9-input odd function generator.

Detection Principle

Even Parity Check

XOR all received bits. If 0, even number of 1s—no error. If 1, error detected.

E = D₀ ⊕ D₁ ⊕ ... ⊕ Dₙ₋₁ ⊕ P
Odd Parity Check

XOR all received bits. If 1, odd number of 1s—no error. If 0, error detected.

E = D₀ ⊕ D₁ ⊕ ... ⊕ Dₙ₋₁ ⊕ P ⊕ 1
Error Signal

Output of checker indicates error status. Often called "syndrome" bit.

E=1: Error detected
Symmetric Operation

Same circuit works for any bit position error—checker is position-agnostic.

XOR is associative

Example: No Error (Even Parity)

Received: 1 0 1 1 0 0 1 0 → XOR = 0 ✓
Check: 1⊕0⊕1⊕1⊕0⊕0⊕1⊕0 = 0 No error

Example: Single-Bit Error Detected

Sent: 1 0 1 1 0 0 1 0
Received: 1 0 0 1 0 0 1 0 ← bit 2 flipped
Check: 1⊕0⊕0⊕1⊕0⊕0⊕1⊕0 = 1 Error detected!

Example: Two-Bit Error (Undetected)

Received: 0 0 0 1 0 0 1 0 ← bits 0,2 flipped
Check: 0⊕0⊕0⊕1⊕0⊕0⊕1⊕0 = 0 No error (false negative!)

Boolean Equations

Even Parity Checker (n-bit data + P)
Error = D₀ ⊕ D₁ ⊕ ... ⊕ Dₙ₋₁ ⊕ P
Odd Parity Checker (n-bit data + P)
Error = D₀ ⊕ D₁ ⊕ ... ⊕ Dₙ₋₁ ⊕ P ⊕ 1
4-bit Data + Parity Check
E = D₃ ⊕ D₂ ⊕ D₁ ⊕ D₀ ⊕ P

Checker = Generator + 1 XOR

Generator: P = D₀ ⊕ D₁ ⊕ ... ⊕ Dₙ₋₁ Checker: E = D₀ ⊕ D₁ ⊕ ... ⊕ Dₙ₋₁ ⊕ P_received If no error: P_received = P_computed E = P_computed ⊕ P_received = 0 (no error) If single-bit error anywhere: E = P_computed ⊕ P_received ≠ 0 → E = 1 (error detected)

RTL Implementation

VERILOG parity_check.v 
// Parameterized Parity Checker
module parity_check #(
    parameter WIDTH = 8,
    parameter ODD   = 0   // 0=even, 1=odd parity expected
)(
    input  [WIDTH-1:0] data,
    input              parity_in,
    output             error
);

// XOR all bits including received parity
assign error = ^data ^ parity_in ^ ODD;

endmodule

// Combined data + parity check (LSB is parity)
module parity_check_combined #(
    parameter WIDTH = 9  // 8 data + 1 parity
)(
    input  [WIDTH-1:0] data_with_parity,
    output             error
);

// XOR reduction of entire word
assign error = ^data_with_parity;

endmodule

// Full system: Generator + Channel + Checker
module parity_system #(
    parameter WIDTH = 8
)(
    input  [WIDTH-1:0] tx_data,
    input  [WIDTH:0]   rx_data,  // Received data + parity
    output [WIDTH:0]   tx_out,   // Transmitted data + parity
    output             rx_error
);

wire tx_parity;

// Transmit side: generate parity
assign tx_parity = ^tx_data;
assign tx_out = {tx_parity, tx_data};

// Receive side: check parity
assign rx_error = ^rx_data;

endmodule

// Memory interface with parity
module memory_with_parity #(
    parameter DATA_WIDTH = 8,
    parameter ADDR_WIDTH = 10
)(
    input                        clk,
    input                        we,
    input  [ADDR_WIDTH-1:0]     addr,
    input  [DATA_WIDTH-1:0]     data_in,
    output [DATA_WIDTH-1:0]     data_out,
    output                       parity_error
);

// Memory stores data + parity
reg [DATA_WIDTH:0] mem [0:(1<<ADDR_WIDTH)-1];
reg [DATA_WIDTH:0] read_data;

// Write with parity generation
always @(posedge clk) begin
    if (we)
        mem[addr] <= {^data_in, data_in};
    read_data <= mem[addr];
end

// Output data and check parity
assign data_out = read_data[DATA_WIDTH-1:0];
assign parity_error = ^read_data;

endmodule
VHDL parity_check.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;

entity parity_check is
    generic (
        WIDTH : natural := 8;
        ODD   : std_logic := '0'
    );
    port (
        data      : in  std_logic_vector(WIDTH-1 downto 0);
        parity_in : in  std_logic;
        error     : out std_logic
    );
end parity_check;

architecture behavioral of parity_check is
begin
    process(data, parity_in)
        variable check : std_logic;
    begin
        check := ODD xor parity_in;
        for i in 0 to WIDTH-1 loop
            check := check xor data(i);
        end loop;
        error <= check;
    end process;
end behavioral;

Limitations

Error Type Detection Correction Notes
Single-bit 100% No Always detected
2-bit 0% No Errors cancel out
3-bit 100% No Odd errors detected
Any odd 100% No Always detected
Any even 0% No Never detected
Burst 50% No Statistical average
No Error Location

Parity only detects presence of error, not which bit is wrong.

Cannot correct
Even Errors Missed

Two errors cancel each other's effect on parity calculation.

Silent data corruption
Solutions

Use Hamming code for SECDED, or CRC for burst error detection.

More redundancy needed
When Sufficient

Low noise channels, memory with low error rates, retransmission possible.

Cost vs. coverage

Applications

UART Communication

Serial frames include parity bit. Receiver checks each byte.

8N1, 8E1, 8O1 modes
Memory Systems

Each word stored with parity. Read operation triggers check.

DRAM, SRAM parity
Bus Protocols

PCI bus uses parity per 32-bit transfer. Target checks received data.

PERR# signal
Magnetic Storage

Historically used in tape drives. Simple, low-overhead checking.

9-track tape

Interview Notes

  • Parity checker is XOR of all bits including parity. Error=1 means odd number of errors detected.
  • Detects all odd-bit errors. Cannot detect even-bit errors—they cancel out. Cannot correct any errors.
  • Checker circuit is same as generator with one extra XOR input for the received parity bit.
  • For error correction, need Hamming code (minimum Hamming distance = 3 for SECDED).
  • Verilog: ^data_with_parity gives 0 for valid even-parity data, 1 for error.
  • 2-D parity (horizontal + vertical) can detect 2-bit errors and correct 1-bit errors.
