Overview

Hamming Code, invented by Richard Hamming in 1950, is a linear error-correcting code that can correct single-bit errors and detect double-bit errors. It works by adding redundant parity bits at specific positions (powers of 2), creating a "syndrome" that identifies the exact error location.

The code is widely used in computer memory (ECC RAM), telecommunications, and any system where data integrity is critical. The most common variant, Hamming(7,4), encodes 4 data bits using 3 parity bits, achieving a total codeword of 7 bits.

Theory

Hamming Distance

Number of bit positions where two codewords differ. Minimum distance d=3 for SEC.

d_min = 3 → corrects 1 error
Parity Bit Positions

Parity bits placed at positions that are powers of 2: 1, 2, 4, 8, 16...

Position = 2⁰, 2¹, 2², ...
Coverage Rule

Each parity bit covers positions where its position bit is set in binary.

P₁: all odd positions
Syndrome

XOR of all parity check results. Non-zero syndrome = error location.

S = {S₃, S₂, S₁}

Redundancy Requirements

Condition 2ᵏ ≥ n + k + 1
4 data bits k = 3 parity bits → Hamming(7,4)
8 data bits k = 4 parity bits → Hamming(12,8)
11 data bits k = 4 parity bits → Hamming(15,11)
Code Rate R = n/(n+k) = data/total

Encoding Process

Hamming(7,4) encoding structure:

Position 7 6 5 4 3 2 1
Binary 111 110 101 100 011 010 001
Content D₄ D₃ D₂ P₃ D₁ P₂ P₁
P₁ covers ✓ - ✓ - ✓ - ✓
P₂ covers ✓ ✓ - - ✓ ✓ -
P₃ covers ✓ ✓ ✓ ✓ - - -
Parity Bit Equations (Even Parity)
P₁ = D₁ ⊕ D₂ ⊕ D₄ (positions 1,3,5,7)
P₂ = D₁ ⊕ D₃ ⊕ D₄ (positions 2,3,6,7)
P₃ = D₂ ⊕ D₃ ⊕ D₄ (positions 4,5,6,7)

Decoding & Correction

Syndrome Calculation
S₁ = P₁ ⊕ D₁ ⊕ D₂ ⊕ D₄
S₂ = P₂ ⊕ D₁ ⊕ D₃ ⊕ D₄
S₃ = P₃ ⊕ D₂ ⊕ D₃ ⊕ D₄

Syndrome Interpretation

S = 000 No error
S = 001 Error at position 1 (P₁)
S = 010 Error at position 2 (P₂)
S = 011 Error at position 3 (D₁)
S = 100 Error at position 4 (P₃)
S = 101 Error at position 5 (D₂)
S = 110 Error at position 6 (D₃)
S = 111 Error at position 7 (D₄)
Correction Process

Convert syndrome to decimal → flip bit at that position.

corrected[S] = received[S] ⊕ 1
Zero Syndrome

S = 0 means no error detected. Data passes through unchanged.

No flip needed

Worked Example

Encoding: Data = 1011

Data bits: D₄=1 D₃=0 D₂=1 D₁=1
P₁ = D₁⊕D₂⊕D₄ = 1⊕1⊕1 = 1
P₂ = D₁⊕D₃⊕D₄ = 1⊕0⊕1 = 0
P₃ = D₂⊕D₃⊕D₄ = 1⊕0⊕1 = 0
Codeword: 1 0 1 0 1 0 1 = 1010101

Decoding with Error at Position 5

Sent: 1 0 1 0 1 0 1
Received: 1 0 0 0 1 0 1 ← position 5 flipped
S₁ = P₁⊕D₁⊕D₂⊕D₄ = 1⊕1⊕0⊕1 = 1
S₂ = P₂⊕D₁⊕D₃⊕D₄ = 0⊕1⊕0⊕1 = 0
S₃ = P₃⊕D₂⊕D₃⊕D₄ = 0⊕0⊕0⊕1 = 1
Syndrome = S₃S₂S₁ = 101 = 5 → Error at position 5!
Corrected: 1 0 1 0 1 0 1 ✓ Original restored

RTL Implementation

VERILOG hamming_74.v 
// Hamming(7,4) Encoder
module hamming_encoder (
    input      [3:0] data,     // D4,D3,D2,D1
    output     [6:0] codeword  // D4,D3,D2,P3,D1,P2,P1
);

wire d4 = data[3];
wire d3 = data[2];
wire d2 = data[1];
wire d1 = data[0];

// Calculate parity bits
wire p1 = d1 ^ d2 ^ d4;  // covers 1,3,5,7
wire p2 = d1 ^ d3 ^ d4;  // covers 2,3,6,7
wire p3 = d2 ^ d3 ^ d4;  // covers 4,5,6,7

// Assemble codeword: positions 7,6,5,4,3,2,1
assign codeword = {d4, d3, d2, p3, d1, p2, p1};

endmodule

// Hamming(7,4) Decoder with SEC
module hamming_decoder (
    input      [6:0] received,
    output     [3:0] data,
    output     [2:0] syndrome,
    output           error_detected
);

// Extract received bits
wire r7 = received[6];  // D4
wire r6 = received[5];  // D3
wire r5 = received[4];  // D2
wire r4 = received[3];  // P3
wire r3 = received[2];  // D1
wire r2 = received[1];  // P2
wire r1 = received[0];  // P1

// Calculate syndrome
wire s1 = r1 ^ r3 ^ r5 ^ r7;  // positions 1,3,5,7
wire s2 = r2 ^ r3 ^ r6 ^ r7;  // positions 2,3,6,7
wire s3 = r4 ^ r5 ^ r6 ^ r7;  // positions 4,5,6,7

assign syndrome = {s3, s2, s1};
assign error_detected = |syndrome;

// Correct the error
wire [6:0] corrected;
genvar i;
generate
    for (i = 0; i < 7; i = i + 1) begin : correct_loop
        assign corrected[i] = received[i] ^ (syndrome == (i + 1));
    end
endgenerate

// Extract data bits from corrected codeword
assign data = {corrected[6], corrected[5], corrected[4], corrected[2]};

endmodule

// Complete Hamming(7,4) SECDED System
module hamming_74_secded (
    input      [3:0] data_in,
    output     [7:0] encoded,   // 7 bits + overall parity
    input      [7:0] received,
    output     [3:0] data_out,
    output           single_error,
    output           double_error
);

// Encoder
wire [6:0] hamming_code;
hamming_encoder enc (.data(data_in), .codeword(hamming_code));

// Add overall parity for DED
wire p0 = ^hamming_code;
assign encoded = {p0, hamming_code};

// Decoder
wire [2:0] syndrome;
wire error_det;
wire [3:0] decoded_data;

hamming_decoder dec (
    .received(received[6:0]),
    .data(decoded_data),
    .syndrome(syndrome),
    .error_detected(error_det)
);

// Check overall parity
wire overall_parity = ^received;

// SECDED logic
assign single_error = error_det && overall_parity;
assign double_error = error_det && !overall_parity;
assign data_out = decoded_data;

endmodule
VHDL hamming_74.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

entity hamming_encoder is
    port (
        data     : in  std_logic_vector(3 downto 0);
        codeword : out std_logic_vector(6 downto 0)
    );
end hamming_encoder;

architecture behavioral of hamming_encoder is
    signal p1, p2, p3 : std_logic;
begin
    -- Calculate parity bits
    p1 <= data(0) xor data(1) xor data(3);
    p2 <= data(0) xor data(2) xor data(3);
    p3 <= data(1) xor data(2) xor data(3);
    
    -- Assemble codeword
    codeword <= data(3) & data(2) & data(1) & p3 & 
                data(0) & p2 & p1;
end behavioral;

entity hamming_decoder is
    port (
        received       : in  std_logic_vector(6 downto 0);
        data           : out std_logic_vector(3 downto 0);
        syndrome       : out std_logic_vector(2 downto 0);
        error_detected : out std_logic
    );
end hamming_decoder;

architecture behavioral of hamming_decoder is
    signal s : std_logic_vector(2 downto 0);
    signal corrected : std_logic_vector(6 downto 0);
begin
    -- Calculate syndrome
    s(0) <= received(0) xor received(2) xor received(4) xor received(6);
    s(1) <= received(1) xor received(2) xor received(5) xor received(6);
    s(2) <= received(3) xor received(4) xor received(5) xor received(6);
    
    syndrome <= s;
    error_detected <= s(0) or s(1) or s(2);
    
    -- Correct error
    correct_gen: for i in 0 to 6 generate
        corrected(i) <= received(i) xor ('1' when to_integer(unsigned(s)) = i+1 else '0');
    end generate;
    
    data <= corrected(6) & corrected(5) & corrected(4) & corrected(2);
end behavioral;

SECDED Extension

Overall Parity (P₀)

Add one extra bit: XOR of entire Hamming codeword.

P₀ = ⊕(all bits)
Single Error

Syndrome ≠ 0 AND overall parity = 1. Correctable.

Flip bit at syndrome position
Double Error

Syndrome ≠ 0 AND overall parity = 0. Detectable, not correctable.

Signal uncorrectable error
No Error

Syndrome = 0 AND overall parity = 0. Data is valid.

Pass through unchanged
Syndrome Overall P Status Action
= 0 = 0 No error Use data as-is
= 0 = 1 P₀ error Ignore (parity-only)
≠ 0 = 1 Single error Correct at syndrome
≠ 0 = 0 Double error Signal UE, discard

Interview Notes

  • Hamming code places parity bits at power-of-2 positions: 1, 2, 4, 8, 16...
  • Each parity bit covers positions whose binary representation has a 1 in that bit position.
  • Syndrome is the concatenation of all parity check results. Non-zero syndrome = error position.
  • Hamming(7,4): 4 data bits, 3 parity bits. Minimum distance d=3 (SEC capability).
  • SECDED adds overall parity. If syndrome≠0 and overall=1: single error. If overall=0: double error.
  • For n data bits, need k parity bits where 2ᵏ ≥ n + k + 1. ECC RAM: 64 data + 8 check bits.
