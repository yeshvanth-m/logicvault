Asynchronous FIFO

The standard solution for transferring data between clock domains. Uses dual-port memory with Gray-coded pointers for safe cross-domain synchronization.

Purpose

CDC Data Transfer

Pointer Encoding

Gray Code

Depth

Power of 2

Latency

2-3 cycles

Architecture

#
Write Side
  • Binary write pointer (wptr)
  • Gray-coded wptr for sync
  • Receives rptr from read domain
  • Generates FULL flag
Read Side
  • Binary read pointer (rptr)
  • Gray-coded rptr for sync
  • Receives wptr from write domain
  • Generates EMPTY flag

Gray Code Pointers

#

Pointers must be synchronized across clock domains. Gray coding ensures only 1 bit changes per increment, making synchronization safe even if sampling occurs during transition.

Address Binary Gray Changed Bits
000000000—
1000100011
2001000111
3001100101
4010001101
5010101111
6011001011
7011101001

Binary ↔ Gray Conversion

gray = binary ^ (binary >> 1)

binary[n] = gray[n] ^ gray[n+1] ^ ... ^ gray[MSB]

FIFO depth must be power of 2! This ensures Gray code wraps correctly. A depth-16 FIFO with 4-bit pointers uses addresses 0-15, and 15→0 (0100→0000) changes only 1 bit.

Full/Empty Detection

#

The tricky part: detecting FULL and EMPTY using Gray-coded pointers that are synchronized (and thus delayed).

EMPTY (read domain)

FIFO is empty when read pointer equals write pointer:

empty = (rptr_gray == wptr_gray_sync)

Safe: synchronized wptr may be old → reports empty when actually has data (conservative)

FULL (write domain)

FIFO is full when write pointer is one loop ahead:

// In Gray code, MSB-1 differs full = (wptr_gray[N:N-1] == ~rptr_sync[N:N-1]) && (wptr_gray[N-2:0] == rptr_sync[N-2:0])

Safe: synchronized rptr may be old → reports full when has space (conservative)

FIFO Sizing

#

FIFO depth depends on the rate mismatch and burst characteristics. The goal: prevent overflow during peak write bursts and underflow during read bursts.

Minimum FIFO Depth (steady state)

Depth ≥ Burst_size × (1 - fread/fwrite)

Scenario Consideration
fwrite > freadFIFO fills; size for max burst duration
fwrite < freadFIFO drains; size for read latency
fwrite ≈ freadSmall FIFO OK, just absorbs jitter
Bursty trafficAdd depth for burst length × duty cycle

Example: 100MHz write clock, 80MHz read clock, 1000-word bursts. Rate mismatch = 1 - 80/100 = 0.2. Min depth = 1000 × 0.2 = 200 words → round up to 256 (power of 2).

RTL (Verilog / VHDL)

#
Verilog async_fifo.v 
// Asynchronous FIFO with Gray-coded pointers
module async_fifo #(
    parameter DATA_WIDTH = 8,
    parameter ADDR_WIDTH = 4   // Depth = 2^4 = 16
)(
    // Write port
    input  wire                  wclk,
    input  wire                  wrst_n,
    input  wire                  wen,
    input  wire [DATA_WIDTH-1:0] wdata,
    output wire                  full,
    
    // Read port
    input  wire                  rclk,
    input  wire                  rrst_n,
    input  wire                  ren,
    output wire [DATA_WIDTH-1:0] rdata,
    output wire                  empty
);

    localparam DEPTH = 1 << ADDR_WIDTH;

    // Memory array
    reg [DATA_WIDTH-1:0] mem [0:DEPTH-1];

    // Pointers (extra bit for full/empty detection)
    reg  [ADDR_WIDTH:0] wptr_bin, wptr_gray;
    reg  [ADDR_WIDTH:0] rptr_bin, rptr_gray;
    
    // Synchronized pointers
    (* ASYNC_REG = "TRUE" *) reg [ADDR_WIDTH:0] wptr_gray_sync [0:1];
    (* ASYNC_REG = "TRUE" *) reg [ADDR_WIDTH:0] rptr_gray_sync [0:1];

    //=========================================
    // WRITE DOMAIN
    //=========================================
    
    wire [ADDR_WIDTH:0] wptr_bin_next = wptr_bin + (wen & ~full);
    wire [ADDR_WIDTH:0] wptr_gray_next = wptr_bin_next ^ (wptr_bin_next >> 1);
    
    always @(posedge wclk or negedge wrst_n) begin
        if (!wrst_n) begin
            wptr_bin  <= '0;
            wptr_gray <= '0;
        end else begin
            wptr_bin  <= wptr_bin_next;
            wptr_gray <= wptr_gray_next;
        end
    end
    
    // Write to memory
    always @(posedge wclk) begin
        if (wen && !full)
            mem[wptr_bin[ADDR_WIDTH-1:0]] <= wdata;
    end
    
    // Sync rptr to write domain
    always @(posedge wclk or negedge wrst_n) begin
        if (!wrst_n) begin
            rptr_gray_sync[0] <= '0;
            rptr_gray_sync[1] <= '0;
        end else begin
            rptr_gray_sync[0] <= rptr_gray;
            rptr_gray_sync[1] <= rptr_gray_sync[0];
        end
    end
    
    // Full detection (compare Gray codes)
    assign full = (wptr_gray_next == {~rptr_gray_sync[1][ADDR_WIDTH:ADDR_WIDTH-1],
                                       rptr_gray_sync[1][ADDR_WIDTH-2:0]});

    //=========================================
    // READ DOMAIN
    //=========================================
    
    wire [ADDR_WIDTH:0] rptr_bin_next = rptr_bin + (ren & ~empty);
    wire [ADDR_WIDTH:0] rptr_gray_next = rptr_bin_next ^ (rptr_bin_next >> 1);
    
    always @(posedge rclk or negedge rrst_n) begin
        if (!rrst_n) begin
            rptr_bin  <= '0;
            rptr_gray <= '0;
        end else begin
            rptr_bin  <= rptr_bin_next;
            rptr_gray <= rptr_gray_next;
        end
    end
    
    // Read from memory
    assign rdata = mem[rptr_bin[ADDR_WIDTH-1:0]];
    
    // Sync wptr to read domain
    always @(posedge rclk or negedge rrst_n) begin
        if (!rrst_n) begin
            wptr_gray_sync[0] <= '0;
            wptr_gray_sync[1] <= '0;
        end else begin
            wptr_gray_sync[0] <= wptr_gray;
            wptr_gray_sync[1] <= wptr_gray_sync[0];
        end
    end
    
    // Empty detection (pointers equal)
    assign empty = (rptr_gray == wptr_gray_sync[1]);

endmodule

Interview Notes

#
  • Why Gray code for pointers? — Only 1 bit changes per increment, so even if sampling occurs during transition, the synchronized value is either the old or new value (both valid), never corrupted.
  • Why power-of-2 depth? — Gray code property (1 bit change) only works when the counter wraps at powers of 2. Other depths cause multi-bit transitions at wrap.
  • Why an extra pointer bit? — To distinguish full from empty. Without it, wptr == rptr could mean either. The MSB wrap-around tells us which.
  • Is FULL/EMPTY pessimistic? — Yes! Due to synchronization delay, EMPTY may be reported when data exists, FULL when space exists. This is safe (prevents overflow/underflow).
  • What's the latency? — 2-3 cycles for synchronized pointers + read latency. Data is available from RAM immediately once read is valid.
  • How do you size a FIFO? — Consider rate mismatch, burst size, and worst-case scenarios. Add margin. Always use power of 2.
  • Can you reset an async FIFO? — Yes, but carefully! Both domains need reset synchronizers. Reset pointers to 0 in both domains.
