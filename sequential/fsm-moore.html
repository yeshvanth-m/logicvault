Sequential FSM

Moore Machine

A finite state machine where outputs depend only on the current state. Outputs are synchronous with the clock and inherently glitch-free, making Moore machines the preferred choice for most digital designs.

Output depends on

State only

Output changes

At clock edge

Glitch-free

Yes (inherently)

Latency

1 cycle delay

Structure

#

Moore Machine Definition

Next State = f(Current State, Input)
Output = g(Current State)   ← No input dependency!

State Diagram Notation

#

In Moore diagrams, outputs are labeled inside states (nodes), not on transitions. Format: State / output

State Table

#

Example: "11" sequence detector (outputs 1 after seeing two consecutive 1s)

Current State Input Next State Output
S0 (idle)0S00
S0 (idle)1S1
S1 (seen 1)0S00
S1 (seen 1)1S2
S2 (seen 11)0S01
S2 (seen 11)1S2

Output depends only on state. S2 always outputs 1, regardless of input. Compare to Mealy which requires fewer states but has output on transition.

Timing Behavior

#

Glitch-free: Since output only changes when state changes (at clock edge), Moore outputs are inherently stable. No glitches from input transitions.

RTL (Verilog / VHDL)

#
Verilog moore_sequence_11.v 
// Moore FSM: Detects "11" sequence
// Output high when in S2 state (after two 1s)
module moore_seq_11 (
    input  wire clk,
    input  wire rst_n,
    input  wire din,
    output wire detected
);

    // State encoding
    localparam [1:0] S0 = 2'b00,  // Idle
                      S1 = 2'b01,  // Seen one 1
                      S2 = 2'b10;  // Seen two 1s (output!)

    reg [1:0] state, next_state;

    // State register
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            state <= S0;
        else
            state <= next_state;
    end

    // Next state logic (combinational)
    always @(*) begin
        case (state)
            S0: next_state = din ? S1 : S0;
            S1: next_state = din ? S2 : S0;
            S2: next_state = din ? S2 : S0;
            default: next_state = S0;
        endcase
    end

    // Moore output: depends ONLY on state
    assign detected = (state == S2);

endmodule

// Alternative: One-hot encoding (faster, more area)
module moore_onehot (
    input  wire clk,
    input  wire rst_n,
    input  wire din,
    output wire detected
);

    // One-hot states
    reg [2:0] state;
    localparam [2:0] S0 = 3'b001,
                      S1 = 3'b010,
                      S2 = 3'b100;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            state <= S0;
        else begin
            state[0] <= (state[0] & ~din) | (state[1] & ~din) | (state[2] & ~din);
            state[1] <= state[0] & din;
            state[2] <= (state[1] & din) | (state[2] & din);
        end
    end

    assign detected = state[2];  // Just check one bit!

endmodule
VHDL moore_sequence_11.vhd 
-- Moore FSM: Detects "11" sequence
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity moore_seq_11 is
    port (
        clk      : in  std_logic;
        rst_n    : in  std_logic;
        din      : in  std_logic;
        detected : out std_logic
    );
end moore_seq_11;

architecture behavioral of moore_seq_11 is
    type state_type is (S0, S1, S2);
    signal state, next_state : state_type;
begin

    -- State register
    process(clk, rst_n)
    begin
        if rst_n = '0' then
            state <= S0;
        elsif rising_edge(clk) then
            state <= next_state;
        end if;
    end process;

    -- Next state logic
    process(state, din)
    begin
        case state is
            when S0 =>
                if din = '1' then
                    next_state <= S1;
                else
                    next_state <= S0;
                end if;

            when S1 =>
                if din = '1' then
                    next_state <= S2;
                else
                    next_state <= S0;
                end if;

            when S2 =>
                if din = '1' then
                    next_state <= S2;
                else
                    next_state <= S0;
                end if;
        end case;
    end process;

    -- Moore output: depends ONLY on state
    detected <= '1' when state = S2 else '0';

end behavioral;

Mealy ↔ Moore Conversion

#

Any Mealy machine can be converted to Moore (and vice versa), but they are not equivalent in terms of states and timing:

Direction Effect
Mealy → Moore May need more states (split states with different outputs on incoming edges). Output delayed by 1 clock.
Moore → Mealy May need fewer states (combine states). Output 1 clock earlier.

Rule of thumb: Use Moore for most designs (simpler timing, glitch-free). Use Mealy when you need faster response or minimum states.
