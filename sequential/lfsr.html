LFSR (Linear Feedback Shift Register)

A shift register whose input is a linear function (XOR) of selected bits. Generates pseudo-random sequences that cycle through 2N-1 states before repeating — essential for CRC, encryption, and test pattern generation.

Feedback

XOR of tap bits

Max sequence

2N - 1 states

Forbidden state

All zeros (stuck)

Key property

Pseudo-random

How It Works

#

An LFSR is a shift register where selected tap positions are XORed together and fed back to the input. The tap positions determine the sequence properties:

  • Maximal-length: Correct taps → cycles through 2N-1 states
  • Non-maximal: Wrong taps → shorter cycles or multiple cycles
  • Polynomial: Taps are defined by characteristic polynomial

Why 2N-1? The all-zeros state is a "stuck" state — XOR of zeros is zero, so it never transitions. A maximal LFSR visits all states except all-zeros.

Fibonacci LFSR (External XOR)

#

The "standard" configuration: XOR gates take taps from multiple positions and feed the result into the first flip-flop. Also called many-to-one or external feedback.

Feedback for polynomial x⁴ + x + 1

D3 = Q0 ⊕ Q2

Galois LFSR (Internal XOR)

#

XOR gates placed between flip-flops at tap positions. Also called one-to-many or internal feedback. Faster because XOR is in parallel with shift.

Fibonacci Pros

Easier to understand, direct polynomial mapping, single XOR chain.

Galois Pros

Faster (XOR in parallel), better for high-speed. Same sequence, different phase.

Maximal-Length Polynomials

#

For maximum sequence length, the feedback polynomial must be primitive. Common maximal-length tap configurations:

Bits Polynomial Taps Sequence Length
3x³ + x + 1[3, 1]7
4x⁴ + x + 1[4, 1]15
7x⁷ + x + 1[7, 1]127
8x⁸ + x⁶ + x⁵ + x⁴ + 1[8, 6, 5, 4]255
16x¹⁶ + x¹⁴ + x¹³ + x¹¹ + 1[16, 14, 13, 11]65,535
32x³² + x²² + x² + x + 1[32, 22, 2, 1]4,294,967,295

CRC polynomials: CRC-16, CRC-32, etc. use standardized polynomials optimized for error detection, not necessarily maximal length.

RTL (Verilog / VHDL)

#
Verilog lfsr.v 
// 8-bit LFSR (Fibonacci) - polynomial x^8 + x^6 + x^5 + x^4 + 1
module lfsr_8bit (
    input  wire clk,
    input  wire rst_n,
    input  wire en,
    output reg [7:0] Q,
    output wire out
);

    wire feedback = Q[7] ^ Q[5] ^ Q[4] ^ Q[3];

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            Q <= 8'b1;  // Non-zero seed
        else if (en)
            Q <= {Q[6:0], feedback};
    end

    assign out = Q[7];

endmodule

// Galois LFSR - often faster
module lfsr_galois_8bit (
    input  wire clk,
    input  wire rst_n,
    input  wire en,
    output reg [7:0] Q
);

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            Q <= 8'b1;
        else if (en) begin
            Q[7] <= Q[0];
            Q[6] <= Q[7];
            Q[5] <= Q[6] ^ Q[0];  // tap
            Q[4] <= Q[5] ^ Q[0];  // tap
            Q[3] <= Q[4] ^ Q[0];  // tap
            Q[2] <= Q[3];
            Q[1] <= Q[2];
            Q[0] <= Q[1];
        end
    end

endmodule

// Parameterized LFSR with common polynomials
module lfsr_param #(
    parameter WIDTH = 16,
    parameter [WIDTH-1:0] TAPS = 16'b1011010000000000  // x^16+x^14+x^13+x^11+1
)(
    input  wire clk,
    input  wire rst_n,
    input  wire en,
    output reg [WIDTH-1:0] Q
);

    wire feedback = ^(Q & TAPS);  // XOR of tapped bits

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            Q <= {{(WIDTH-1){1'b0}}, 1'b1};
        else if (en)
            Q <= {Q[WIDTH-2:0], feedback};
    end

endmodule
VHDL lfsr.vhd 
-- 8-bit LFSR (Fibonacci) - polynomial x^8 + x^6 + x^5 + x^4 + 1
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity lfsr_8bit is
    port (
        clk   : in  std_logic;
        rst_n : in  std_logic;
        en    : in  std_logic;
        Q     : out std_logic_vector(7 downto 0);
        dout  : out std_logic
    );
end lfsr_8bit;

architecture behavioral of lfsr_8bit is
    signal reg_q : std_logic_vector(7 downto 0);
    signal feedback : std_logic;
begin
    -- Taps: x^8 + x^6 + x^5 + x^4 + 1
    feedback <= reg_q(7) xor reg_q(5) xor reg_q(4) xor reg_q(3);

    process(clk, rst_n)
    begin
        if rst_n = '0' then
            reg_q <= "00000001";  -- Non-zero seed
        elsif rising_edge(clk) then
            if en = '1' then
                reg_q <= reg_q(6 downto 0) & feedback;
            end if;
        end if;
    end process;

    Q    <= reg_q;
    dout <= reg_q(7);
end behavioral;

-- 4-bit LFSR example - x^4 + x + 1
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity lfsr_4bit is
    port (
        clk   : in  std_logic;
        rst_n : in  std_logic;
        Q     : out std_logic_vector(3 downto 0)
    );
end lfsr_4bit;

architecture behavioral of lfsr_4bit is
    signal reg_q : std_logic_vector(3 downto 0);
begin
    process(clk, rst_n)
    begin
        if rst_n = '0' then
            reg_q <= "0001";
        elsif rising_edge(clk) then
            reg_q <= reg_q(2 downto 0) & (reg_q(3) xor reg_q(0));
        end if;
    end process;

    Q <= reg_q;
end behavioral;

Applications

#

CRC Computation

Error detection codes (CRC-16, CRC-32) are computed using LFSR-based division.

PRNG

Pseudo-random number generators for games, simulation, and (weak) cryptography.

BIST

Built-In Self Test uses LFSRs to generate test patterns and signature analysis.

Scrambling

Data scrambling in communication (USB, SATA, PCIe) prevents long runs of 0s/1s.

Spread Spectrum

PN sequences for CDMA, GPS, and spread spectrum communications.

Encryption

Stream ciphers (though modern ciphers use more complex structures).

Security warning: LFSRs are NOT cryptographically secure — the sequence can be predicted from a short sample using the Berlekamp-Massey algorithm. Use proper cryptographic PRNGs for security applications.
