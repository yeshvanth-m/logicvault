Reset Design

Proper reset design ensures reliable initialization and recovery. The "async assert, sync release" pattern is industry standard for robust reset handling.

Reset Types

#
Synchronous Reset

Reset sampled on clock edge. Cleaner for timing analysis but requires clock to be running. Can miss reset if clock is stopped.

always @(posedge clk)
  if (!rst_n) q <= 0;
  else q <= d;
Asynchronous Reset

Reset takes effect immediately. Works without clock but can cause metastability on release. Uses dedicated FF reset pins.

always @(posedge clk or negedge rst_n)
  if (!rst_n) q <= 0;
  else q <= d;

Async Assert, Sync Release

#

The best practice combines advantages of both approaches: reset asserts immediately (async) but releases synchronously to prevent metastability.

On Reset Assert (rst_n → 0)

Both FFs immediately preset to 1, causing rst_sync_n to go low. No clock needed — instant reset!

On Reset Release (rst_n → 1)

FFs shift 1→1→output over two clock cycles. Synchronized release prevents metastability in downstream logic.

Why this works: The D inputs are tied to 1, so after reset releases, the FFs will eventually output 1 (rst_sync_n goes high). The 2-FF chain ensures this transition is clean and aligned to the clock.

Reset Distribution

#

In large designs, reset must reach all FFs reliably. A reset tree (like a clock tree) ensures proper distribution with controlled skew.

Caution: Large reset skew can cause issues! If reset releases at different times across blocks, signals between blocks may be undefined during the transition.

Reset Domain Crossing

#

Each clock domain needs its own reset synchronizer. A global async reset should be synchronized separately for each domain.

RTL (Verilog / VHDL)

#
Verilog reset_sync.v 
// Async Assert, Sync Release Reset Synchronizer
module reset_sync (
    input  wire clk,
    input  wire rst_async_n,   // Async reset input (active low)
    output wire rst_sync_n     // Sync reset output (active low)
);

    // Two-stage synchronizer with async preset
    (* ASYNC_REG = "TRUE" *) reg [1:0] rst_sync_reg;

    always @(posedge clk or negedge rst_async_n) begin
        if (!rst_async_n)
            rst_sync_reg <= 2'b00;    // Async assert (immediate)
        else
            rst_sync_reg <= {rst_sync_reg[0], 1'b1};  // Sync release
    end

    assign rst_sync_n = rst_sync_reg[1];

endmodule

// Multi-domain reset controller
module reset_controller (
    input  wire clk_a,
    input  wire clk_b,
    input  wire clk_c,
    input  wire rst_n,         // Global async reset
    output wire rst_a_n,       // Sync reset for domain A
    output wire rst_b_n,       // Sync reset for domain B
    output wire rst_c_n        // Sync reset for domain C
);

    reset_sync u_rst_a (
        .clk         (clk_a),
        .rst_async_n (rst_n),
        .rst_sync_n  (rst_a_n)
    );

    reset_sync u_rst_b (
        .clk         (clk_b),
        .rst_async_n (rst_n),
        .rst_sync_n  (rst_b_n)
    );

    reset_sync u_rst_c (
        .clk         (clk_c),
        .rst_async_n (rst_n),
        .rst_sync_n  (rst_c_n)
    );

endmodule

// Example: Using synchronized reset in a module
module example_ff (
    input  wire clk,
    input  wire rst_sync_n,    // From reset_sync
    input  wire d,
    output reg  q
);

    // Still use async reset in sensitivity for synthesis
    // The reset_sync ensures clean release
    always @(posedge clk or negedge rst_sync_n) begin
        if (!rst_sync_n)
            q <= 1'b0;
        else
            q <= d;
    end

endmodule
VHDL reset_sync.vhd 
-- Async Assert, Sync Release Reset Synchronizer
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity reset_sync is
    port (
        clk         : in  std_logic;
        rst_async_n : in  std_logic;
        rst_sync_n  : out std_logic
    );
end reset_sync;

architecture rtl of reset_sync is
    signal rst_sync_reg : std_logic_vector(1 downto 0);
    
    attribute ASYNC_REG : string;
    attribute ASYNC_REG of rst_sync_reg : signal is "TRUE";
begin

    process(clk, rst_async_n)
    begin
        if rst_async_n = '0' then
            rst_sync_reg <= "00";  -- Async assert
        elsif rising_edge(clk) then
            rst_sync_reg <= rst_sync_reg(0) & '1';  -- Sync release
        end if;
    end process;

    rst_sync_n <= rst_sync_reg(1);

end rtl;

Interview Notes

#
  • Why async assert, sync release? — Async assert works without clock (can reset when clock is stopped). Sync release prevents metastability in downstream logic.
  • What happens with pure async reset? — If reset releases near a clock edge, FFs may go metastable. Different FFs may capture different values.
  • Why tie D input to 1 in the synchronizer? — Ensures reset eventually releases (FFs shift 1→1→out). Active-low reset means output=1 is "not reset".
  • What about multiple clock domains? — Each domain needs its own reset synchronizer clocked by that domain's clock.
  • Active high vs active low reset? — Both work. Active low (rst_n) is traditional because it uses preset pins. Just be consistent!
  • What's reset recovery time? — Time for reset release to propagate through the synchronizer (2+ clock cycles for 2-FF sync).
  • How to handle reset glitches? — Add a glitch filter (RC circuit) at the board level. The sync FFs provide some filtering too.
