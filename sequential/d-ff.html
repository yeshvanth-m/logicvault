D Flip-Flop

The D flip-flop (DFF) is the fundamental building block of synchronous digital design. It samples the D input only at the clock edge and holds the value until the next edge — enabling predictable, clock-synchronized circuits.

Inputs

D, CLK

Outputs

Q, Q'

Trigger

Rising or Falling edge

Circuit type

Edge-triggered

Memory

1 bit

Gate count

~6-12 (varies)

How It Works

#

Unlike the D latch (which is transparent), the D flip-flop samples D only at the clock edge. The output Q takes the value of D at that instant and holds it stable until the next clock edge — regardless of how D changes.

At Rising Edge (↑)

Q ← D

Q captures value of D at that instant

Between Edges

Q holds

Q ignores D changes — stable output

Edge-triggered = no transparency: This solves the race condition problems of latches. The DFF is the workhorse of all synchronous digital design.

Master-Slave Architecture

#

A common implementation uses two D latches in series with inverted clock signals. This creates true edge-triggered behavior:

CLK = 0

Master transparent, Slave latched

D → Qm (internal), Q holds previous

CLK = 1

Master latched, Slave transparent

Qm → Q, D blocked from changing Qm

Rising Edge (0→1)

D captured to Q

Master closes, slave opens — value transfers

Timing Parameters

#

Critical timing parameters that define flip-flop behavior. Violating these causes metastability — unpredictable output.

Setup Time (tsu)

D stable BEFORE edge

D must be stable this long before clock edge

Hold Time (th)

D stable AFTER edge

D must remain stable this long after clock edge

Clock-to-Q (tcq)

Edge → Q valid

Propagation delay from clock edge to stable output

Metastability: If setup/hold times are violated, the flip-flop may enter a metastable state where Q is neither 0 nor 1 for an unpredictable duration. This is the root cause of many clock domain crossing bugs.

Async Reset/Preset

#

Practical flip-flops include asynchronous control signals that override normal operation regardless of the clock:

Async Reset (CLR)

Q → 0 immediately

Forces Q=0, ignores clock. Used for power-on initialization.

Async Preset (PRE)

Q → 1 immediately

Forces Q=1, ignores clock. Less common than reset.

Priority

Reset > Preset > Clock

Async signals have highest priority

RTL (Verilog / VHDL)

#
Verilog dff.v 
// D Flip-Flop - Positive Edge Triggered
module dff (
    input  wire clk,
    input  wire D,
    output reg  Q
);

    always @(posedge clk) begin
        Q <= D;
    end

endmodule

// D Flip-Flop with Synchronous Reset
module dff_sync_rst (
    input  wire clk,
    input  wire rst_n,   // Active-low sync reset
    input  wire D,
    output reg  Q
);

    always @(posedge clk) begin
        if (!rst_n)
            Q <= 1'b0;
        else
            Q <= D;
    end

endmodule
VHDL dff.vhd 
-- D Flip-Flop - Positive Edge Triggered
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity dff is
    port (
        clk : in  std_logic;
        D   : in  std_logic;
        Q   : out std_logic
    );
end dff;

architecture behavioral of dff is
begin
    process(clk)
    begin
        if rising_edge(clk) then
            Q <= D;
        end if;
    end process;
end behavioral;
Verilog dff_async.v 
// D Flip-Flop with Asynchronous Reset
module dff_async_rst (
    input  wire clk,
    input  wire rst_n,   // Active-low ASYNC reset
    input  wire D,
    output reg  Q
);

    // Note: rst_n in sensitivity list = asynchronous
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            Q <= 1'b0;    // Async reset - immediate
        else
            Q <= D;       // Normal operation
    end

endmodule

// D Flip-Flop with Async Reset AND Preset
module dff_async_rst_pre (
    input  wire clk,
    input  wire rst_n,
    input  wire pre_n,   // Active-low preset
    input  wire D,
    output reg  Q
);

    always @(posedge clk or negedge rst_n or negedge pre_n) begin
        if (!rst_n)
            Q <= 1'b0;    // Reset has priority
        else if (!pre_n)
            Q <= 1'b1;    // Preset
        else
            Q <= D;
    end

endmodule

Applications

#

Registers

N flip-flops in parallel form an N-bit register — the basis of all CPU registers, pipeline stages, etc.

Counters

Chained flip-flops with feedback create binary counters. Used in timers, frequency dividers, address generators.

Shift Registers

Cascaded DFFs where Q of one feeds D of next. Used in serial-parallel conversion, LFSR, delay lines.

State Machines

DFFs hold FSM state. Combinational logic computes next state from current state + inputs.

Synchronizers

Two DFFs in series synchronize signals crossing clock domains, reducing metastability risk.

Pipeline Stages

DFFs between combinational stages enable pipelining — the key to high-performance processor design.
