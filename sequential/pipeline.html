Sequential Performance

Pipeline

Technique to increase throughput by dividing a task into stages that execute in parallel. Each stage processes different data simultaneously.

Goal

↑ Throughput

Trade-off

↑ Latency per item

Key Metric

Items/cycle

Use Case

CPU, DSP, Data Path

Pipeline Basics

#
Without Pipeline

One operation at a time. Total delay = sum of all stages. Low throughput, lower clock frequency.

With Pipeline

Multiple operations in flight. Clock period = slowest stage + register overhead. Higher throughput.

Performance Analysis

#
Metric Formula Notes
LatencyN × TclkN = number of stages
Throughput1 / TclkItems per second (steady state)
TclkTstage + TregSlowest stage + register overhead
Speedup≈ NIdeal case (balanced stages)

Diminishing Returns: Adding more stages increases register overhead and complicates hazard handling. Optimal depth depends on technology and design.

Pipeline Hazards

#
Structural Hazard

Two stages need same resource.

Solution: Duplicate resources or stall pipeline.

Memory: IF and MEM both need memory access
Data Hazard

Instruction depends on result of previous.

Solution: Forwarding/bypass, stall, or reorder.

ADD R1, R2, R3 SUB R4, R1, R5 // needs R1
Control Hazard

Branch changes program flow.

Solution: Predict, speculate, or flush and stall.

BEQ R1, R2, target // which instr next?

Valid/Ready Flow Control

#

Standard handshake protocol for pipeline stages. Handles backpressure when downstream is not ready.

Producer (Upstream)

Asserts valid when data is available. Holds data stable until ready is seen.

Consumer (Downstream)

Asserts ready when it can accept data. Transfer occurs when both signals high.

RTL (Verilog / VHDL)

#
Verilog pipeline.v 
// Simple pipeline register (always flows)
module pipe_reg #(
    parameter WIDTH = 32
)(
    input  wire             clk,
    input  wire             rst_n,
    input  wire [WIDTH-1:0] d,
    output reg  [WIDTH-1:0] q
);

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            q <= {WIDTH{1'b0}};
        else
            q <= d;
    end

endmodule

// Pipeline stage with valid/ready handshake
module pipe_stage #(
    parameter WIDTH = 32
)(
    input  wire             clk,
    input  wire             rst_n,
    
    // Upstream interface
    input  wire             in_valid,
    output wire             in_ready,
    input  wire [WIDTH-1:0] in_data,
    
    // Downstream interface
    output reg              out_valid,
    input  wire             out_ready,
    output reg  [WIDTH-1:0] out_data
);

    // We're ready if downstream is ready or we have no data
    assign in_ready = out_ready | ~out_valid;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            out_valid <= 1'b0;
            out_data  <= {WIDTH{1'b0}};
        end else begin
            if (in_ready) begin
                out_valid <= in_valid;
                out_data  <= in_data;
            end
        end
    end

endmodule

// Skid buffer (full throughput with backpressure)
module skid_buffer #(
    parameter WIDTH = 32
)(
    input  wire             clk,
    input  wire             rst_n,
    
    input  wire             in_valid,
    output reg              in_ready,
    input  wire [WIDTH-1:0] in_data,
    
    output reg              out_valid,
    input  wire             out_ready,
    output reg  [WIDTH-1:0] out_data
);

    reg [WIDTH-1:0] buf_data;
    reg            buf_valid;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            out_valid <= 1'b0;
            out_data  <= 0;
            buf_valid <= 1'b0;
            buf_data  <= 0;
            in_ready  <= 1'b1;
        end else begin
            // Output accepted?
            if (out_ready && out_valid) begin
                if (buf_valid) begin
                    // Move buffer to output
                    out_data  <= buf_data;
                    buf_valid <= 1'b0;
                    in_ready  <= 1'b1;
                end else if (in_valid && in_ready) begin
                    // Pass through
                    out_data <= in_data;
                end else begin
                    out_valid <= 1'b0;
                end
            end else if (!out_valid && in_valid && in_ready) begin
                // Fill empty output
                out_valid <= 1'b1;
                out_data  <= in_data;
            end else if (out_valid && !out_ready && in_valid && in_ready) begin
                // Store in buffer, stop accepting
                buf_valid <= 1'b1;
                buf_data  <= in_data;
                in_ready  <= 1'b0;
            end
        end
    end

endmodule
VHDL pipeline.vhd 
-- Pipeline stage with valid/ready
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity pipe_stage is
    generic (WIDTH : integer := 32);
    port (
        clk       : in  std_logic;
        rst_n     : in  std_logic;
        -- Upstream
        in_valid  : in  std_logic;
        in_ready  : out std_logic;
        in_data   : in  std_logic_vector(WIDTH-1 downto 0);
        -- Downstream
        out_valid : out std_logic;
        out_ready : in  std_logic;
        out_data  : out std_logic_vector(WIDTH-1 downto 0)
    );
end pipe_stage;

architecture rtl of pipe_stage is
    signal valid_r : std_logic;
    signal data_r  : std_logic_vector(WIDTH-1 downto 0);
    signal ready_i : std_logic;
begin

    ready_i <= out_ready or (not valid_r);
    in_ready <= ready_i;
    out_valid <= valid_r;
    out_data <= data_r;

    process(clk, rst_n)
    begin
        if rst_n = '0' then
            valid_r <= '0';
            data_r <= (others => '0');
        elsif rising_edge(clk) then
            if ready_i = '1' then
                valid_r <= in_valid;
                data_r <= in_data;
            end if;
        end if;
    end process;

end rtl;

Interview Notes

#
  • Latency vs throughput trade-off? — Pipeline increases latency (more stages) but dramatically increases throughput (parallel execution).
  • What limits pipeline depth? — Register overhead, power, hazard handling complexity, and diminishing frequency gains.
  • What is a bubble? — Empty pipeline slot (no valid data) caused by stall or flush. Wastes throughput.
  • Forwarding vs stalling? — Forwarding bypasses data from later stage to earlier. Avoids stall but adds mux in critical path.
  • What's a skid buffer? — Extra buffer stage that maintains throughput during backpressure. Stores one item when downstream stalls.
  • Pipeline retiming? — Synthesis technique to move registers across combinational logic to balance stage delays.
