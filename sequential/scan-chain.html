Sequential DFT

Scan Chain / DFT

Design for Testability — converting flip-flops to scan flip-flops to enable controllability and observability of internal state during manufacturing test.

Purpose

Manufacturing Test

Coverage

Stuck-at Faults

Overhead

~5-15% Area

Mode

scan_en=1

Why Scan?

#

Without DFT, testing sequential circuits is extremely difficult — reaching specific states requires many clock cycles. Scan converts the design into a shift register for test, enabling direct state control and observation.

Controllability

Ability to set internal flip-flops to desired states. Scan lets you shift in any pattern via scan_in.

Observability

Ability to observe internal state at outputs. Scan lets you shift out all flip-flop values via scan_out.

Fault Coverage

Percentage of testable faults actually detected. Good scan design achieves >95% stuck-at coverage.

Test Patterns

ATPG (Automatic Test Pattern Generation) creates patterns to detect faults. Fewer patterns = faster test.

Fault Models

#

Scan testing detects manufacturing defects by modeling them as faults. Different fault models target different physical defect types.

Fault ModelDescriptionDetectsPattern Count
Stuck-at (SA) Node permanently at 0 (SA0) or 1 (SA1) Opens, shorts to VDD/GND Baseline
Transition (TF) Node can't transition 0→1 (slow-to-rise) or 1→0 (slow-to-fall) Resistive defects, weak transistors ~2× SA
Path Delay Cumulative delay on critical paths exceeds spec Speed-limiting defects Variable
Bridging Short between two nodes (AND or OR bridge) Metal shorts ~10× SA
IDDQ Elevated quiescent current indicates defect Gate oxide defects, stuck-on transistors Few patterns
Stuck-at-0 (SA0)

To detect SA0 at node X: drive X=1 and propagate to output. If stuck, output stays wrong.

Stuck-at-1 (SA1)

To detect SA1 at node X: drive X=0 and propagate to output. If stuck, output stays wrong.

Transition Faults

Require at-speed testing: launch edge → capture edge within one clock cycle. Detects timing defects SA misses.

Fault Coverage Target

Production: >95% stuck-at, >90% transition. Below this, defective parts escape to customers.

Scan Test Flow

#

Each test pattern involves three phases: Shift In, Capture, and Shift Out. For transition faults, two capture cycles provide the launch-capture sequence.

// Basic Scan Test Sequence (Stuck-at Testing)

  ┌─────────────────────────────────────────────────────────────┐
  │  Phase 1: SHIFT IN                                          │
  │  scan_en = 1                                                │
  │  Clock N times (N = chain length)                           │
  │  Pattern shifts in via scan_in                              │
  └─────────────────────────────────────────────────────────────┘
                              ↓
  ┌─────────────────────────────────────────────────────────────┐
  │  Phase 2: CAPTURE                                           │
  │  scan_en = 0                                                │
  │  One functional clock cycle                                 │
  │  Combinational logic evaluates, results captured in FFs    │
  └─────────────────────────────────────────────────────────────┘
                              ↓
  ┌─────────────────────────────────────────────────────────────┐
  │  Phase 3: SHIFT OUT                                         │
  │  scan_en = 1                                                │
  │  Clock N times                                              │
  │  Compare scan_out with expected values (on ATE)             │
  │  (Simultaneously shift in next pattern)                     │
  └─────────────────────────────────────────────────────────────┘


// Transition Fault Testing (At-Speed)
// Requires two capture edges: LAUNCH and CAPTURE

Method 1: Launch-Off-Shift (LOS)
  - Last shift clock = launch edge
  - First capture clock = capture edge
  - Timing: shift_clk → (1 cycle) → capture_clk

Method 2: Launch-Off-Capture (LOC) / Broadside
  - scan_en goes low
  - First capture clock = launch edge  
  - Second capture clock = capture edge
  - More coverage, but needs 2 capture cycles

// Timing Diagram (LOS):
           ___     ___     ___     ___     ___
  clk   __|   |___|   |___|   |___|   |___|   |__
          shift   shift   LAUNCH  CAPTURE
                           ↑         ↑
        __________________|         |
  scan_en                  |_________|
                          
        Pattern in FFs → Transition → Capture response

ATPG (Automatic Test Pattern Generation)

#

ATPG tools automatically create test patterns that detect modeled faults. The D-algorithm and PODEM are classic algorithms; modern tools use advanced techniques for compression and coverage.

ToolVendorFeatures
TetraMAXSynopsysIndustry standard, compression, diagnosis
ModusCadenceHierarchical ATPG, low-power patterns
TessentSiemens EDACell-aware ATPG, IJTAG integration
 
// ATPG Fundamentals

1. FAULT ACTIVATION
   - Set input values to create opposite logic at fault site
   - SA0 fault: must drive node to logic 1
   - SA1 fault: must drive node to logic 0

2. FAULT PROPAGATION  
   - Sensitize path from fault site to observable output
   - Non-controlling values on side inputs
   - D or D' (fault effect) propagates through gates

3. JUSTIFICATION
   - Work backward to find primary input values
   - That produce required internal values

// Example: Detect SA0 fault on wire W

     A ──┐                
         │ AND ── W ── ...─── output
     B ──┘       ↑
              SA0 here

  To detect: A=1, B=1 → W should be 1
             If SA0, W stays 0 → propagates as error

// ATPG Report Metrics:
  - Test Coverage: (detected faults / total faults) × 100%
  - Pattern Count: number of test vectors generated
  - ATPG Untestables (AU): faults that cannot be tested
    - Tied signals, redundant logic, blocked paths
  - Fault classes: DS (detected), PT (possibly tested),
                   UD (undetected), AU (ATPG untestable)

Scan Compression

#

Modern SoCs have millions of scan flip-flops. Without compression, test data volume and time are prohibitive. Compression reduces both by 10-100×.

Decompressor

Expands few external scan inputs to many internal chains. LFSR-based or combinational XOR network.

Compressor

XOR tree compresses many chain outputs to few pins. Must handle unknown (X) values gracefully.

X-Tolerance

X-masking or X-blocking prevents unknowns from corrupting signature. Critical for real silicon.

Benefits

10-100× less test data, 10-100× faster test time, same or better fault coverage.

💡 Commercial Solutions: Synopsys DFTMAX, Cadence Modus, Siemens Tessent — all provide compression with X-tolerance and diagnosis support.

Scan Insertion Flow

#

Standard cells include scan variants (SDFF). EDA tools replace normal FFs with scan FFs, then chain them together. Typical constraints:

Step Action Tool
1Replace DFF → Scan DFFSynthesis
2Chain scan FFs togetherDFT Compiler
3Add scan controllerDFT Compiler
4Generate ATPG patternsTetraMAX/Modus
5Run fault simulationTetraMAX/Modus
Scan Chain Length

Long chains = fewer pins, longer test time. Short chains = more pins, faster test. Balance per project needs.

Clock Domains

Each clock domain typically has separate scan chains. Cross-domain FFs need special handling.

Scan Design Rules

#

Violating DFT rules reduces fault coverage or causes test failures. EDA tools flag violations; fix them before tapeout.

RuleViolationImpactFix
No async set/reset during scan Async signals active in test mode FFs reset during shift, data lost Gate async signals with test_mode
Single clock per chain Multi-clock FFs in same chain Shift corruption, hold violations Separate chains per clock domain
No combinational loops Feedback without FF Oscillation, simulation hang Break loop with FF
No gated clocks in scan Clock gating active during shift Some FFs don't shift Bypass clock gates in test mode
No tristate on scan path Tristate buffer in scan chain Bus contention or floating Use MUX instead of tristate
Balanced chain lengths Chains differ by >10% Inefficient test time Re-stitch chains to balance 
// Common Violation: Async Reset Active During Scan

// BAD: Reset clears FFs during shift!
always @(posedge clk or negedge rst_n)
    if (!rst_n) q <= 0;  // ← Fires during scan if rst_n=0
    else q <= d;

// GOOD: Gate reset with test_mode
wire rst_gated = rst_n | test_mode;  // Disable reset in test
always @(posedge clk or negedge rst_gated)
    if (!rst_gated) q <= 0;
    else q <= d;


// Common Violation: Clock Gating Active in Test

// BAD: Gated clock stops shifting!
assign gclk = clk & enable;  // ← If enable=0, chain breaks

// GOOD: Bypass in test mode
assign gclk = clk & (enable | scan_en);  // Force clock during scan

// OR use ICG (Integrated Clock Gating) cell:
ICG u_icg (
    .clk(clk),
    .en(enable),
    .te(scan_en),    // Test enable bypasses gate
    .gclk(gclk)
);

RTL (Verilog / VHDL)

#
Verilog scan_ff.v 
// Scan flip-flop (behavioral model)
module scan_dff (
    input  wire clk,
    input  wire rst_n,
    input  wire d,        // Normal data input
    input  wire si,       // Scan input
    input  wire scan_en,  // Scan enable
    output reg  q         // Output (also scan_out)
);

    wire mux_out;
    
    // 2:1 MUX: scan_en selects between D and SI
    assign mux_out = scan_en ? si : d;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            q <= 1'b0;
        else
            q <= mux_out;
    end

endmodule

// Example: 4-bit register with scan chain
module scan_reg4 (
    input  wire       clk,
    input  wire       rst_n,
    input  wire [3:0] d,
    input  wire       scan_en,
    input  wire       scan_in,
    output wire       scan_out,
    output wire [3:0] q
);

    wire [3:0] si_chain;
    
    // Chain: scan_in → FF0 → FF1 → FF2 → FF3 → scan_out
    assign si_chain = {q[2:0], scan_in};

    scan_dff u_ff0 (.clk(clk), .rst_n(rst_n), .d(d[0]), 
                    .si(scan_in),   .scan_en(scan_en), .q(q[0]));
    scan_dff u_ff1 (.clk(clk), .rst_n(rst_n), .d(d[1]), 
                    .si(q[0]),      .scan_en(scan_en), .q(q[1]));
    scan_dff u_ff2 (.clk(clk), .rst_n(rst_n), .d(d[2]), 
                    .si(q[1]),      .scan_en(scan_en), .q(q[2]));
    scan_dff u_ff3 (.clk(clk), .rst_n(rst_n), .d(d[3]), 
                    .si(q[2]),      .scan_en(scan_en), .q(q[3]));

    assign scan_out = q[3];

endmodule

// Simple scan controller
module scan_ctrl #(
    parameter CHAIN_LEN = 32  // Number of FFs in chain
)(
    input  wire       clk,
    input  wire       rst_n,
    input  wire       test_mode,    // Test mode enable
    input  wire       shift,        // Shift command
    input  wire       capture,      // Capture command
    output reg        scan_en,      // To all scan FFs
    output reg  [5:0] shift_cnt     // Shift counter
);

    localparam IDLE    = 2'b00;
    localparam SHIFT   = 2'b01;
    localparam CAPTURE = 2'b10;
    
    reg [1:0] state;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            state     <= IDLE;
            scan_en   <= 1'b0;
            shift_cnt <= 0;
        end else if (test_mode) begin
            case (state)
                IDLE: begin
                    if (shift) begin
                        state     <= SHIFT;
                        scan_en   <= 1'b1;
                        shift_cnt <= CHAIN_LEN;
                    end
                end
                
                SHIFT: begin
                    shift_cnt <= shift_cnt - 1;
                    if (shift_cnt == 1) begin
                        state   <= capture ? CAPTURE : IDLE;
                        scan_en <= 1'b0;
                    end
                end
                
                CAPTURE: begin
                    // One cycle capture, then back to shift out
                    state     <= SHIFT;
                    scan_en   <= 1'b1;
                    shift_cnt <= CHAIN_LEN;
                end
            endcase
        end else begin
            scan_en <= 1'b0;
            state   <= IDLE;
        end
    end

endmodule
VHDL scan_ff.vhd 
-- Scan flip-flop
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity scan_dff is
    port (
        clk     : in  std_logic;
        rst_n   : in  std_logic;
        d       : in  std_logic;  -- Normal data
        si      : in  std_logic;  -- Scan input
        scan_en : in  std_logic;  -- Scan enable
        q       : out std_logic   -- Output
    );
end scan_dff;

architecture rtl of scan_dff is
    signal mux_out : std_logic;
begin

    mux_out <= si when scan_en = '1' else d;

    process(clk, rst_n)
    begin
        if rst_n = '0' then
            q <= '0';
        elsif rising_edge(clk) then
            q <= mux_out;
        end if;
    end process;

end rtl;

-- 4-bit register with scan chain
entity scan_reg4 is
    port (
        clk      : in  std_logic;
        rst_n    : in  std_logic;
        d        : in  std_logic_vector(3 downto 0);
        scan_en  : in  std_logic;
        scan_in  : in  std_logic;
        scan_out : out std_logic;
        q        : out std_logic_vector(3 downto 0)
    );
end scan_reg4;

architecture rtl of scan_reg4 is
    signal q_int : std_logic_vector(3 downto 0);
begin

    u_ff0: entity work.scan_dff
        port map (clk, rst_n, d(0), scan_in, scan_en, q_int(0));
    u_ff1: entity work.scan_dff
        port map (clk, rst_n, d(1), q_int(0), scan_en, q_int(1));
    u_ff2: entity work.scan_dff
        port map (clk, rst_n, d(2), q_int(1), scan_en, q_int(2));
    u_ff3: entity work.scan_dff
        port map (clk, rst_n, d(3), q_int(2), scan_en, q_int(3));

    q <= q_int;
    scan_out <= q_int(3);

end rtl;

Interview Notes

#
Q: Why is scan needed?

Sequential circuits hard to test without DFT. Scan provides controllability (shift in any state) and observability (shift out all states).

Q: Stuck-at vs transition faults?

Stuck-at: node stuck at 0/1 (static). Transition: can't switch fast enough (timing). Transition tests catch defects SA misses.

Q: LOS vs LOC testing?

LOS: launch from last shift, faster but less coverage. LOC: launch from capture, extra cycle but better transition coverage.

Q: What is ATPG?

Auto Test Pattern Generation. Creates patterns to detect faults. Goal: maximize coverage, minimize pattern count.

Q: Scan compression ratio?

Ratio of internal chains to external pins. 64 chains / 4 pins = 16× compression. Reduces test time proportionally.

Q: X-tolerance in compression?

Unknowns (X) can corrupt XOR compressor output. X-masking, X-blocking, or X-bounding prevents this.

Q: Fault coverage targets?

Stuck-at: >95%. Transition: >90%. Lower means defective chips escape. Balance coverage vs test cost.

Q: Why gate resets in test mode?

Async resets can fire during scan shift, clearing data. Gate with test_mode signal to prevent.

Q: Scan chain length trade-off?

Long chains: fewer pins, slower test. Short chains: more pins, faster. Balance based on pin budget and test time.

Q: ATPG untestable faults?

Tied signals, redundant logic, blocked paths. Review for possible design issues or accept if truly unreachable.

JTAG

BIST

Built-In Self-Test. On-chip pattern generation (LFSR) and response compression (MISR). No ATE needed.

LFSR

Linear Feedback Shift Register. Used for pseudo-random pattern generation in BIST and compression.
