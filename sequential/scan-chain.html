Sequential DFT

Scan Chain / DFT

Design for Testability — converting flip-flops to scan flip-flops to enable controllability and observability of internal state during manufacturing test.

Purpose

Manufacturing Test

Coverage

Stuck-at Faults

Overhead

~5-15% Area

Mode

scan_en=1

Why Scan?

#

Without DFT, testing sequential circuits is extremely difficult — reaching specific states requires many clock cycles. Scan converts the design into a shift register for test, enabling direct state control and observation.

Controllability

Ability to set internal flip-flops to desired states. Scan lets you shift in any pattern via scan_in.

Observability

Ability to observe internal state at outputs. Scan lets you shift out all flip-flop values via scan_out.

Fault Coverage

Percentage of testable faults actually detected. Good scan design achieves >95% stuck-at coverage.

Test Patterns

ATPG (Automatic Test Pattern Generation) creates patterns to detect faults. Fewer patterns = faster test.

Scan Insertion Flow

#

Standard cells include scan variants (SDFF). EDA tools replace normal FFs with scan FFs, then chain them together. Typical constraints:

Step Action Tool
1Replace DFF → Scan DFFSynthesis
2Chain scan FFs togetherDFT Compiler
3Add scan controllerDFT Compiler
4Generate ATPG patternsTetraMAX/Modus
5Run fault simulationTetraMAX/Modus
Scan Chain Length

Long chains = fewer pins, longer test time. Short chains = more pins, faster test. Balance per project needs.

Clock Domains

Each clock domain typically has separate scan chains. Cross-domain FFs need special handling.

RTL (Verilog / VHDL)

#
Verilog scan_ff.v 
// Scan flip-flop (behavioral model)
module scan_dff (
    input  wire clk,
    input  wire rst_n,
    input  wire d,        // Normal data input
    input  wire si,       // Scan input
    input  wire scan_en,  // Scan enable
    output reg  q         // Output (also scan_out)
);

    wire mux_out;
    
    // 2:1 MUX: scan_en selects between D and SI
    assign mux_out = scan_en ? si : d;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            q <= 1'b0;
        else
            q <= mux_out;
    end

endmodule

// Example: 4-bit register with scan chain
module scan_reg4 (
    input  wire       clk,
    input  wire       rst_n,
    input  wire [3:0] d,
    input  wire       scan_en,
    input  wire       scan_in,
    output wire       scan_out,
    output wire [3:0] q
);

    wire [3:0] si_chain;
    
    // Chain: scan_in → FF0 → FF1 → FF2 → FF3 → scan_out
    assign si_chain = {q[2:0], scan_in};

    scan_dff u_ff0 (.clk(clk), .rst_n(rst_n), .d(d[0]), 
                    .si(scan_in),   .scan_en(scan_en), .q(q[0]));
    scan_dff u_ff1 (.clk(clk), .rst_n(rst_n), .d(d[1]), 
                    .si(q[0]),      .scan_en(scan_en), .q(q[1]));
    scan_dff u_ff2 (.clk(clk), .rst_n(rst_n), .d(d[2]), 
                    .si(q[1]),      .scan_en(scan_en), .q(q[2]));
    scan_dff u_ff3 (.clk(clk), .rst_n(rst_n), .d(d[3]), 
                    .si(q[2]),      .scan_en(scan_en), .q(q[3]));

    assign scan_out = q[3];

endmodule

// Simple scan controller
module scan_ctrl #(
    parameter CHAIN_LEN = 32  // Number of FFs in chain
)(
    input  wire       clk,
    input  wire       rst_n,
    input  wire       test_mode,    // Test mode enable
    input  wire       shift,        // Shift command
    input  wire       capture,      // Capture command
    output reg        scan_en,      // To all scan FFs
    output reg  [5:0] shift_cnt     // Shift counter
);

    localparam IDLE    = 2'b00;
    localparam SHIFT   = 2'b01;
    localparam CAPTURE = 2'b10;
    
    reg [1:0] state;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            state     <= IDLE;
            scan_en   <= 1'b0;
            shift_cnt <= 0;
        end else if (test_mode) begin
            case (state)
                IDLE: begin
                    if (shift) begin
                        state     <= SHIFT;
                        scan_en   <= 1'b1;
                        shift_cnt <= CHAIN_LEN;
                    end
                end
                
                SHIFT: begin
                    shift_cnt <= shift_cnt - 1;
                    if (shift_cnt == 1) begin
                        state   <= capture ? CAPTURE : IDLE;
                        scan_en <= 1'b0;
                    end
                end
                
                CAPTURE: begin
                    // One cycle capture, then back to shift out
                    state     <= SHIFT;
                    scan_en   <= 1'b1;
                    shift_cnt <= CHAIN_LEN;
                end
            endcase
        end else begin
            scan_en <= 1'b0;
            state   <= IDLE;
        end
    end

endmodule
VHDL scan_ff.vhd 
-- Scan flip-flop
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity scan_dff is
    port (
        clk     : in  std_logic;
        rst_n   : in  std_logic;
        d       : in  std_logic;  -- Normal data
        si      : in  std_logic;  -- Scan input
        scan_en : in  std_logic;  -- Scan enable
        q       : out std_logic   -- Output
    );
end scan_dff;

architecture rtl of scan_dff is
    signal mux_out : std_logic;
begin

    mux_out <= si when scan_en = '1' else d;

    process(clk, rst_n)
    begin
        if rst_n = '0' then
            q <= '0';
        elsif rising_edge(clk) then
            q <= mux_out;
        end if;
    end process;

end rtl;

-- 4-bit register with scan chain
entity scan_reg4 is
    port (
        clk      : in  std_logic;
        rst_n    : in  std_logic;
        d        : in  std_logic_vector(3 downto 0);
        scan_en  : in  std_logic;
        scan_in  : in  std_logic;
        scan_out : out std_logic;
        q        : out std_logic_vector(3 downto 0)
    );
end scan_reg4;

architecture rtl of scan_reg4 is
    signal q_int : std_logic_vector(3 downto 0);
begin

    u_ff0: entity work.scan_dff
        port map (clk, rst_n, d(0), scan_in, scan_en, q_int(0));
    u_ff1: entity work.scan_dff
        port map (clk, rst_n, d(1), q_int(0), scan_en, q_int(1));
    u_ff2: entity work.scan_dff
        port map (clk, rst_n, d(2), q_int(1), scan_en, q_int(2));
    u_ff3: entity work.scan_dff
        port map (clk, rst_n, d(3), q_int(2), scan_en, q_int(3));

    q <= q_int;
    scan_out <= q_int(3);

end rtl;

Interview Notes

#
  • Why is scan needed? — Sequential circuits hard to test: can't easily set internal states. Scan provides controllability (shift in) and observability (shift out).
  • What are stuck-at faults? — Manufacturing defects causing a node to be permanently stuck at 0 (SA0) or 1 (SA1). Most common fault model.
  • What is ATPG? — Automatic Test Pattern Generation. Creates input vectors that detect specific faults. Trade-off: pattern count vs coverage.
  • Scan overhead? — Area: MUX per FF (~5-15%). Performance: extra MUX delay in timing path. Power: scan chain toggles during test.
  • What is fault coverage? — Percentage of modeled faults detected by test patterns. Target >95% for production quality.
  • Shift vs capture? — Shift: scan_en=1, chain acts as shift register. Capture: scan_en=0, one normal clock to capture response.
  • What is compression? — Reduces test data volume. Compressors/decompressors at scan I/O reduce test time and ATE memory.
