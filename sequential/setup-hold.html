Setup & Hold Time

Fundamental timing constraints that determine the maximum clock frequency and reliability of synchronous digital circuits. Mastering these concepts is essential for timing closure.

Setup Time (tsu)

Before clock edge

Hold Time (th)

After clock edge

Clock-to-Q (tco)

Output delay

Slack

Timing margin

Timing Definitions

#
Setup Time (tsu)

Minimum time data must be stable before the active clock edge. Violating setup causes incorrect data capture.

Hold Time (th)

Minimum time data must remain stable after the active clock edge. Violating hold causes metastability.

Clock-to-Q (tco)

Time from clock edge until the output is valid. Also called propagation delay or tcq.

Combinational Delay (tcomb)

Delay through logic gates between flip-flops. This is what limits your maximum frequency.

Setup Time Analysis

#

Setup analysis ensures data arrives at FF2 early enough to meet the setup requirement. Uses slow corner (worst-case max delays).

Setup Constraint

tco + tcomb + tsu < Tclk + tskew

Setup Slack (positive = passing)

Slack = Tclk - (tco + tcomb + tsu) + tskew

Maximum Frequency: fmax = 1 / (tco + tcomb + tsu - tskew). The critical path determines your chip's speed!

Hold Time Analysis

#

Hold analysis ensures data doesn't change too soon after the clock edge. Uses fast corner (minimum delays). Hold is independent of clock frequency.

Hold Constraint

tco + tcomb > th + tskew

Hold Slack (positive = passing)

Slack = (tco + tcomb) - th - tskew

Critical: Hold violations cannot be fixed by slowing the clock! They cause functional failures at any frequency and must be fixed in design.

Violations & Fixes

#
Setup Violation Fixes
  • Reduce clock frequency
  • Pipeline long paths (add FFs)
  • Use faster cells
  • Optimize logic (reduce tcomb)
  • Adjust clock skew (useful skew)
  • Better placement/routing
Hold Violation Fixes
  • Insert delay buffers
  • Use slower cells
  • Adjust clock skew
  • Add routing detours
  • Balance clock tree
  • Cannot fix with frequency!

Interview Notes

#
  • Setup vs Hold — which is worse? — Hold violations are worse! Setup can be fixed by running slower, but hold violations cause failures at any frequency.
  • What happens on setup violation? — The FF captures the wrong value (old data instead of new). Functionally incorrect but doesn't cause metastability.
  • What happens on hold violation? — The FF may go metastable, output is unpredictable. Can also capture corrupted data.
  • How do you fix negative setup slack? — Pipeline the path, use faster logic, lower frequency, useful skew, optimize routing.
  • Why can't you slow the clock to fix hold? — Hold is measured on the same clock edge. The constraint is tco + tcomb > th, no clock period involved.
  • What is useful skew? — Intentionally delaying the capture clock to give more setup slack (steals from the next cycle).
  • What corner for setup vs hold? — Setup: slow corner (max delays). Hold: fast corner (min delays).
  • What's clock pessimism removal (CPR)? — Removes false pessimism when the same clock path is analyzed with both min and max delays.
