Sequential Level-Sensitive

SR Latch

The SR (Set-Reset) latch is the simplest sequential circuit. It can store one bit of information and forms the basis for all flip-flops and memory elements.

Inputs

S (Set), R (Reset)

Outputs

Q, Q'

Gate count

2 (NOR or NAND)

Circuit type

Level-sensitive

Memory

1 bit

Forbidden state

S=1, R=1

How It Works

#

The SR latch uses cross-coupled feedback between two gates. Each gate's output feeds into the other's input, creating a stable feedback loop that can hold state:

Set (S=1, R=0)

Q → 1

Forces output Q high, stores a '1'

Reset (S=0, R=1)

Q → 0

Forces output Q low, stores a '0'

Hold (S=0, R=0)

Q → Q (no change)

Maintains previous state — memory!

Key insight: The SR latch is level-sensitive — it responds to input levels, not edges. Any change in S or R immediately affects the output (after gate delays).

NOR-based SR Latch

#

The classic SR latch uses two cross-coupled NOR gates. Active-HIGH inputs: S=1 sets, R=1 resets.

Characteristic Equations

Q = (R + Q')'

Q' = (S + Q)'

NAND-based SR Latch (S̄R̄ Latch)

#

Using NAND gates creates an SR latch with active-LOW inputs. Often written as S̄R̄ latch. S̄=0 sets, R̄=0 resets.

S̄R̄ QQ' Action
11QprevQ'prevHold (Memory)
1001Reset
0110Set
00✗✗Forbidden

NAND is preferred: In CMOS, NAND gates are faster than NOR gates due to parallel PMOS. Most real implementations use NAND-based latches.

The Forbidden State Problem

#

When S=1 and R=1 simultaneously (NOR latch), both Q and Q' become 0. This violates the fundamental requirement that Q and Q' be complements.

Race condition: If S and R return to 0 simultaneously, the final state is unpredictable — it depends on which gate is slightly faster. This is called a race condition or metastability.

Problem

S=R=1 → Q=Q'=0

Outputs are no longer complements

Solution 1

Gated SR Latch

Add enable input to control when inputs are sampled

Solution 2

D Latch / D Flip-Flop

Single data input eliminates S=R=1 possibility

RTL (Verilog / VHDL)

#
Verilog sr_latch.v 
// SR Latch - Behavioral
module sr_latch (
    input  wire S,
    input  wire R,
    output reg  Q,
    output wire Qn
);

    assign Qn = ~Q;

    always @(S, R) begin
        case ({S, R})
            2'b01: Q <= 1'b0;  // Reset
            2'b10: Q <= 1'b1;  // Set
            2'b11: Q <= 1'bx;  // Forbidden - undefined
            // 2'b00: Hold - no change
        endcase
    end

endmodule

// SR Latch - Gate-level (NOR)
module sr_latch_nor (
    input  wire S,
    input  wire R,
    output wire Q,
    output wire Qn
);

    nor (Q,  R, Qn);
    nor (Qn, S, Q);

endmodule
VHDL sr_latch.vhd 
-- SR Latch - Behavioral
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity sr_latch is
    port (
        S  : in  std_logic;
        R  : in  std_logic;
        Q  : out std_logic;
        Qn : out std_logic
    );
end sr_latch;

architecture behavioral of sr_latch is
    signal Q_int : std_logic := '0';
begin
    process(S, R)
    begin
        if S = '1' and R = '0' then
            Q_int <= '1';  -- Set
        elsif S = '0' and R = '1' then
            Q_int <= '0';  -- Reset
        elsif S = '1' and R = '1' then
            Q_int <= 'X';  -- Forbidden
        end if;
        -- S=0, R=0: Hold (no change)
    end process;

    Q  <= Q_int;
    Qn <= not Q_int;
end behavioral;

Applications

#

Switch Debouncing

Mechanical switches bounce multiple times. SR latch with two-throw switch eliminates bounce.

Memory Element

Foundation for SRAM cells. Cross-coupled inverters (derived from SR) store bits.

Control Circuits

Start/stop control, alarm systems, state flags in control logic.
