Sequential Level-Sensitive

D Latch

The D (Data) latch solves the SR latch's forbidden state problem by using a single data input. When enabled, Q follows D. When disabled, Q holds its value.

Inputs

D (Data), E (Enable)

Outputs

Q, Q'

Gate count

4 NAND gates

Circuit type

Level-sensitive

Memory

1 bit

Also called

Transparent Latch

How It Works

#

The D latch adds an enable signal to control when the latch captures data. The key insight: internally, D connects to both S and R through logic that makes S = D and R = D' when enabled.

Enable = 1 (Transparent)

Q = D

Output follows input — "transparent" mode

Enable = 0 (Latched)

Q = Qprev

Output holds last value — "latched" mode

Why no forbidden state? Since S = D and R = D', and D can only be 0 or 1, we can never have S=R=1. The forbidden state is structurally impossible!

Truth Table

0XQprevHold (Latched)

Characteristic Equation

Qnext = E · D + E' · Q

Circuit Implementation

#

The D latch is built by adding gating logic to an SR latch. The enable signal controls whether D reaches the internal SR latch.

The Transparency Problem

#

While transparent mode is useful, it causes problems in synchronous systems. When enabled, any change in D immediately appears at Q — including glitches and multiple transitions.

Timing hazard: If a latch's output feeds back to its input (directly or through logic), and the latch is transparent, the circuit can oscillate or race uncontrollably.

Problem

Transparency allows races

Changes propagate immediately through combinational logic

Solution

Edge-triggered Flip-Flop

D flip-flop samples only at clock edge, eliminating transparency

RTL (Verilog / VHDL)

#
Verilog d_latch.v 
// D Latch - Behavioral
module d_latch (
    input  wire D,
    input  wire E,      // Enable (level-sensitive)
    output reg  Q,
    output wire Qn
);

    assign Qn = ~Q;

    always @(D, E) begin
        if (E)
            Q <= D;  // Transparent when enabled
        // Hold when E=0 (implicit)
    end

endmodule

// Alternative: using always_latch (SystemVerilog)
module d_latch_sv (
    input  logic D,
    input  logic E,
    output logic Q
);

    always_latch begin
        if (E)
            Q = D;
    end

endmodule
VHDL d_latch.vhd 
-- D Latch - Behavioral
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity d_latch is
    port (
        D  : in  std_logic;
        E  : in  std_logic;  -- Enable (level-sensitive)
        Q  : out std_logic;
        Qn : out std_logic
    );
end d_latch;

architecture behavioral of d_latch is
    signal Q_int : std_logic;
begin
    process(D, E)
    begin
        if E = '1' then
            Q_int <= D;  -- Transparent when enabled
        end if;
        -- Hold when E='0' (implicit latch)
    end process;

    Q  <= Q_int;
    Qn <= not Q_int;
end behavioral;

Synthesis warning: Most synthesis tools flag latches as warnings because they're prone to timing issues. Prefer flip-flops for synchronous design.
