Sequential FSM

Sequence Detector

A finite state machine that monitors a serial input stream and asserts an output when a specific bit pattern is detected. The classic example of FSM design methodology.

States needed

N+1 (for N-bit pattern)

Key choice

Overlapping?

Implementation

Mealy or Moore

Common use

Protocol sync

Overlapping vs Non-Overlapping

#

The critical design decision: can the end of one detected sequence be the start of another?

Overlapping

After detection, reuse matching suffix as start of next search.

Input: 101011011

Detects "1011" at positions 2 and 5

Non-Overlapping

After detection, reset and start fresh search.

Input: 101011011

Detects "1011" only at position 2

Design Process

#
  1. Define the pattern — e.g., "1011"
  2. Choose overlapping or not
  3. Choose Mealy or Moore
  4. Draw state diagram — each state = "how much of pattern matched so far"
  5. Handle mismatches — go back to appropriate state (not always S0!)
  6. Create state table
  7. Implement in HDL

State naming: Use states like S0 (nothing matched), S1 (matched "1"), S10 (matched "10"), S101 (matched "101"), S1011 (matched "1011" = detected!)

Example: 1011 Detector (Overlapping, Moore)

#
State Meaning Input 0 Input 1 Output
S0Nothing matchedS0S10
S1Matched "1"S10S10
S10Matched "10"S0S1010
S101Matched "101"S10S10110
S1011Matched "1011"S10*S11

Overlapping trick: From S1011, input 0 → S10 (not S0!) because "10110" ends with "10", which is a prefix of "1011". This enables detecting overlapping sequences.

RTL (Verilog / VHDL)

#
Verilog seq_detect_1011.v 
// Sequence Detector: "1011" (Overlapping, Moore)
module seq_detect_1011 (
    input  wire clk,
    input  wire rst_n,
    input  wire din,
    output wire detected
);

    // State encoding
    localparam [2:0] S0    = 3'b000,  // Nothing matched
                      S1    = 3'b001,  // Matched "1"
                      S10   = 3'b010,  // Matched "10"
                      S101  = 3'b011,  // Matched "101"
                      S1011 = 3'b100;  // Matched "1011" - DETECTED

    reg [2:0] state, next_state;

    // State register
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n)
            state <= S0;
        else
            state <= next_state;
    end

    // Next state logic
    always @(*) begin
        case (state)
            S0:    next_state = din ? S1    : S0;
            S1:    next_state = din ? S1    : S10;
            S10:   next_state = din ? S101  : S0;
            S101:  next_state = din ? S1011 : S10;   // 0→S10 (prefix "10")
            S1011: next_state = din ? S1    : S10;   // Overlapping!
            default: next_state = S0;
        endcase
    end

    // Moore output
    assign detected = (state == S1011);

endmodule

// Non-overlapping version: change S1011 transitions
module seq_detect_1011_nonoverlap (
    input  wire clk,
    input  wire rst_n,
    input  wire din,
    output wire detected
);

    localparam [2:0] S0=0, S1=1, S10=2, S101=3, S1011=4;
    reg [2:0] state, next_state;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) state <= S0;
        else        state <= next_state;
    end

    always @(*) begin
        case (state)
            S0:    next_state = din ? S1    : S0;
            S1:    next_state = din ? S1    : S10;
            S10:   next_state = din ? S101  : S0;
            S101:  next_state = din ? S1011 : S10;
            S1011: next_state = din ? S1    : S0;  // Reset to S0!
            default: next_state = S0;
        endcase
    end

    assign detected = (state == S1011);

endmodule

// Mealy version (output 1 cycle earlier)
module seq_detect_1011_mealy (
    input  wire clk,
    input  wire rst_n,
    input  wire din,
    output wire detected
);

    localparam [1:0] S0=0, S1=1, S10=2, S101=3;  // Only 4 states!
    reg [1:0] state, next_state;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) state <= S0;
        else        state <= next_state;
    end

    always @(*) begin
        case (state)
            S0:   next_state = din ? S1   : S0;
            S1:   next_state = din ? S1   : S10;
            S10:  next_state = din ? S101 : S0;
            S101: next_state = din ? S1   : S10;  // No S1011 state!
            default: next_state = S0;
        endcase
    end

    // Mealy: output on transition
    assign detected = (state == S101) && din;

endmodule
VHDL seq_detect_1011.vhd 
-- Sequence Detector: "1011" (Overlapping, Moore)
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity seq_detect_1011 is
    port (
        clk      : in  std_logic;
        rst_n    : in  std_logic;
        din      : in  std_logic;
        detected : out std_logic
    );
end seq_detect_1011;

architecture behavioral of seq_detect_1011 is
    type state_type is (S0, S1, S10, S101, S1011);
    signal state, next_state : state_type;
begin

    -- State register
    process(clk, rst_n)
    begin
        if rst_n = '0' then
            state <= S0;
        elsif rising_edge(clk) then
            state <= next_state;
        end if;
    end process;

    -- Next state logic
    process(state, din)
    begin
        case state is
            when S0 =>
                if din = '1' then next_state <= S1;
                else              next_state <= S0;
                end if;

            when S1 =>
                if din = '1' then next_state <= S1;
                else              next_state <= S10;
                end if;

            when S10 =>
                if din = '1' then next_state <= S101;
                else              next_state <= S0;
                end if;

            when S101 =>
                if din = '1' then next_state <= S1011;
                else              next_state <= S10;   -- prefix "10"
                end if;

            when S1011 =>
                if din = '1' then next_state <= S1;    -- overlapping
                else              next_state <= S10;   -- overlapping
                end if;
        end case;
    end process;

    -- Moore output
    detected <= '1' when state = S1011 else '0';

end behavioral;

Applications

#

Protocol Sync

Detect start/sync patterns in serial protocols (UART, I2C, SPI frame boundaries).

Packet Headers

Identify packet delimiters in network protocols (Ethernet preamble 0x55...55 0xD5).

Pattern Matching

Hardware string search, barcode scanning, signature detection.

Error Detection

Detect illegal sequences or forbidden bit patterns in data streams.

Longer patterns: For patterns longer than 4-5 bits, consider using a shift register comparison instead of explicit FSM states — simpler and scales better.
