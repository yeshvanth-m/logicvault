Clock Domain Crossing (CDC)

When signals cross between clock domains, metastability and data coherency become critical concerns. Proper CDC techniques ensure reliable communication between asynchronous clock domains.

2-FF Synchronizer

Pulse Transfer

Single-Bit CDC (Level Signals)

#

For level signals (signals that stay at their new value for multiple clock cycles), a standard 2-FF synchronizer is sufficient.

Requirement: The signal must be stable for at least 2 destination clock cycles to guarantee capture. Short pulses can be missed!

Pulse Synchronization

#

Short pulses (single-cycle) can be missed by a 2-FF synchronizer if the destination clock is slower. Two solutions:

Toggle Synchronizer

Convert pulse to toggle (level change), synchronize the level, then detect edge in destination domain. Works when source is faster than destination.

Handshake Protocol

Source sends request, waits for acknowledge before sending next. Guarantees no pulses lost, but adds latency.

Multi-Bit CDC Problem

#

Never synchronize a multi-bit bus directly! Different bits may resolve at different times, creating corrupted intermediate values.

Solutions: (1) Gray code for counters (only 1 bit changes), (2) Handshake with MUX (freeze data during transfer), (3) Async FIFO for streaming data.

Handshake Protocol

#

The handshake protocol safely transfers multi-bit data by ensuring the data is stable before being sampled.

Phase Action Description
1REQ ↑Source places data, asserts request
2ACK ↑Destination samples data, asserts acknowledge
3REQ ↓Source sees ACK, deasserts request
4ACK ↓Destination sees REQ low, deasserts acknowledge

Gray Code for Pointers

#

For FIFO pointers and counters, Gray code ensures only one bit changes per increment — making 2-FF synchronization safe.

RTL (Verilog / VHDL)

#
Verilog cdc_modules.v 
// Toggle Synchronizer (Pulse CDC)
module pulse_sync (
    input  wire clk_src,
    input  wire clk_dst,
    input  wire rst_n,
    input  wire pulse_in,    // Single-cycle pulse in src domain
    output wire pulse_out   // Single-cycle pulse in dst domain
);

    // Source domain: convert pulse to toggle
    reg toggle_src;
    always @(posedge clk_src or negedge rst_n) begin
        if (!rst_n)
            toggle_src <= 1'b0;
        else if (pulse_in)
            toggle_src <= ~toggle_src;
    end

    // Destination domain: synchronize toggle
    (* ASYNC_REG = "TRUE" *) reg [2:0] sync_reg;  // 2 for sync + 1 for edge
    
    always @(posedge clk_dst or negedge rst_n) begin
        if (!rst_n)
            sync_reg <= 3'b0;
        else
            sync_reg <= {sync_reg[1:0], toggle_src};
    end

    // Edge detection (XOR of last two synced values)
    assign pulse_out = sync_reg[2] ^ sync_reg[1];

endmodule

// Handshake CDC for multi-bit data
module handshake_cdc #(
    parameter WIDTH = 8
)(
    // Source domain
    input  wire             clk_src,
    input  wire             rst_src_n,
    input  wire [WIDTH-1:0] data_in,
    input  wire             valid_in,
    output wire             ready_out,
    
    // Destination domain
    input  wire             clk_dst,
    input  wire             rst_dst_n,
    output reg  [WIDTH-1:0] data_out,
    output wire             valid_out
);

    // Source domain registers
    reg [WIDTH-1:0] data_hold;
    reg             req_src;
    
    // Synchronized signals
    (* ASYNC_REG = "TRUE" *) reg [1:0] req_sync;
    (* ASYNC_REG = "TRUE" *) reg [1:0] ack_sync;
    
    reg ack_dst;
    
    // Sync REQ to destination
    always @(posedge clk_dst or negedge rst_dst_n) begin
        if (!rst_dst_n)
            req_sync <= 2'b0;
        else
            req_sync <= {req_sync[0], req_src};
    end
    
    // Sync ACK to source
    always @(posedge clk_src or negedge rst_src_n) begin
        if (!rst_src_n)
            ack_sync <= 2'b0;
        else
            ack_sync <= {ack_sync[0], ack_dst};
    end
    
    // Source: hold data and manage request
    always @(posedge clk_src or negedge rst_src_n) begin
        if (!rst_src_n) begin
            data_hold <= '0;
            req_src   <= 1'b0;
        end else if (valid_in && ready_out) begin
            data_hold <= data_in;
            req_src   <= 1'b1;
        end else if (ack_sync[1]) begin
            req_src   <= 1'b0;
        end
    end
    
    assign ready_out = ~req_src && ~ack_sync[1];
    
    // Destination: capture data and manage ack
    always @(posedge clk_dst or negedge rst_dst_n) begin
        if (!rst_dst_n) begin
            data_out <= '0;
            ack_dst  <= 1'b0;
        end else if (req_sync[1] && !ack_dst) begin
            data_out <= data_hold;
            ack_dst  <= 1'b1;
        end else if (!req_sync[1]) begin
            ack_dst  <= 1'b0;
        end
    end
    
    assign valid_out = req_sync[1] && !ack_dst;

endmodule

// Gray code conversion functions
function [3:0] bin2gray;
    input [3:0] bin;
    bin2gray = bin ^ (bin >> 1);
endfunction

function [3:0] gray2bin;
    input [3:0] gray;
    gray2bin[3] = gray[3];
    gray2bin[2] = gray[3] ^ gray[2];
    gray2bin[1] = gray[3] ^ gray[2] ^ gray[1];
    gray2bin[0] = gray[3] ^ gray[2] ^ gray[1] ^ gray[0];
endfunction
VHDL pulse_sync.vhd 
-- Toggle Synchronizer (Pulse CDC)
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity pulse_sync is
    port (
        clk_src   : in  std_logic;
        clk_dst   : in  std_logic;
        rst_n     : in  std_logic;
        pulse_in  : in  std_logic;
        pulse_out : out std_logic
    );
end pulse_sync;

architecture rtl of pulse_sync is
    signal toggle_src : std_logic;
    signal sync_reg   : std_logic_vector(2 downto 0);
    
    attribute ASYNC_REG : string;
    attribute ASYNC_REG of sync_reg : signal is "TRUE";
begin

    -- Source: convert pulse to toggle
    process(clk_src, rst_n)
    begin
        if rst_n = '0' then
            toggle_src <= '0';
        elsif rising_edge(clk_src) then
            if pulse_in = '1' then
                toggle_src <= not toggle_src;
            end if;
        end if;
    end process;

    -- Destination: synchronize and detect edge
    process(clk_dst, rst_n)
    begin
        if rst_n = '0' then
            sync_reg <= (others => '0');
        elsif rising_edge(clk_dst) then
            sync_reg <= sync_reg(1 downto 0) & toggle_src;
        end if;
    end process;

    pulse_out <= sync_reg(2) xor sync_reg(1);

end rtl;

Interview Notes

#
  • Can you synchronize a bus with 2-FF? — No! Different bits may be captured at different times. Use Gray code (for counters) or handshake (for arbitrary data) or async FIFO.
  • What's special about Gray code? — Only 1 bit changes between consecutive values, making it safe to synchronize even if sampling occurs during transition.
  • When to use toggle sync vs handshake? — Toggle sync: fast pulse transfer, no data payload. Handshake: slower but transfers multi-bit data safely.
  • How to transfer data between clock domains? — For streaming: async FIFO. For single transfers: handshake protocol. For control signals: 2-FF sync with proper constraints.
  • What's the latency of a 2-FF synchronizer? — 2-3 destination clock cycles (due to synchronization uncertainty plus FF delays).
  • What if source clock is faster than destination? — Use toggle synchronizer for pulses, or ensure the signal holds long enough for the destination to capture it.
