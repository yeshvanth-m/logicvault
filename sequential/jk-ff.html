JK Flip-Flop

The JK flip-flop is the most versatile flip-flop. It eliminates the forbidden state of SR by defining J=K=1 as a toggle operation — the output inverts on each clock edge.

Inputs

J, K, CLK

Outputs

Q, Q'

Trigger

Edge-triggered

Special mode

J=K=1 → Toggle

Forbidden state

None!

Named after

Jack Kilby (likely)

Truth Table

#
JK Qnext Action
00QHold (No change)
010Reset
101Set
11Q'Toggle!

Characteristic Equation

Qnext = J · Q' + K' · Q

Toggle Mode (J=K=1)

#

When J=K=1, the JK flip-flop toggles (inverts) its output on each clock edge. This makes it perfect for building counters:

Frequency divider: A JK flip-flop with J=K=1 divides the clock frequency by 2. Chain N of them to divide by 2N.

Excitation Table

#

The excitation table shows what J,K inputs are needed to achieve a desired state transition. Essential for designing counters and FSMs:

QQnext JK Interpretation
000XStay 0: J must be 0, K doesn't matter
011X0→1: J must be 1 (set), K doesn't matter
10X11→0: K must be 1 (reset), J doesn't matter
11X0Stay 1: K must be 0, J doesn't matter

Don't cares (X): The X entries are powerful — they give flexibility in K-map simplification when designing counters.

RTL (Verilog / VHDL)

#
Verilog jk_ff.v 
// JK Flip-Flop
module jk_ff (
    input  wire clk,
    input  wire J,
    input  wire K,
    output reg  Q,
    output wire Qn
);

    assign Qn = ~Q;

    always @(posedge clk) begin
        case ({J, K})
            2'b00: Q <= Q;      // Hold
            2'b01: Q <= 1'b0;   // Reset
            2'b10: Q <= 1'b1;   // Set
            2'b11: Q <= ~Q;     // Toggle
        endcase
    end

endmodule

// Alternative using characteristic equation
module jk_ff_v2 (
    input  wire clk, J, K,
    output reg  Q
);

    always @(posedge clk)
        Q <= (J & ~Q) | (~K & Q);  // Q_next = J·Q' + K'·Q

endmodule
VHDL jk_ff.vhd 
-- JK Flip-Flop
library IEEE;
use IEEE.STD_LOGIC_1164.all;

entity jk_ff is
    port (
        clk : in  std_logic;
        J   : in  std_logic;
        K   : in  std_logic;
        Q   : out std_logic;
        Qn  : out std_logic
    );
end jk_ff;

architecture behavioral of jk_ff is
    signal Q_int : std_logic := '0';
begin
    process(clk)
    begin
        if rising_edge(clk) then
            if J = '0' and K = '0' then
                Q_int <= Q_int;    -- Hold
            elsif J = '0' and K = '1' then
                Q_int <= '0';      -- Reset
            elsif J = '1' and K = '0' then
                Q_int <= '1';      -- Set
            else
                Q_int <= not Q_int; -- Toggle
            end if;
        end if;
    end process;

    Q  <= Q_int;
    Qn <= not Q_int;
end behavioral;

JK vs D Flip-Flop

#
Property JK Flip-Flop D Flip-Flop
Inputs J, K (2 inputs) D (1 input)
Modes Hold, Set, Reset, Toggle Set, Reset only
Toggle capability Native (J=K=1) Requires feedback (D=Q')
Gate count Higher Lower
Modern usage Less common (textbooks, counters) Dominant (all synchronous design)

Industry reality: D flip-flops dominate modern design because synthesis tools optimize everything into DFFs anyway. JK is mostly seen in academic contexts and legacy designs.
