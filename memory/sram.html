Memory Volatile

Static RAM (SRAM)

Fast, volatile memory that uses bistable latching circuitry to store each bit. No refresh required — data persists as long as power is supplied.

Cell Type

6 Transistors

Access Time

~1-10 ns

Refresh

Not Required

Use Case

Cache, Registers

6T SRAM Cell

#

The standard SRAM cell uses 6 transistors: 2 cross-coupled inverters (4T) for storage + 2 access transistors for read/write.

Storage (4T)

Two cross-coupled inverters form a bistable latch. When Q=1, Q̄=0 and vice versa. The feedback maintains the state indefinitely.

Access (2T)

NMOS pass transistors M5 and M6 connect the cell to bit lines when Word Line (WL) is high. Enables read and write operations.

Read/Write Operations

#
Read Operation
  1. Precharge BL and BL̄ to VDD
  2. Assert Word Line (WL=1)
  3. Cell pulls one bit line low
  4. Sense amplifier detects difference
  5. Deassert WL

Read is non-destructive — cell retains data

Write Operation
  1. Drive BL to desired value, BL̄ to complement
  2. Assert Word Line (WL=1)
  3. Bit line drivers overpower cell
  4. Cell flips to new state
  5. Deassert WL

Write drivers must be stronger than cell

Read Stability: During read, the cell must not flip. The internal inverter must be stronger than the access transistor pulling toward the precharge voltage.

RTL (Verilog / VHDL)

#
Verilog sram.v 
// Single-port SRAM (synchronous read/write)
module sram_sp #(
    parameter DATA_WIDTH = 8,
    parameter ADDR_WIDTH = 10,   // 1024 locations
    parameter DEPTH = 1 << ADDR_WIDTH
)(
    input  wire                  clk,
    input  wire                  cs,      // Chip select
    input  wire                  we,      // Write enable
    input  wire [ADDR_WIDTH-1:0] addr,
    input  wire [DATA_WIDTH-1:0] din,
    output reg  [DATA_WIDTH-1:0] dout
);

    // Memory array
    reg [DATA_WIDTH-1:0] mem [0:DEPTH-1];

    always @(posedge clk) begin
        if (cs) begin
            if (we)
                mem[addr] <= din;      // Write
            else
                dout <= mem[addr];     // Read
        end
    end

endmodule

// Dual-port SRAM (one read, one write port)
module sram_dp #(
    parameter DATA_WIDTH = 8,
    parameter ADDR_WIDTH = 10
)(
    input  wire                  clk,
    // Write port
    input  wire                  we,
    input  wire [ADDR_WIDTH-1:0] waddr,
    input  wire [DATA_WIDTH-1:0] din,
    // Read port
    input  wire                  re,
    input  wire [ADDR_WIDTH-1:0] raddr,
    output reg  [DATA_WIDTH-1:0] dout
);

    localparam DEPTH = 1 << ADDR_WIDTH;
    reg [DATA_WIDTH-1:0] mem [0:DEPTH-1];

    // Write port
    always @(posedge clk) begin
        if (we)
            mem[waddr] <= din;
    end

    // Read port
    always @(posedge clk) begin
        if (re)
            dout <= mem[raddr];
    end

endmodule

// True dual-port SRAM (both ports can read/write)
module sram_tdp #(
    parameter DATA_WIDTH = 8,
    parameter ADDR_WIDTH = 10
)(
    // Port A
    input  wire                  clk_a,
    input  wire                  we_a,
    input  wire [ADDR_WIDTH-1:0] addr_a,
    input  wire [DATA_WIDTH-1:0] din_a,
    output reg  [DATA_WIDTH-1:0] dout_a,
    // Port B
    input  wire                  clk_b,
    input  wire                  we_b,
    input  wire [ADDR_WIDTH-1:0] addr_b,
    input  wire [DATA_WIDTH-1:0] din_b,
    output reg  [DATA_WIDTH-1:0] dout_b
);

    localparam DEPTH = 1 << ADDR_WIDTH;
    reg [DATA_WIDTH-1:0] mem [0:DEPTH-1];

    // Port A
    always @(posedge clk_a) begin
        if (we_a)
            mem[addr_a] <= din_a;
        dout_a <= mem[addr_a];
    end

    // Port B
    always @(posedge clk_b) begin
        if (we_b)
            mem[addr_b] <= din_b;
        dout_b <= mem[addr_b];
    end

endmodule
VHDL sram.vhd 
-- Single-port SRAM
library IEEE;
use IEEE.STD_LOGIC_1164.all;
use IEEE.NUMERIC_STD.all;

entity sram_sp is
    generic (
        DATA_WIDTH : integer := 8;
        ADDR_WIDTH : integer := 10
    );
    port (
        clk  : in  std_logic;
        cs   : in  std_logic;
        we   : in  std_logic;
        addr : in  std_logic_vector(ADDR_WIDTH-1 downto 0);
        din  : in  std_logic_vector(DATA_WIDTH-1 downto 0);
        dout : out std_logic_vector(DATA_WIDTH-1 downto 0)
    );
end sram_sp;

architecture rtl of sram_sp is
    constant DEPTH : integer := 2**ADDR_WIDTH;
    type mem_array is array(0 to DEPTH-1) 
        of std_logic_vector(DATA_WIDTH-1 downto 0);
    signal mem : mem_array;
begin

    process(clk)
    begin
        if rising_edge(clk) then
            if cs = '1' then
                if we = '1' then
                    mem(to_integer(unsigned(addr))) <= din;
                else
                    dout <= mem(to_integer(unsigned(addr)));
                end if;
            end if;
        end if;
    end process;

end rtl;

Interview Notes

#
  • Why 6T, not 4T or 8T? — 4T uses resistors (high power). 8T adds read ports (better stability but larger area). 6T is the standard trade-off.
  • SRAM vs DRAM? — SRAM: faster, no refresh, 6T/cell (larger). DRAM: slower, needs refresh, 1T1C/cell (denser). SRAM for cache, DRAM for main memory.
  • What is read stability? — During read, the cell must resist flipping when bit line voltage tries to disturb the storage node. Solved by making cell transistors stronger than access transistors.
  • What is write-ability? — Write drivers must overpower the cell's internal feedback. Access transistors must be strong enough to pull internal nodes.
  • Why precharge bit lines? — Precharging to VDD (or VDD/2) creates a known starting point. The cell then pulls one line low, creating a detectable differential.
  • What does the sense amplifier do? — Detects the small voltage difference between BL and BL̄ (often only 50-100mV) and amplifies it to full logic levels.
  • Single-port vs dual-port? — Single-port: one read OR write per cycle. Dual-port: simultaneous read and write (common in register files).
