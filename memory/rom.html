Memory Non-Volatile

Read-Only Memory (ROM)

Non-volatile memory that stores data permanently. Contents are fixed at manufacture or programmed once. Used for boot code, lookup tables, and constants.

Volatility

Non-Volatile

Write

None / Once

Implementation

AND-OR / LUT

Use Case

Boot, Constants

In RTL: We typically model ROM as a lookup table initialized with constants. Synthesis tools infer the appropriate implementation (LUTs, block RAM, etc.).

Internal Structure

#

ROM is fundamentally an address decoder + fixed data. It can be viewed as implementing a truth table where address is input and stored data is output.

NOR-based ROM

Each word line activates a row. Transistors at programmed locations pull bit lines low. Fast read, larger cell size.

NAND-based ROM

Cells in series like NAND flash. Denser but slower access. Common in flash memory.

ROM as Logic

#

ROM can implement any combinational function! With n address bits, you can store 2ⁿ entries — perfect for lookup tables, character generators, and function approximation.

Example: Sine LUT

Store pre-computed sine values. 8-bit angle input → 8-bit sine output. 256 entries × 8 bits = 256 bytes.

sin[0°] = 0x00 sin[90°] = 0x7F sin[180°] = 0x00 sin[270°] = 0x81
Example: BCD to 7-Seg

4-bit BCD input → 7-bit segment output. Classic ROM application! Only 16 entries needed.

0 → 0111111 (abcdef-) 1 → 0000110 (--bc---) 9 → 1101111 (abcd-fg)

RTL (Verilog / VHDL)

Verilog rom.v 
// ROM using case statement (small ROMs)
module rom_case #(
    parameter ADDR_WIDTH = 4,
    parameter DATA_WIDTH = 8
)(
    input  wire [ADDR_WIDTH-1:0] addr,
    output reg  [DATA_WIDTH-1:0] data
);

    always @(*) begin
        case (addr)
            4'h0: data = 8'h3F;  // 7-seg '0'
            4'h1: data = 8'h06;  // 7-seg '1'
            4'h2: data = 8'h5B;  // 7-seg '2'
            4'h3: data = 8'h4F;  // 7-seg '3'
            4'h4: data = 8'h66;  // 7-seg '4'
            4'h5: data = 8'h6D;  // 7-seg '5'
            4'h6: data = 8'h7D;  // 7-seg '6'
            4'h7: data = 8'h07;  // 7-seg '7'
            4'h8: data = 8'h7F;  // 7-seg '8'
            4'h9: data = 8'h6F;  // 7-seg '9'
            default: data = 8'h00;
        endcase
    end

endmodule

// ROM using memory array (larger ROMs)
module rom_array #(
    parameter ADDR_WIDTH = 8,
    parameter DATA_WIDTH = 8,
    parameter DEPTH = 1 << ADDR_WIDTH
)(
    input  wire                  clk,
    input  wire [ADDR_WIDTH-1:0] addr,
    output reg  [DATA_WIDTH-1:0] data
);

    // Initialize from file
    reg [DATA_WIDTH-1:0] rom [0:DEPTH-1];
    
    initial begin
        $readmemh("rom_contents.hex", rom);
    end

    // Synchronous read
    always @(posedge clk) begin
        data <= rom[addr];
    end

endmodule

// Sine lookup table ROM
module sine_lut (
    input  wire       clk,
    input  wire [7:0] angle,    // 0-255 maps to 0°-360°
    output reg  [7:0] sine_val  // Signed: -127 to +127
);

    reg [7:0] sine_rom [0:255];

    initial begin
        // First quadrant only shown
        sine_rom[0]   = 8'sd0;
        sine_rom[16]  = 8'sd25;
        sine_rom[32]  = 8'sd49;
        sine_rom[48]  = 8'sd71;
        sine_rom[64]  = 8'sd90;   // ~sin(90°)
        sine_rom[80]  = 8'sd106;
        sine_rom[96]  = 8'sd117;
        sine_rom[112] = 8'sd125;
        sine_rom[128] = 8'sd127;  // sin(180° in this mapping)
        // ... fill rest
    end

    always @(posedge clk) begin
        sine_val <= sine_rom[angle];
    end

endmodule
VHDL rom.vhd 
-- ROM using constant array
library IEEE;
use IEEE.STD_LOGIC_1164.all;
use IEEE.NUMERIC_STD.all;

entity rom_array is
    generic (
        ADDR_WIDTH : integer := 4;
        DATA_WIDTH : integer := 8
    );
    port (
        clk  : in  std_logic;
        addr : in  std_logic_vector(ADDR_WIDTH-1 downto 0);
        data : out std_logic_vector(DATA_WIDTH-1 downto 0)
    );
end rom_array;

architecture rtl of rom_array is
    type rom_type is array(0 to 2**ADDR_WIDTH-1) 
        of std_logic_vector(DATA_WIDTH-1 downto 0);
    
    constant ROM : rom_type := (
        x"3F", x"06", x"5B", x"4F",  -- 0-3
        x"66", x"6D", x"7D", x"07",  -- 4-7
        x"7F", x"6F", x"77", x"7C",  -- 8-B
        x"39", x"5E", x"79", x"71"   -- C-F
    );
begin

    process(clk)
    begin
        if rising_edge(clk) then
            data <= ROM(to_integer(unsigned(addr)));
        end if;
    end process;

end rtl;

Interview Notes

#
  • How is ROM different from RAM? — ROM is non-volatile and read-only (or write-once). RAM is volatile and freely read/write. ROM stores constants; RAM stores variables.
  • Can ROM implement any function? — Yes! ROM with n inputs can implement any function of n variables. It's essentially a complete truth table.
  • ROM vs PLA? — ROM has a fixed AND array (decoder) and programmable OR array. PLA has both programmable. ROM is simpler but may waste space for sparse functions.
  • How do synthesis tools handle ROM? — Small ROMs become LUT cascades. Larger ROMs use block RAM initialized with constants.
  • What's $readmemh? — Verilog system task to initialize memory from a hex file. Useful for loading ROM contents from external files.
  • Sync vs async ROM read? — Async: combinational (addr → data). Sync: registered (addr on clock → data next cycle). Sync is more common in FPGAs.
