Register File

Small, fast multi-port memory used in CPU cores to store operands. Supports multiple simultaneous reads and writes for pipelined execution.

Size

8-128 registers

Read Ports

2-6 typical

Write Ports

1-2 typical

Access

Same cycle R+W

Architecture

#
Write Path

Decoder selects which register to write. Write enable gates the clock to flip-flops. Data latched on clock edge.

Read Path

Multiplexer selects register output. Can be asynchronous (combinational) or synchronous (registered output).

Port Configurations

#
Configuration Ports Use Case
1R1W1 read, 1 writeSimple in-order CPUs
2R1W2 read, 1 writeStandard RISC (rs1, rs2, rd)
3R1W3 read, 1 writeFMA operations
2R2W2 read, 2 writeDual-issue cores
6R3W6 read, 3 writeSuperscalar OoO

Area Impact: Each read port adds a full MUX tree. Each write port adds decoder logic. A 6R3W file can be 10× the area of 2R1W!

Read-After-Write Hazard

#

What happens when you read and write the same register in the same cycle?

Read-First (Old Value)

Read completes before write. Reads return the old value. Simple but may need forwarding logic.

R1 = 5 Write R1 ← 10, Read R1 → Returns 5
Write-First (New Value)

Write completes before read. Reads return the new value. Requires internal bypass/forwarding.

R1 = 5 Write R1 ← 10, Read R1 → Returns 10

RTL (Verilog / VHDL)

#
Verilog regfile.v 
// 32x32 Register File: 2 read ports, 1 write port
// Read-first behavior (old value on RAW conflict)
module regfile_2r1w #(
    parameter DATA_WIDTH = 32,
    parameter ADDR_WIDTH = 5,      // 32 registers
    parameter ZERO_REG   = 1       // R0 hardwired to 0
)(
    input  wire                  clk,
    input  wire                  rst_n,
    
    // Read port A
    input  wire [ADDR_WIDTH-1:0] raddr_a,
    output wire [DATA_WIDTH-1:0] rdata_a,
    
    // Read port B
    input  wire [ADDR_WIDTH-1:0] raddr_b,
    output wire [DATA_WIDTH-1:0] rdata_b,
    
    // Write port
    input  wire                  we,
    input  wire [ADDR_WIDTH-1:0] waddr,
    input  wire [DATA_WIDTH-1:0] wdata
);

    localparam NUM_REGS = 1 << ADDR_WIDTH;
    reg [DATA_WIDTH-1:0] regs [0:NUM_REGS-1];

    integer i;

    // Write logic
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            for (i = 0; i < NUM_REGS; i = i + 1)
                regs[i] <= {DATA_WIDTH{1'b0}};
        end else if (we) begin
            // Don't write to R0 if ZERO_REG enabled
            if (!(ZERO_REG && waddr == 0))
                regs[waddr] <= wdata;
        end
    end

    // Async read (combinational)
    assign rdata_a = (ZERO_REG && raddr_a == 0) ? 
                     {DATA_WIDTH{1'b0}} : regs[raddr_a];
    assign rdata_b = (ZERO_REG && raddr_b == 0) ? 
                     {DATA_WIDTH{1'b0}} : regs[raddr_b];

endmodule

// Register File with write-first bypass
module regfile_bypass #(
    parameter DATA_WIDTH = 32,
    parameter ADDR_WIDTH = 5
)(
    input  wire                  clk,
    input  wire                  rst_n,
    
    input  wire [ADDR_WIDTH-1:0] raddr_a,
    output wire [DATA_WIDTH-1:0] rdata_a,
    
    input  wire [ADDR_WIDTH-1:0] raddr_b,
    output wire [DATA_WIDTH-1:0] rdata_b,
    
    input  wire                  we,
    input  wire [ADDR_WIDTH-1:0] waddr,
    input  wire [DATA_WIDTH-1:0] wdata
);

    localparam NUM_REGS = 1 << ADDR_WIDTH;
    reg [DATA_WIDTH-1:0] regs [0:NUM_REGS-1];

    // Write logic
    integer i;
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            for (i = 0; i < NUM_REGS; i = i + 1)
                regs[i] <= 0;
        end else if (we && waddr != 0) begin
            regs[waddr] <= wdata;
        end
    end

    // Read with bypass: if reading same addr as writing, return write data
    wire bypass_a = we && (raddr_a == waddr) && (waddr != 0);
    wire bypass_b = we && (raddr_b == waddr) && (waddr != 0);

    assign rdata_a = (raddr_a == 0) ? 0 :
                     bypass_a ? wdata : regs[raddr_a];
    assign rdata_b = (raddr_b == 0) ? 0 :
                     bypass_b ? wdata : regs[raddr_b];

endmodule
VHDL regfile.vhd 
-- 32x32 Register File: 2 read, 1 write
library IEEE;
use IEEE.STD_LOGIC_1164.all;
use IEEE.NUMERIC_STD.all;

entity regfile_2r1w is
    generic (
        DATA_WIDTH : integer := 32;
        ADDR_WIDTH : integer := 5
    );
    port (
        clk     : in  std_logic;
        rst_n   : in  std_logic;
        -- Read port A
        raddr_a : in  std_logic_vector(ADDR_WIDTH-1 downto 0);
        rdata_a : out std_logic_vector(DATA_WIDTH-1 downto 0);
        -- Read port B
        raddr_b : in  std_logic_vector(ADDR_WIDTH-1 downto 0);
        rdata_b : out std_logic_vector(DATA_WIDTH-1 downto 0);
        -- Write port
        we      : in  std_logic;
        waddr   : in  std_logic_vector(ADDR_WIDTH-1 downto 0);
        wdata   : in  std_logic_vector(DATA_WIDTH-1 downto 0)
    );
end regfile_2r1w;

architecture rtl of regfile_2r1w is
    constant NUM_REGS : integer := 2**ADDR_WIDTH;
    type reg_array is array(0 to NUM_REGS-1) 
        of std_logic_vector(DATA_WIDTH-1 downto 0);
    signal regs : reg_array;
begin

    -- Write process
    process(clk, rst_n)
    begin
        if rst_n = '0' then
            for i in 0 to NUM_REGS-1 loop
                regs(i) <= (others => '0');
            end loop;
        elsif rising_edge(clk) then
            if we = '1' and unsigned(waddr) /= 0 then
                regs(to_integer(unsigned(waddr))) <= wdata;
            end if;
        end if;
    end process;

    -- Async read with R0=0
    rdata_a <= (others => '0') when unsigned(raddr_a) = 0
               else regs(to_integer(unsigned(raddr_a)));
    rdata_b <= (others => '0') when unsigned(raddr_b) = 0
               else regs(to_integer(unsigned(raddr_b)));

end rtl;

Interview Notes

#
  • Why is R0 hardwired to zero? — Common in RISC ISAs (MIPS, RISC-V). Provides a constant zero without using an immediate. Simplifies comparisons and moves.
  • Register file vs SRAM? — Register file: small, multi-port, very fast, flip-flop based. SRAM: larger, typically 1-2 ports, 6T cells. Register file is in critical path.
  • How to handle RAW hazard? — Either forward from write port (write-first) or use pipeline bypass/forwarding from later stages.
  • Why async reads? — Allows read in same cycle as decode. Read address comes from instruction bits; data needed immediately for execute stage.
  • How does area scale with ports? — Read ports: N×log₂(depth) MUX. Write ports: decoder + enable logic per register. Multi-port can replicate entire array.
  • What is register renaming? — OoO CPUs map architectural registers to larger physical register file. Eliminates false dependencies (WAR, WAW).
