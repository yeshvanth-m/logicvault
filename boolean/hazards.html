Hazard Analysis

Hazards are unwanted transient glitches in combinational circuits caused by unequal propagation delays. Understanding and eliminating hazards is crucial for reliable digital circuit design.

Static-1 Hazard

1 → 0 → 1 glitch

Static-0 Hazard

0 → 1 → 0 glitch

Detection

K-Map adjacency

Elimination

Add redundant terms

Types of Hazards

#

Static-1 Hazard

Output should remain 1 but momentarily goes to 0 due to different path delays.

Dynamic Hazard

Output should change once (0→1 or 1→0) but changes multiple times.

Expected: ____/‾‾‾‾‾‾‾‾ Actual: ____/‾\_/‾‾‾‾ ↑ multiple

Key insight: Static hazards occur in SOP circuits when switching between adjacent 1s (or 0s in POS) that are covered by different product terms. The variable that changes may momentarily make both terms 0.

Detecting Hazards

#

Use the K-Map to detect static-1 hazards in SOP implementations:

Detection Method (for SOP)
  1. Create the K-Map for the function
  2. Find adjacent 1s that are covered by different groups
  3. Each such pair indicates a potential static-1 hazard
  4. The hazard occurs when switching between these adjacent cells

Consider: F = A'C + BC

AB\C 0 1
00 0 1
01 0 1
11 0 1
10 0 1

Hazard detected! The cells at AB=01,C=1 and AB=11,C=1 are adjacent (B changes from 0→1), but covered by different groups (A'C vs BC). When B transitions, a glitch may occur.

Eliminating Hazards

#
Elimination Method
  1. Identify all pairs of adjacent 1s in different groups
  2. Add a redundant product term to cover each hazardous pair
  3. The redundant term bridges the transition between groups
  4. Result: logically equivalent, but hazard-free
Original (has hazard)

F = A'C + BC

Hazard-free

F = A'C + BC + A'B

Why it works: The redundant term A'B covers both cells (01,1 and 11,1). During the B transition, A'B remains 1 (since A=0 and B transitions through both states), preventing the glitch.

Worked Example

#

Design a hazard-free circuit for: F(A,B,C,D) = Σm(1,3,5,7,8,9,12,13)

AB\CD 00 01 11 10
00 0 1 1 0
01 0 1 1 0
11 1 1 0 0
10 1 1 0 0

Minimum SOP: F = A'D + AC'D'

Hazard analysis:

  • Cells (11,01) and (11,00): Adjacent, both covered by AC'D' ✓ No hazard
  • Cells (01,01) and (11,01): Adjacent, A'D and AC'D' — HAZARD!
Minimum SOP

F = A'D + AC'D'

Hazard-free SOP

F = A'D + AC'D' + BC'D

Trade-off: Hazard-free circuits require additional gates (redundant terms). In high-speed synchronous designs, hazards are often acceptable since outputs are sampled at clock edges when they're stable.
