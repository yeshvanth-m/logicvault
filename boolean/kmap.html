Karnaugh Maps (K-Map)

Karnaugh maps provide a visual method for simplifying Boolean expressions. By grouping adjacent 1s in the map, you can quickly derive minimal SOP (Sum of Products) or POS (Product of Sums) expressions.

Invented by

Maurice Karnaugh (1953)

Variables supported

Practical up to 5-6

Key concept

Gray code adjacency

Result

Minimal two-level logic

Grouping Rules

#
📦
Groups must be powers of 2
(1, 2, 4, 8, 16...)
🔲
Groups must be rectangular
(squares or rectangles)
🔗
Groups can wrap around
edges (top↔bottom, left↔right)
📈
Make groups as large as possible
🔄
Groups can overlap
✅
Every 1 must be in at least one group

Gray code ordering: Adjacent cells differ by only one variable — this is why grouping works! The order 00, 01, 11, 10 ensures single-bit changes.

2-Variable K-Map

#

The simplest K-Map with 2² = 4 cells. Example: F(A,B) = Σm(1,2,3)

A\B 0 1
0 0 1
1 1 1
Group 1: B (vertical)
Group 2: A (horizontal)
F = A + B

3-Variable K-Map

#

A 2×4 map with 2³ = 8 cells. Note the Gray code ordering: 00, 01, 11, 10. Example: F(A,B,C) = Σm(0,2,4,5,6)

A\BC 00 01 11 10
0 1 0 0 1
1 1 1 0 1
Group 1: C' (wraps left↔right)
Group 2: A·B'
F = C' + A·B'

4-Variable K-Map

#

A 4×4 map with 2⁴ = 16 cells. Both axes use Gray code. Example: F(A,B,C,D) = Σm(0,1,2,5,8,9,10)

AB\CD 00 01 11 10
00 1 1 0 1
01 0 1 0 0
11 0 0 0 0
10 1 1 0 1
Group 1: B'·D' (4 corners!)
Group 2: A'·C'·D
F = B'·D' + A'·C'·D

Corner trick: All 4 corners are adjacent! They form a valid group because the K-Map wraps both horizontally AND vertically (like a torus).

Don't Care Conditions (X)

#

Don't care conditions (marked X or d) represent input combinations that either never occur or whose output doesn't matter. You can treat X as 0 or 1 — whichever gives simpler groupings.

AB\CD 00 01 11 10
00 1 0 0 1
01 0 0 X 0
11 0 0 X 1
10 1 0 0 1

Strategy with Don't Cares

  1. Include X in a group if it makes the group larger
  2. Ignore X if it doesn't help simplification
  3. Never group only X values (must include at least one required 1)
F = D' (using X at position 1111 to extend group)

K-Map for POS (Product of Sums)

#

To get POS form instead of SOP:

POS from K-Map

  1. Group the 0s instead of 1s
  2. Write the term that makes each group 0
  3. Complement each term (so it produces 1 when we want it)
  4. AND all maxterms together

When to use POS: If there are fewer 0s than 1s in the truth table, POS will be simpler. Also, POS maps directly to NOR-NOR implementations.
