Fundamentals Theory

Boolean Algebra

Boolean algebra is the mathematical foundation for digital logic design. It defines operations on binary values (0 and 1) and provides laws for simplifying and manipulating logical expressions.

Invented by

George Boole (1854)

Binary values

0 (False), 1 (True)

Basic operations

AND, OR, NOT

Application

Circuit minimization

Basic Laws

#

🔄 Identity Law

A + 0 = A
A · 1 = A

ORing with 0 or ANDing with 1 leaves the value unchanged.

🔳 Null Law

A + 1 = 1
A · 0 = 0

ORing with 1 gives 1; ANDing with 0 gives 0.

🪞 Idempotent Law

A + A = A
A · A = A

ORing or ANDing a variable with itself has no effect.

⚡ Complement Law

A + A' = 1
A · A' = 0

A variable ORed with its complement is 1; ANDed is 0.

🔁 Double Negation

(A')' = A
—

Inverting twice returns the original value.

🔀 Commutative Law

A + B = B + A
A · B = B · A

Order of operands doesn't matter.

📦 Associative Law

(A + B) + C = A + (B + C)
(A · B) · C = A · (B · C)

Grouping of operands doesn't matter.

🌳 Distributive Law

A · (B + C) = A·B + A·C
A + (B · C) = (A+B) · (A+C)

AND distributes over OR, and OR distributes over AND.

Key Theorems

#
Theorem OR Form AND Form (Dual)
Absorption A + A·B = A A · (A + B) = A
Redundancy A + A'·B = A + B A · (A' + B) = A · B
Consensus A·B + A'·C + B·C = A·B + A'·C (A+B)·(A'+C)·(B+C) = (A+B)·(A'+C)

Absorption is key: It's the most frequently used theorem in manual simplification. Remember: A "absorbs" A·B because if A is true, A·B doesn't add anything.

De Morgan's Laws

#

De Morgan's laws are essential for converting between AND and OR operations, and for pushing inversions inward or outward in expressions.

📐 First Law

(A + B)' = A' · B'

"Break the bar, change the sign" — The complement of OR becomes AND of complements.

📐 Second Law

(A · B)' = A' + B'

"Break the bar, change the sign" — The complement of AND becomes OR of complements.

Example: Applying De Morgan's Law

Original: (A·B + C)'
Apply De Morgan: (A·B)' · C' // break bar, + → ·
Apply De Morgan again: (A' + B') · C' // break bar on A·B
Result: A'·C' + B'·C' // distribute if needed

Hardware insight: De Morgan's laws explain why NAND = bubbled OR and NOR = bubbled AND. This is the foundation of NAND-only or NOR-only implementations.

Simplification Examples

#

Example 1: Using Absorption

F = A·B + A·B·C
F = A·B · (1 + C) // factor out A·B
F = A·B · 1 // (1 + C) = 1 by null law
F = A·B // simplified!

Example 2: Using Complement + Distribution

F = A·B + A·B' + A'·B
F = A·(B + B') + A'·B // factor A (distributive law)
F = A·1 + A'·B // complement law: B + B' = 1
F = A + A'·B // identity law: A·1 = A
F = A + B // redundancy theorem (proof below)

Proof: A + A'·B = A + B (Redundancy Theorem)

Method 1: Using OR distribution (shorter)

A + A'·B
= (A + A')·(A + B) // OR distributes over AND: X + Y·Z = (X+Y)·(X+Z)
= 1·(A + B) // complement law: A + A' = 1
= A + B // identity law ∎

Method 2: Using AND distribution (step-by-step)

A + A'·B
= A·1 + A'·B // identity law: A = A·1
= A·(1 + B) + A'·B // null law: 1 = 1 + B
= A + A·B + A'·B // distributive law: A·(1 + B) = A + A·B
= A + B·(A + A') // distributive law: A·B + A'·B = B·(A + A')
= A + B·1 // complement law: A + A' = 1
= A + B // identity law: B·1 = B ∎

Example 3: Complex Expression

F = (A + B)·(A + C)
F = A·A + A·C + B·A + B·C // distribute (FOIL)
F = A + A·C + A·B + B·C // A·A = A
F = A·(1 + C + B) + B·C // factor A
F = A + B·C // 1 + anything = 1

Duality Principle

#

Every Boolean algebraic expression has a dual. To find the dual:

  • Swap all AND (·) with OR (+)
  • Swap all OR (+) with AND (·)
  • Swap all 0s with 1s
  • Swap all 1s with 0s
  • Keep variables and complements unchanged
Original Expression Dual Expression
A + 0 = A A · 1 = A
A + A' = 1 A · A' = 0
A + A·B = A A · (A + B) = A
(A + B)' = A'·B' (A · B)' = A' + B'

Why duality matters: If you prove a theorem, you get its dual for free! This halves the work in proving Boolean identities.
