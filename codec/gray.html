Overview

Gray code (also called reflected binary code) is a binary numbering system where two successive values differ in only one bit. This property makes it invaluable for rotary encoders, reducing glitch errors when transitioning between values, and safely crossing clock domains in digital systems.

Unlike standard binary where 7→8 (0111→1000) changes all 4 bits simultaneously, in Gray code the transition 0100→1100 changes only one bit. This eliminates transient errors from different bits changing at slightly different times.

Why "Gray" Code?

Named after Frank Gray, a Bell Labs researcher who patented it in 1953 for use in pulse code communication. Originally called "reflected binary code" due to its construction method using reflection.

Hamming Distance = 1

Every adjacent pair differs by exactly one bit. This is the defining property.

Conversion Formulas

Binary to Gray

Conversion Formula
G_{n-1} = B_{n-1} (MSB unchanged)
Gᵢ = Bᵢ ⊕ Bᵢ₊₁ (XOR adjacent bits)

Example: Binary 1011 → Gray

Binary:  B₃  B₂  B₁  B₀
          1   0   1   1

G₃ = B₃         = 1
G₂ = B₃ ⊕ B₂    = 1 ⊕ 0 = 1
G₁ = B₂ ⊕ B₁    = 0 ⊕ 1 = 1
G₀ = B₁ ⊕ B₀    = 1 ⊕ 1 = 0

Gray:    G₃  G₂  G₁  G₀
          1   1   1   0  = 1110

Gray to Binary

Conversion Formula
B_{n-1} = G_{n-1} (MSB unchanged)
Bᵢ = Gᵢ ⊕ Bᵢ₊₁ (XOR Gray with higher binary)

Example: Gray 1110 → Binary

Gray:    G₃  G₂  G₁  G₀
          1   1   1   0

B₃ = G₃         = 1
B₂ = G₂ ⊕ B₃    = 1 ⊕ 1 = 0
B₁ = G₁ ⊕ B₂    = 1 ⊕ 0 = 1
B₀ = G₀ ⊕ B₁    = 0 ⊕ 1 = 1

Binary:  B₃  B₂  B₁  B₀
          1   0   1   1  = 1011

Key Differences

Binary → Gray Parallel: all XORs can compute simultaneously
Gray → Binary Sequential: each bit depends on higher bit
Delay Difference B→G: O(1), G→B: O(N) or O(log N) with prefix

RTL Implementation

VERILOG gray_code.v 
// Binary to Gray Code Converter
module bin2gray #(
    parameter WIDTH = 4
)(
    input  [WIDTH-1:0] bin,
    output [WIDTH-1:0] gray
);

// G = B XOR (B >> 1)
assign gray = bin ^ (bin >> 1);

endmodule

// Gray Code to Binary Converter
module gray2bin #(
    parameter WIDTH = 4
)(
    input  [WIDTH-1:0] gray,
    output [WIDTH-1:0] bin
);

// Ripple XOR (serial dependency)
genvar i;
generate
    for (i = 0; i < WIDTH; i = i + 1) begin : bit_loop
        assign bin[i] = ^gray[WIDTH-1:i];  // XOR all higher bits
    end
endgenerate

endmodule

// Alternative: Unrolled for timing
module gray2bin_4bit (
    input  [3:0] gray,
    output [3:0] bin
);

assign bin[3] = gray[3];
assign bin[2] = gray[3] ^ gray[2];
assign bin[1] = gray[3] ^ gray[2] ^ gray[1];
assign bin[0] = gray[3] ^ gray[2] ^ gray[1] ^ gray[0];

endmodule

// Gray counter (increments in Gray code)
module gray_counter #(
    parameter WIDTH = 4
)(
    input              clk,
    input              rst_n,
    output [WIDTH-1:0] gray_out
);

reg [WIDTH-1:0] bin_cnt;

always @(posedge clk or negedge rst_n) begin
    if (!rst_n)
        bin_cnt <= 0;
    else
        bin_cnt <= bin_cnt + 1;
end

// Convert binary counter to Gray
assign gray_out = bin_cnt ^ (bin_cnt >> 1);

endmodule

// FIFO pointer using Gray code (CDC safe)
module gray_fifo_ptr #(
    parameter DEPTH = 8,
    parameter PTR_W = $clog2(DEPTH) + 1
)(
    input              clk,
    input              rst_n,
    input              inc,
    output [PTR_W-1:0] ptr_gray,  // For CDC
    output [PTR_W-1:0] ptr_bin    // For local use
);

reg [PTR_W-1:0] ptr;

always @(posedge clk or negedge rst_n) begin
    if (!rst_n)
        ptr <= 0;
    else if (inc)
        ptr <= ptr + 1;
end

assign ptr_bin  = ptr;
assign ptr_gray = ptr ^ (ptr >> 1);

endmodule
VHDL gray_code.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

-- Binary to Gray Converter
entity bin2gray is
    generic (WIDTH : integer := 4);
    port (
        bin  : in  std_logic_vector(WIDTH-1 downto 0);
        gray : out std_logic_vector(WIDTH-1 downto 0)
    );
end bin2gray;

architecture behavioral of bin2gray is
begin
    gray <= bin xor ('0' & bin(WIDTH-1 downto 1));
end behavioral;

-- Gray to Binary Converter
entity gray2bin is
    generic (WIDTH : integer := 4);
    port (
        gray : in  std_logic_vector(WIDTH-1 downto 0);
        bin  : out std_logic_vector(WIDTH-1 downto 0)
    );
end gray2bin;

architecture behavioral of gray2bin is
    signal bin_int : std_logic_vector(WIDTH-1 downto 0);
begin
    bin_int(WIDTH-1) <= gray(WIDTH-1);
    
    gen_bits: for i in WIDTH-2 downto 0 generate
        bin_int(i) <= gray(i) xor bin_int(i+1);
    end generate;
    
    bin <= bin_int;
end behavioral;

Applications

Rotary Encoders

Position sensors where only one bit changes per step. Eliminates glitch errors.

Motor position sensing
Asynchronous FIFOs

Gray pointers cross clock domains safely. One-bit change = metastability safe.

CDC FIFO design
Karnaugh Maps

Row/column labels use Gray order so adjacent cells differ by one variable.

00, 01, 11, 10
Genetic Algorithms

Gray encoding preserves locality in search space for optimization.

Mutation locality

Why Gray Code for CDC?

When crossing clock domains, if multiple bits change simultaneously, metastability could cause some bits to be captured early and others late, resulting in a completely wrong value.

Binary 7→8 0111→1000 (4 bits change, could read 0000, 1111, etc.)
Gray 7→8 0100→1100 (1 bit changes, can only read 7 or 8)

Interview Notes

  • Gray code: Adjacent values differ by exactly 1 bit. Hamming distance = 1.
  • Binary→Gray: G = B XOR (B >> 1). Parallel, single gate delay.
  • Gray→Binary: Bᵢ = XOR of all Gray bits from MSB to position i. Serial dependency.
  • CDC use: Gray-encoded FIFO pointers prevent multi-bit change errors across clock domains.
  • K-map connection: Gray order (00,01,11,10) ensures adjacent cells differ by one variable.
  • Rotary encoders: Only one sensor changes state per step, preventing glitch errors.
