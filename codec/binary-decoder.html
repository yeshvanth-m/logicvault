Overview

A Binary Decoder (or N-to-2ⁿ decoder) converts an N-bit binary input into 2ⁿ output lines, where exactly one output is active (HIGH) at any time. The decoder essentially generates all minterms of the input variables — each output corresponds to one minterm.

Decoders are fundamental to digital systems: they enable memory chip selection, instruction decoding in CPUs, address decoding in memory-mapped I/O, and serve as universal logic function generators when combined with OR gates.

Decoder vs DEMUX

Decoder Enable signal controls all outputs (0 or 1)
DEMUX Data input routed to selected output line
Relationship Decoder with enable ≈ 1-to-N DEMUX

Truth Table

1 0 0 0 0 0 1

3-to-8 Decoder (74LS138)

E₁ E̅₂ E̅₃ A₂ A₁ A₀ Outputs
0 X X X X X All HIGH
X 1 X X X X All HIGH
X X 1 X X X All HIGH
1 0 0 0 0 0 Y̅₀ = 0
1 0 0 0 0 1 Y̅₁ = 0
1 0 0 1 1 1 Y̅₇ = 0

74LS138 Enable Logic

Three enables E₁ (active-high), E̅₂ and E̅₃ (active-low)
Enabled when E₁=1 AND E̅₂=0 AND E̅₃=0
Outputs Active-low (selected output goes LOW)

Boolean Equations

2-to-4 Decoder

Output Equations (Active-High)
Y₀ = EN · A̅₁ · A̅₀ = EN · m₀
Y₁ = EN · A̅₁ · A₀ = EN · m₁
Y₂ = EN · A₁ · A̅₀ = EN · m₂
Y₃ = EN · A₁ · A₀ = EN · m₃

General Pattern

N-to-2ⁿ Decoder
Yᵢ = EN · mᵢ(A_{N-1}, ..., A₀)

where mᵢ = minterm i of input variables
Minterm Generator

Each output is a unique minterm. Decoder = complete minterm generator.

Yᵢ = mᵢ(A, B, C...)
Function Generator

Any function = OR of minterms. Decoder + OR gate = any function.

F = Σm(1,3,5) = Y₁+Y₃+Y₅

Implementations

AND Gate Array

2ⁿ N-input AND gates. Each gate implements one minterm.

+ N inverters
NAND Implementation

For active-low outputs (74LS138). NAND = inverted AND.

Y̅ᵢ = (inputs)' NAND
Tree Structure

Build larger from smaller: 4-to-16 from two 3-to-8s.

Hierarchical decoding
Gate Count

N inverters + 2ⁿ N-input AND gates.

3-to-8: 3 NOT + 8 AND3

Using Decoder as Function Generator

Example: Implement F(A,B,C) = Σm(1, 2, 4, 7) using 3-to-8 decoder

  A ────┬────┐
  B ────┼────┤ 3-to-8   Y₀ ────────────────────
  C ────┼────┤ Decoder  Y₁ ────┐
        │    │          Y₂ ────┼───┬─── OR ──── F
  EN=1 ─┴────┘          Y₃ ────│   │
                        Y₄ ────┼───┤
                        Y₅ ────│   │
                        Y₆ ────┘   │
                        Y₇ ────────┘

F = Y₁ + Y₂ + Y₄ + Y₇ = Σm(1,2,4,7)

RTL Implementation

VERILOG decoder.v 
// 2-to-4 Decoder with Enable
module decoder_2to4 (
    input      [1:0] a,
    input            en,
    output reg [3:0] y
);

always @(*) begin
    if (en)
        y = 4'b0001 << a;  // Shift 1 to selected position
    else
        y = 4'b0000;        // Disabled
end

endmodule

// 3-to-8 Decoder (74LS138 style)
module decoder_3to8 (
    input      [2:0] a,       // Address inputs
    input            e1,      // Active-high enable
    input            e2_n,    // Active-low enable
    input            e3_n,    // Active-low enable
    output     [7:0] y_n      // Active-low outputs
);

wire en = e1 & ~e2_n & ~e3_n;

assign y_n = en ? ~(8'b00000001 << a) : 8'b11111111;

endmodule

// Parametrized N-to-2^N Decoder
module decoder #(
    parameter N = 3  // Input bits
)(
    input  [N-1:0]         a,
    input                   en,
    output [(1<<N)-1:0] y
);

genvar i;
generate
    for (i = 0; i < (1<<N); i = i + 1) begin : dec_out
        assign y[i] = en & (a == i);
    end
endgenerate

endmodule

// Using decoder for function generation
module function_from_decoder (
    input  a, b, c,
    output f
);

wire [7:0] m;  // Minterms

decoder #(3) dec (
    .a({a, b, c}),
    .en(1'b1),
    .y(m)
);

// F = Σm(1, 2, 4, 7)
assign f = m[1] | m[2] | m[4] | m[7];

endmodule
VHDL decoder.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

-- 3-to-8 Decoder with Enable
entity decoder_3to8 is
    port (
        a  : in  std_logic_vector(2 downto 0);
        en : in  std_logic;
        y  : out std_logic_vector(7 downto 0)
    );
end decoder_3to8;

architecture behavioral of decoder_3to8 is
begin
    process(a, en)
    begin
        y <= (others => '0');
        if en = '1' then
            y(to_integer(unsigned(a))) <= '1';
        end if;
    end process;
end behavioral;

-- Alternative: Concurrent assignment
architecture dataflow of decoder_3to8 is
begin
    y(0) <= en and not a(2) and not a(1) and not a(0);
    y(1) <= en and not a(2) and not a(1) and     a(0);
    y(2) <= en and not a(2) and     a(1) and not a(0);
    y(3) <= en and not a(2) and     a(1) and     a(0);
    y(4) <= en and     a(2) and not a(1) and not a(0);
    y(5) <= en and     a(2) and not a(1) and     a(0);
    y(6) <= en and     a(2) and     a(1) and not a(0);
    y(7) <= en and     a(2) and     a(1) and     a(0);
end dataflow;

Cascading Decoders

Larger decoders can be built from smaller ones using enable inputs for hierarchical decoding.

4-to-16 from Two 3-to-8 Decoders

A₃ ──────────┬──────────────────────────────┐
             │                              │
             │  ┌─────────────────┐         │
             └──┤ EN  3-to-8 Dec  ├─ Y₁₅-Y₈ (High group)
A₂A₁A₀ ─────────┤ A₂A₁A₀          │         │
                └─────────────────┘         │
                                            │
                ┌─────────────────┐         │
        NOT ◁───┤ EN  3-to-8 Dec  ├─ Y₇-Y₀  (Low group)
A₂A₁A₀ ─────────┤ A₂A₁A₀          │
                └─────────────────┘

When A₃=0: Low decoder enabled  → Y₀-Y₇
When A₃=1: High decoder enabled → Y₈-Y₁₅

5-to-32 from One 2-to-4 + Four 3-to-8

A₄A₃ ────┬────[2-to-4 Dec]───┬── EN₀ → [3-to-8] → Y₀-Y₇
         │                   ├── EN₁ → [3-to-8] → Y₈-Y₁₅
A₂A₁A₀ ──┼───────────────────┼── EN₂ → [3-to-8] → Y₁₆-Y₂₃
         │                   └── EN₃ → [3-to-8] → Y₂₄-Y₃₁
         │
         └── Address lines to all four 3-to-8 decoders

2-to-4 decoder acts as "row select"
Four 3-to-8 decoders share address lines

Gate Savings

Direct 4-to-16 4 inverters + 16 × 4-input ANDs
Cascaded 3-to-8 2 × (3 inv + 8 × 3-AND) + 1 NOT
Benefit Fewer gate inputs, modular design

Applications

Memory Address Decoding

Select memory chip based on address bits. Common in microprocessor systems.

A15-A13 → chip select
Instruction Decoding

Opcode field → control signals. First stage of CPU execution.

Opcode → micro-ops
Seven-Segment Display

BCD to segment decoder is specialized decoder.

4 bits → 7 segments
ROM Implementation

Decoder + OR array = Read-Only Memory structure.

Address → data word

Interview Notes

  • N-to-2ⁿ decoder: N inputs → 2ⁿ one-hot outputs. Exactly one output active per input.
  • Decoder = complete minterm generator. Each output is one minterm of input variables.
  • Any Boolean function: Decoder + OR gate. F = OR of minterms where F=1.
  • 74LS138: 3-to-8 decoder with 3 enables (E₁, E̅₂, E̅₃), active-low outputs.
  • DEMUX vs Decoder: DEMUX routes data input; decoder generates all minterms with enable.
  • Address decoding: High-order bits select chip; low-order bits select within chip.
