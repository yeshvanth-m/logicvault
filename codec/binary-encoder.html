Overview

A Binary Encoder is a combinational circuit that converts 2ⁿ input lines into an N-bit binary code. It performs the inverse operation of a decoder. The encoder assumes that only one input line is active (HIGH) at any given time — a constraint known as one-hot encoding.

When multiple inputs are active simultaneously, the output becomes ambiguous. For systems where multiple inputs may be active, a Priority Encoder should be used instead, which encodes the highest-priority active input.

Pattern Recognition

Each output bit is 1 when the corresponding bit in the input's binary representation is 1:

Y₂ (MSB) OR of inputs where bit 2 = 1 (4,5,6,7)
Y₁ OR of inputs where bit 1 = 1 (2,3,6,7)
Y₀ (LSB) OR of inputs where bit 0 = 1 (1,3,5,7)

Implementations

OR Gate Implementation

Each output is OR of input lines where that bit position is 1.

Gate Count (8-to-3)

Three 4-input OR gates. No AND gates needed for basic encoder.

Valid Input Detection

Add "valid" output to detect if any input is active.

RTL Implementation

VERILOG encoder.v 
// 4-to-2 Binary Encoder
module encoder_4to2 (
    input      [3:0] in,
    output reg [1:0] out,
    output           valid
);

assign valid = |in;  // Any input active?

always @(*) begin
    case (in)
        4'b0001: out = 2'd0;
        4'b0010: out = 2'd1;
        4'b0100: out = 2'd2;
        4'b1000: out = 2'd3;
        default: out = 2'bxx; // Invalid/undefined
    endcase
end

endmodule

// 8-to-3 Binary Encoder (OR-based)
module encoder_8to3 (
    input      [7:0] in,
    output     [2:0] out,
    output           valid
);

// OR-based implementation
assign out[2] = in[4] | in[5] | in[6] | in[7];
assign out[1] = in[2] | in[3] | in[6] | in[7];
assign out[0] = in[1] | in[3] | in[5] | in[7];

assign valid = |in;

endmodule

// Parametrized Encoder (2^N to N)
module encoder #(
    parameter N = 3  // Output bits (8-to-3)
)(
    input  [(1<<N)-1:0] in,
    output reg [N-1:0] out,
    output              valid
);

integer i;

assign valid = |in;

always @(*) begin
    out = {N{1'b0}};
    for (i = 0; i < (1<<N); i = i + 1) begin
        if (in[i])
            out = i[N-1:0];
    end
end

endmodule
VHDL encoder.vhd 
library IEEE;
use IEEE.STD_LOGIC_1164.ALL;
use IEEE.NUMERIC_STD.ALL;

-- 8-to-3 Binary Encoder
entity encoder_8to3 is
    port (
        input_lines : in  std_logic_vector(7 downto 0);
        output_code : out std_logic_vector(2 downto 0);
        valid       : out std_logic
    );
end encoder_8to3;

architecture behavioral of encoder_8to3 is
begin
    -- OR-based equations
    output_code(2) <= input_lines(4) or input_lines(5) or 
                       input_lines(6) or input_lines(7);
    output_code(1) <= input_lines(2) or input_lines(3) or 
                       input_lines(6) or input_lines(7);
    output_code(0) <= input_lines(1) or input_lines(3) or 
                       input_lines(5) or input_lines(7);
    
    valid <= or input_lines;
end behavioral;

Applications

Keyboard Encoding

Convert key-press (one-hot) to key code. Typically uses priority encoder.

64 keys → 6-bit code
Address Encoding

Chip select lines to binary address in memory systems.

Bank select → bank number
Interrupt Controller

Multiple interrupt lines encoded to interrupt vector.

IRQ lines → IRQ number
Data Compression

Convert thermometer code to binary for data compression.

One-hot → compact binary

Interview Notes

  • Binary encoder converts 2ⁿ inputs to N-bit output. It's the inverse of a decoder.
  • One-hot constraint: Only one input should be active. Multiple active → undefined output.
  • For multiple active inputs, use a Priority Encoder (encodes highest-priority input).
  • Implementation: Each output bit is OR of inputs where that bit position is 1 in binary.
  • Valid signal: OR all inputs to detect if any input is active (useful for no-input case).
  • Practical use: Keyboard scanners, interrupt controllers, address generation.
