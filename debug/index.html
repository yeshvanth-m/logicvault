Debug Interfaces

Protocols and hardware for chip debug, test, and programming — from boundary scan testing to modern on-chip debug.

Overview

Debug interfaces are essential for IC development, production testing, and field debugging. They provide external access to internal chip state, memory, and control registers.

On-Chip Debug

Halt the CPU, set breakpoints, read registers, single-step through code. Essential for firmware development.

Memory Access

Read and write RAM, flash, and peripheral registers without CPU involvement. Used for flash programming.

Boundary Scan

Test PCB connections by controlling and observing I/O pins. Detects solder defects without physical probes.

Trace

Stream real-time execution data (program counter, data values) for performance analysis and debugging.
