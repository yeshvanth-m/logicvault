Debug Interface IEEE 1149.1

JTAG

Joint Test Action Group — a serial debug and test interface for IC testing, boundary scan, and on-chip debugging. The foundation of chip debug since 1990.

Standard

IEEE 1149.1

Signals

4+1 wires

Speed

~10-50 MHz

Use

Debug + Test

Interface Signals

#

JTAG uses 4 mandatory signals plus an optional reset. All are active-high, synchronous to TCK.

SignalDirectionDescription
TCKInTest Clock — all operations synchronous to rising edge
TMSInTest Mode Select — controls TAP state machine transitions
TDIInTest Data In — serial data input, sampled on TCK rising edge
TDOOutTest Data Out — serial data output, changes on TCK falling edge
TRSTIn (opt)Test Reset — asynchronous reset of TAP controller (active-low)

TAP Controller (State Machine)

#

The Test Access Port (TAP) Controller is a 16-state FSM controlled by TMS. It orchestrates all JTAG operations. TMS is sampled on TCK rising edge.

Shift-DR State

Serial data shifted through selected Data Register. TDI → DR → TDO. Stays here while TMS=0.

Shift-IR State

Serial data shifted into Instruction Register. Selects which DR will be active for subsequent Shift-DR.

Run-Test/Idle

Idle state between operations. Self-test runs here. Returns here after Update states.

Update-DR/IR

Latches shifted data into parallel output register. Data becomes effective on exit.

Reset Sequence: Apply 5 TCK cycles with TMS=1 to reach Test-Logic-Reset from any state. This is guaranteed by the state machine structure.

Data Registers

#

JTAG has two register types: the Instruction Register (IR) selects the active data register, and Data Registers (DR) hold the actual data being transferred.

RegisterWidthPurpose
IR (Instruction)≥2 bitsSelects which DR is connected between TDI/TDO
BYPASS1 bitShortest path TDI→TDO. Used to skip chip in chain.
IDCODE32 bitsManufacturer ID, part number, version. Read-only.
Boundary ScanVariesOne cell per I/O pin. Used for board-level testing.
Device-SpecificVariesDebug registers, memory access, flash programming, etc.

Register Read/Write Operations

#

Every JTAG transaction follows a two-phase pattern: first load an instruction into IR to select the data register, then shift data through the selected DR.

Read Operation (Example: Read IDCODE)

┌─────────────────────────────────────────────────────────────────────┐
│                    STEP-BY-STEP: Read IDCODE                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  PHASE 1: Load Instruction (select IDCODE register)                │
│  ──────────────────────────────────────────────────                 │
│  1. Navigate to Shift-IR state via TMS sequence:                   │
│     TLR → RTI → Select-DR → Select-IR → Capture-IR → Shift-IR      │
│     TMS:  0      1           1            0            0            │
│                                                                     │
│  2. In Shift-IR: clock in IDCODE instruction (e.g., 0x01)          │
│     - Shift LSB first, one bit per TCK rising edge                 │
│     - IR width depends on device (typically 4-8 bits)              │
│     - On last bit, set TMS=1 to exit to Exit1-IR                   │
│                                                                     │
│  3. Navigate to Update-IR (TMS=1), then Run-Test/Idle (TMS=0)      │
│     - Instruction is now latched, IDCODE register is selected      │
│                                                                     │
│  PHASE 2: Read Data Register                                       │
│  ──────────────────────────                                         │
│  4. Navigate to Shift-DR state:                                    │
│     RTI → Select-DR → Capture-DR → Shift-DR                        │
│     TMS:  1           0             0                               │
│                                                                     │
│  5. In Capture-DR: IDCODE value is loaded into shift register      │
│                                                                     │
│  6. In Shift-DR: clock out 32 bits on TDO                          │
│     - Each TCK rising edge shifts one bit out                      │
│     - TDI is don't care for reads (but must be driven)             │
│     - On bit 32, set TMS=1 to exit                                 │
│                                                                     │
│  7. Navigate to Update-DR (TMS=1), then Run-Test/Idle (TMS=0)      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘

Write Operation (Example: Write Debug Register)

┌─────────────────────────────────────────────────────────────────────┐
│                    STEP-BY-STEP: Write to DR                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  PHASE 1: Load Instruction (select target register)                │
│  ──────────────────────────────────────────────────                 │
│  1. Same as read: navigate to Shift-IR                             │
│  2. Shift in instruction code for target DR (e.g., DEBUG_ACCESS)   │
│  3. Update-IR to latch instruction                                 │
│                                                                     │
│  PHASE 2: Write Data Register                                      │
│  ───────────────────────────                                        │
│  4. Navigate to Shift-DR state                                     │
│                                                                     │
│  5. In Shift-DR: clock in data on TDI                              │
│     - Shift LSB first, one bit per TCK rising edge                 │
│     - Previous DR contents shift out on TDO (can be ignored)       │
│     - Continue for full register width                             │
│     - On last bit, set TMS=1 to exit                               │
│                                                                     │
│  6. Navigate to Update-DR (TMS=1)                                  │
│     *** Data is latched on this transition ***                     │
│     - Write takes effect here                                      │
│                                                                     │
│  7. Return to Run-Test/Idle (TMS=0)                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘

Key Point: Data is NOT written until Update-DR!
During Shift-DR, you're only loading the shift register.
The parallel output (actual register value) updates in Update-DR.

Capture vs Shift vs Update

Capture: Load parallel data into shift register. Shift: Move bits serially. Update: Latch shift register to parallel output.

IR vs DR

IR selects which DR is active. Always load IR first to select the target register, then access DR to transfer data.

LSB First

JTAG shifts LSB first. Bit 0 enters first on TDI and exits first on TDO. Plan your bit ordering accordingly.

Simultaneous Read/Write

During Shift-DR, TDI loads new data while TDO outputs old data. Every read is also a write (and vice versa).

Boundary Scan (Original Use Case)

#

JTAG was originally designed for PCB testing — verifying solder joints without physical probes. Each I/O pin has a Boundary Scan Cell (BSC) that can capture, update, or bypass the pin.

SAMPLE Mode

Capture current pin states into BSCs without affecting chip operation. Non-intrusive observation.

EXTEST Mode

Drive output pins from BSC values. Used to test PCB interconnects between chips.

INTEST Mode

Drive core logic from input BSCs. Test internal logic with controlled inputs.

Board-Level Test

Chip A drives via EXTEST, Chip B captures via SAMPLE. Verifies solder joints on PCB traces.

Modern MCU Debug Use

#

In modern microcontrollers, JTAG extends beyond boundary scan to provide full debug capability: halting the CPU, reading/writing memory, setting breakpoints, and programming flash.

Debug Architecture (ARM CoreSight Example)

Debug Operations

OperationHow It Works
Halt CPUWrite to Debug Halting Control Register (DHCSR). CPU stops at current instruction.
Single StepSet C_STEP bit in DHCSR, then resume. CPU executes one instruction and halts.
Read MemoryWrite address to TAR (Transfer Address Register), read from DRW (Data Read/Write).
Write MemoryWrite address to TAR, write data to DRW. Used for flash programming too.
BreakpointWrite address to FPB (Flash Patch and Breakpoint) comparator. CPU halts on match.
Read RegistersHalt CPU, then read from Core Debug Registers via APB-AP.
⚠️ Security: Production devices often disable JTAG via fuses (read-out protection). Once locked, debug access is permanently disabled to protect firmware IP.

Daisy Chaining Multiple Devices

#

Multiple JTAG devices can be connected in a chain. All devices share TCK and TMS. TDO of one device connects to TDI of the next. The debugger shifts data through the entire chain.

Targeting a Device

Put other devices in BYPASS (1-bit DR). Shift through 1 bit per bypassed device + target device's DR length.

Chain Discovery

Shift in 0s, shift out 1s to count devices. Read IDCODE from each to identify them.

RTL (Verilog)

# 
// ═══════════════════════════════════════════════════════
// TAP Controller State Machine
// ═══════════════════════════════════════════════════════
module tap_controller (
    input  wire       tck,
    input  wire       trst_n,
    input  wire       tms,
    output reg  [3:0] state
);

// State encoding
localparam TEST_LOGIC_RESET = 4'hF;
localparam RUN_TEST_IDLE    = 4'hC;
localparam SELECT_DR_SCAN   = 4'h7;
localparam CAPTURE_DR       = 4'h6;
localparam SHIFT_DR         = 4'h2;
localparam EXIT1_DR         = 4'h1;
localparam PAUSE_DR         = 4'h3;
localparam EXIT2_DR         = 4'h0;
localparam UPDATE_DR        = 4'h5;
localparam SELECT_IR_SCAN   = 4'h4;
localparam CAPTURE_IR       = 4'hE;
localparam SHIFT_IR         = 4'hA;
localparam EXIT1_IR         = 4'h9;
localparam PAUSE_IR         = 4'hB;
localparam EXIT2_IR         = 4'h8;
localparam UPDATE_IR        = 4'hD;

always @(posedge tck or negedge trst_n) begin
    if (!trst_n) begin
        state <= TEST_LOGIC_RESET;
    end else begin
        case (state)
            TEST_LOGIC_RESET: state <= tms ? TEST_LOGIC_RESET : RUN_TEST_IDLE;
            RUN_TEST_IDLE:    state <= tms ? SELECT_DR_SCAN   : RUN_TEST_IDLE;
            SELECT_DR_SCAN:   state <= tms ? SELECT_IR_SCAN   : CAPTURE_DR;
            CAPTURE_DR:       state <= tms ? EXIT1_DR         : SHIFT_DR;
            SHIFT_DR:         state <= tms ? EXIT1_DR         : SHIFT_DR;
            EXIT1_DR:         state <= tms ? UPDATE_DR        : PAUSE_DR;
            PAUSE_DR:         state <= tms ? EXIT2_DR         : PAUSE_DR;
            EXIT2_DR:         state <= tms ? UPDATE_DR        : SHIFT_DR;
            UPDATE_DR:        state <= tms ? SELECT_DR_SCAN   : RUN_TEST_IDLE;
            SELECT_IR_SCAN:   state <= tms ? TEST_LOGIC_RESET : CAPTURE_IR;
            CAPTURE_IR:       state <= tms ? EXIT1_IR         : SHIFT_IR;
            SHIFT_IR:         state <= tms ? EXIT1_IR         : SHIFT_IR;
            EXIT1_IR:         state <= tms ? UPDATE_IR        : PAUSE_IR;
            PAUSE_IR:         state <= tms ? EXIT2_IR         : PAUSE_IR;
            EXIT2_IR:         state <= tms ? UPDATE_IR        : SHIFT_IR;
            UPDATE_IR:        state <= tms ? SELECT_DR_SCAN   : RUN_TEST_IDLE;
            default:          state <= TEST_LOGIC_RESET;
        endcase
    end
end

endmodule


// ═══════════════════════════════════════════════════════
// JTAG Top Module with IR and Bypass DR
// ═══════════════════════════════════════════════════════
module jtag_tap #(
    parameter IR_WIDTH = 4
)(
    input  wire              tck,
    input  wire              trst_n,
    input  wire              tms,
    input  wire              tdi,
    output wire              tdo,
    output wire              tdo_en,
    output wire [IR_WIDTH-1:0] ir_out
);

wire [3:0] state;
tap_controller u_tap (
    .tck     (tck),
    .trst_n  (trst_n),
    .tms     (tms),
    .state   (state)
);

// State decode
wire shift_ir  = (state == 4'hA);
wire shift_dr  = (state == 4'h2);
wire update_ir = (state == 4'hD);
wire capture_dr = (state == 4'h6);

// Instruction Register
reg [IR_WIDTH-1:0] ir_shift, ir_reg;
always @(posedge tck or negedge trst_n) begin
    if (!trst_n) begin
        ir_shift <= {IR_WIDTH{1'b1}};  // BYPASS instruction
        ir_reg   <= {IR_WIDTH{1'b1}};
    end else begin
        if (shift_ir)
            ir_shift <= {tdi, ir_shift[IR_WIDTH-1:1]};
        if (update_ir)
            ir_reg <= ir_shift;
    end
end
assign ir_out = ir_reg;

// Bypass Register (1-bit)
reg bypass_reg;
always @(posedge tck) begin
    if (capture_dr)
        bypass_reg <= 1'b0;
    else if (shift_dr)
        bypass_reg <= tdi;
end

// TDO Mux
reg tdo_reg;
always @(negedge tck) begin
    if (shift_ir)
        tdo_reg <= ir_shift[0];
    else if (shift_dr)
        tdo_reg <= bypass_reg;
    else
        tdo_reg <= 1'b0;
end

assign tdo = tdo_reg;
assign tdo_en = shift_ir | shift_dr;

endmodule

Interview Notes

#
Q: How many states in TAP FSM?

16 states. Key states: Test-Logic-Reset, Run-Test/Idle, Shift-IR, Shift-DR, Update-IR, Update-DR.

Q: How to reset TAP from any state?

Apply 5 TCK cycles with TMS=1. State machine always reaches Test-Logic-Reset within 5 TMS=1 transitions.

Q: JTAG vs SWD?

JTAG: 4-5 wires, daisy-chain capable, boundary scan. SWD: 2 wires, ARM-only, no daisy-chain, smaller/cheaper.

Q: Why is BYPASS instruction all 1s?

Captures pattern if IR is uninitialized. Fail-safe — unknown devices don't block chain.

Q: What's in the IDCODE?

32 bits: 4-bit version, 16-bit part number, 11-bit manufacturer ID, LSB always 1.

Q: How does boundary scan test PCBs?

Chip A: EXTEST drives pins. Chip B: SAMPLE captures pins. Compare captured vs expected to detect opens/shorts.

Q: When does a write take effect?

On Update-DR transition. During Shift-DR you're loading the shift register; the parallel output updates only in Update-DR state.

Q: Steps to read a register?

1) Shift-IR: load instruction selecting target DR. 2) Update-IR. 3) Capture-DR: load data. 4) Shift-DR: clock out bits. 5) Update-DR.
