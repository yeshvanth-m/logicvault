Debug Interface ARM ADIv5

SWD (Serial Wire Debug)

ARM's 2-wire alternative to JTAG. Provides full debug access using only clock and bidirectional data lines. Standard on all Cortex-M microcontrollers.

Standard

ARM ADIv5/v6

Signals

2 wires

Speed

Up to 50+ MHz

Use

Debug Only

Interface Signals

#

SWD uses just 2 signals — dramatically simpler than JTAG's 4-5 wires. This reduces PCB area, connector size, and cost.

SignalDirectionDescription
SWCLKIn (to target)Clock signal from debug probe. Data sampled on rising edge.
SWDIOBidirectionalData line. Host drives during request, target drives during response.
GND—Ground reference (required for signal integrity)
SWOOut (optional)Serial Wire Output for trace data (uses JTAG TDO pin)
Pin Sharing: SWD shares pins with JTAG — SWCLK = TCK, SWDIO = TMS. Most Cortex-M chips support both protocols on the same pins.

Protocol Basics

#

SWD is a packet-based protocol with defined request/response phases. Unlike JTAG's state machine, SWD uses explicit turnaround cycles when SWDIO direction changes.

Host → Target

Host (debugger) drives SWDIO during request packet: 8-bit header specifying read/write, DP/AP, and address.

Turnaround

One SWCLK cycle where both sides release SWDIO. Prevents bus contention during direction change.

Target → Host

Target drives SWDIO during ACK (3 bits) and optionally RDATA (32 bits + parity).

Idle

Host drives SWDIO low between transactions. At least 8 idle cycles recommended between packets.

Line Reset

SWD line reset: drive SWDIO high for at least 50 SWCLK cycles, then drive low for 2 cycles. This resets the SW-DP state machine.

Read/Write Operations

#

Example: Read IDCODE

// Read IDCODE from DP register A[3:2] = 0x0
// Request: Start=1, APnDP=0, RnW=1, A=00, Parity, Stop=0, Park=1

Request packet:  1 0 1 00 1 0 1  = 0xA5
                 │ │ │ ── │ │ └─ Park
                 │ │ │    │ └─── Stop  
                 │ │ │    └───── Parity (XOR of bits 1-4)
                 │ │ └────────── A[3:2] = 00
                 │ └──────────── RnW = 1 (Read)
                 └────────────── APnDP = 0 (DP)

SWCLK:  ─┐┌─┐┌─┐┌─┐┌─┐┌─┐┌─┐┌─┐┌───────── ...
SWDIO:  ─1─0─1─0─0─1─0─1─│trn│ACK│─RDATA──│P│

Response: ACK = 001 (OK)
          RDATA = 32-bit IDCODE value
          Parity = XOR of all RDATA bits

Example: Write to AP

// Write to AP register (e.g., TAR = Transfer Address Register)
// Request: Start=1, APnDP=1, RnW=0, A=01 (TAR), Parity, Stop=0, Park=1

Request:  1 1 0 01 0 0 1  = 0x8B
          │ │ │ ── │ │ └─ Park
          │ │ │    │ └─── Stop
          │ │ │    └───── Parity
          │ │ └────────── A[3:2] = 01 (TAR register)
          │ └──────────── RnW = 0 (Write)
          └────────────── APnDP = 1 (AP access)

After ACK, host sends:
  - 32-bit WDATA (address to write to TAR)
  - 1-bit parity

DP and AP Registers

#

SWD accesses two types of registers: Debug Port (DP) for protocol control, and Access Port (AP) for memory/peripheral access.

Debug Port (DP) Registers

A[3:2]ReadWritePurpose
0x0IDCODEABORTRead: Device ID. Write: Abort stuck transactions.
0x1CTRL/STATCTRL/STATControl/status: power, errors, transaction results.
0x2RESENDSELECTRead: Repeat last data. Write: Select AP and bank.
0x3RDBUFF—Read: Buffered AP read result. Triggers actual read.

Access Port (AP) Registers — AHB-AP Example

A[3:2]RegisterPurpose
0x0CSWControl/Status: size, auto-increment, etc.
0x1TARTransfer Address Register (memory address to access)
0x2—Reserved
0x3DRWData Read/Write (actual memory data)

Memory Read Sequence

// Read 32 bits from memory address 0x20000000

1. SELECT: Write to DP SELECT register
   - Select AHB-AP (APSEL = 0)
   - Select bank 0

2. CSW: Write to AP CSW register  
   - Set Size = Word (32-bit)
   - Enable auto-increment if reading multiple words

3. TAR: Write to AP TAR register
   - Write address: 0x20000000

4. DRW: Read from AP DRW register
   - First read returns previous data (posted read)
   - Must read RDBUFF to get actual result

5. RDBUFF: Read from DP RDBUFF
   - Returns actual memory value at 0x20000000
Posted Reads: AP reads are posted — the first DRW read triggers the memory access but returns stale data. Read RDBUFF (or do another DRW read) to get actual data.

JTAG ↔ SWD Switching

#

Most ARM devices with SWD also support JTAG on the same pins. A special sequence switches between protocols.

Switch from JTAG to SWD

1. Send at least 50 TCK cycles with TMS=1 (JTAG line reset)
2. Send 16-bit JTAG-to-SWD sequence: 0xE79E (LSB first)
3. Send at least 50 SWCLK cycles with SWDIO=1 (SWD line reset)
4. Send 8 idle cycles (SWDIO=0)
5. Read IDCODE to verify SWD mode

Sequence on wire:
TCK/SWCLK:  ────────────────────────────────────────────
TMS/SWDIO:  1111...1111 0111100111100111 1111...1111 00000000
            ← 50+ ones → ←── 0xE79E ──→  ← 50+ ones → ← idle →

Switch from SWD to JTAG

1. Send at least 50 SWCLK cycles with SWDIO=1 (SWD line reset)
2. Send 16-bit SWD-to-JTAG sequence: 0xE73C (LSB first)
3. Send at least 5 TCK cycles with TMS=1 (JTAG reset)
4. Perform normal JTAG operations

RTL (Verilog)

# 
// ═══════════════════════════════════════════════════════
// SWD Protocol Decoder (Simplified)
// ═══════════════════════════════════════════════════════
module swd_decoder (
    input  wire        swclk,
    input  wire        rst_n,
    input  wire        swdio_in,
    output reg         swdio_out,
    output reg         swdio_oe,    // Output enable
    output reg  [31:0] rdata,
    output reg         rdata_valid,
    input  wire [31:0] wdata,
    output reg  [1:0]  addr,        // A[3:2]
    output reg         ap_ndp,      // 0=DP, 1=AP
    output reg         rnw,         // 0=Write, 1=Read
    output reg         request_valid
);

// States
localparam IDLE     = 4'd0;
localparam REQUEST  = 4'd1;
localparam TURNAROUND1 = 4'd2;
localparam ACK      = 4'd3;
localparam TURNAROUND2 = 4'd4;
localparam DATA     = 4'd5;
localparam PARITY   = 4'd6;

reg [3:0] state;
reg [5:0] bit_cnt;
reg [7:0] request_reg;
reg [31:0] data_reg;
reg parity_calc;

// Request packet fields
wire start_bit  = request_reg[0];
wire apndp_bit  = request_reg[1];
wire rnw_bit    = request_reg[2];
wire [1:0] addr_bits = request_reg[4:3];
wire parity_bit = request_reg[5];
wire stop_bit   = request_reg[6];
wire park_bit   = request_reg[7];

// Parity check
wire parity_ok = (parity_bit == (apndp_bit ^ rnw_bit ^ addr_bits[0] ^ addr_bits[1]));

always @(posedge swclk or negedge rst_n) begin
    if (!rst_n) begin
        state <= IDLE;
        bit_cnt <= 0;
        swdio_out <= 1'b0;
        swdio_oe <= 1'b0;
        request_valid <= 1'b0;
        rdata_valid <= 1'b0;
    end else begin
        case (state)
            IDLE: begin
                swdio_oe <= 1'b0;
                request_valid <= 1'b0;
                rdata_valid <= 1'b0;
                if (swdio_in == 1'b1) begin  // Start bit
                    state <= REQUEST;
                    bit_cnt <= 1;
                    request_reg[0] <= 1'b1;
                end
            end
            
            REQUEST: begin
                request_reg[bit_cnt] <= swdio_in;
                if (bit_cnt == 7) begin
                    state <= TURNAROUND1;
                    bit_cnt <= 0;
                end else begin
                    bit_cnt <= bit_cnt + 1;
                end
            end
            
            TURNAROUND1: begin
                // Both sides release line
                swdio_oe <= 1'b0;
                state <= ACK;
                bit_cnt <= 0;
                // Decode request
                ap_ndp <= apndp_bit;
                rnw <= rnw_bit;
                addr <= addr_bits;
                request_valid <= parity_ok && start_bit && !stop_bit && park_bit;
            end
            
            ACK: begin
                swdio_oe <= 1'b1;  // Target drives
                case (bit_cnt)
                    0: swdio_out <= 1'b1;  // ACK[0] = OK
                    1: swdio_out <= 1'b0;  // ACK[1]
                    2: begin
                        swdio_out <= 1'b0; // ACK[2]
                        state <= TURNAROUND2;
                        bit_cnt <= 0;
                    end
                endcase
                if (bit_cnt < 3) bit_cnt <= bit_cnt + 1;
            end
            
            TURNAROUND2: begin
                swdio_oe <= 1'b0;
                state <= DATA;
                bit_cnt <= 0;
                parity_calc <= 0;
            end
            
            DATA: begin
                if (rnw) begin
                    // Read: target drives data
                    swdio_oe <= 1'b1;
                    swdio_out <= rdata[bit_cnt];
                    parity_calc <= parity_calc ^ rdata[bit_cnt];
                end else begin
                    // Write: host drives data
                    swdio_oe <= 1'b0;
                    data_reg[bit_cnt] <= swdio_in;
                    parity_calc <= parity_calc ^ swdio_in;
                end
                
                if (bit_cnt == 31) begin
                    state <= PARITY;
                    bit_cnt <= 0;
                end else begin
                    bit_cnt <= bit_cnt + 1;
                end
            end
            
            PARITY: begin
                if (rnw) begin
                    swdio_out <= parity_calc;
                    rdata_valid <= 1'b1;
                end else begin
                    // Check parity for write
                    if (swdio_in == parity_calc) begin
                        rdata <= data_reg;  // Store received write data
                        rdata_valid <= 1'b1;
                    end
                end
                state <= IDLE;
                swdio_oe <= 1'b0;
            end
            
            default: state <= IDLE;
        endcase
    end
end

endmodule

Interview Notes

#
Q: How many wires does SWD need?

2 wires: SWCLK (clock) and SWDIO (bidirectional data). Plus GND reference.

Q: What is a turnaround cycle?

One clock cycle where both host and target release SWDIO. Prevents bus contention when direction changes.

Q: What are the ACK responses?

OK (001), WAIT (010), FAULT (100). WAIT means retry, FAULT means check error registers.

Q: What is a posted read?

AP reads are pipelined. First DRW read initiates transfer but returns previous data. Read RDBUFF for actual result.

Q: How to switch JTAG→SWD?

50+ clocks with TMS=1, then 16-bit sequence 0xE79E, then 50+ clocks with SWDIO=1, then idle cycles.

Q: SWD vs JTAG advantage?

SWD: Fewer pins (2 vs 4-5), faster for debug. JTAG: Supports daisy-chain, boundary scan, non-ARM devices.
